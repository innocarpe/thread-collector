# 작년 여름 업비트 API가 12분간 불안정했을 때

작년 여름 업비트 API가 12분간 불안정했을 때.

내 주문 엔진에 재시도 로직이 없었으면 중복 주문으로 얼마가 날아갔을지 모른다.
지금은 3단계로 들어가 있다.
API 타임아웃이면 20ms 대기 후 재시도.
그래도 실패면 주문 취소 후 재호출.
n번째까지 모든 시나리오에 대비가 돼있다.

자동매매의 핵심은 코딩 실력이 아니라 예외처리다.
