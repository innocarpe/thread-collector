# 최악 시나리오 하나 그려본다

최악 시나리오 하나 그려본다.

내일 아침 일어났더니 주거래 거래소가 해킹당해서 출금 정지.

서버에 올려놓은 API 키는 무효화됐고, 현물 포지션 60%가 거기 묶여 있다.

이런 상황에서 뭘 할 수 있나?

답은 "지금 미리 준비해놓은 것만".

그래서 내 자산은 거래소 3곳에 분산되어 있고, 각 거래소 비중은 40%를 넘기지 않는다.

콜드월렛에 35%는 항상 떼놓는다.

해킹 시나리오는 가정이 아니라 실제 역사다. 마운트곡스, FTX, 해킹당한 거래소 수십 개.

한 번 일어날 일이 아니라 다시 일어날 일로 본다.

내 시스템의 최대 약점은 전략이 아니라 거래소 리스크다.

이걸 인정하면 자산 배치가 달라진다.

덧붙이자면, 거래소 분산하면 슬리피지랑 관리 부담이 늘어난다. 수익률이 좀 깎이는 건 감수해야 함. 그게 보험료라고 생각하면 편함.
