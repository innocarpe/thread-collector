# 오늘은 저의 업무 셋팅을 공개합니다

오늘은 저의 업무 셋팅을 공개합니다.제가 평소에 어떤 도구를 쓰고, 어떤 구조로 일하고,어떻게 하나의 서비스를 만들어서 운영까지 가져가는지, 

에이전프리너 답게, 가능한 한 있는 그대로 최대한 숨김없이 다 알려드릴 수 있도록 하겠습니다. 

AI시대에 개발자들은 실제로 어떻게 일하는지, 궁금하셨던 분들이 많으실 텐데, 오늘 그 궁금증이 상당 부분 해소되실 겁니다. 끝까지 보시면

"아 이래서 이 사람이 이 방식으로 하는 거구나"가 확실하게 이해되실 겁니다.

저는 메인 개발 도구로 클로드코드를 씁니다. 이외에 코덱스와 제미나이CLI 그리고 오마이오픈코드, 커서, 안티그라비티도 가끔씩 사용하고 있습니다.

그런데 그냥 툴을 열고 "이거 만들어줘" 하는 방식은 당연히 아닙니다.

프로젝트를 시작하기 전에, CLAUDE.md파일을 먼저 작성합니다.
쉽게 말해 "에이전트 운영 매뉴얼"입니다.

이 프로젝트에서 코드 스타일은 이렇게 쓴다,
커밋 메시지는 이 형식을 따른다,
테스트는 반드시 이 기준을 통과해야 한다,
보안 관련 로직은 이런 원칙으로 짠다,

이런 규칙들을 AI에게 미리 전부 주입하는 겁니다. 이걸 왜 하냐면, AI는 규칙을 안 주면 매번 다른 스타일로 코드를 짭니다. 프로젝트가 커질수록 코드가 뒤엉켜서 나중에 사람이 손댈 수 없는 상태가 되기 때문입니다.

다음으로 MCP 서버 셋팅을 합니다. MCP는 Model Context Protocol의 약자인데, 쉽게 말씀드리면 AI에게 "외부 도구를 직접 쓸 수 있는 손"을 달아주는 겁니다.

저는 현재 

웹검색은 주로 브레이브서치, 패럴웹서치, 링크업, 엑사서치, 타빌리를 연결하고,

브라우져자동화는 퍼피티어와 유료는 브라우저베이스 API를 연결해서 다양한것을 처리하고 있습니다.

이외 마소와 애져 공식문서를 바로 탐독하는 에이드로이드와 XSS나 인젝션, OWASP Top10주제와 같은 취약점에 대해서 SecDim보안코딩랩을 이용하고 있습니다.

이외에 깃허브, 리니어(이슈 관리), 슬랙, 구글워크스페이스, 엑사검색, 구글검색, 메모리, 그리고 자체 제작 워크플로우 서버까지, 전부 MCP로 연결해놨습니다.

그외 필요한 MCP는 각 프로젝트별로 별도로 구성해서 처리하여 사용하고 있습니다.

작업에 따라 MCP를 자유자재로 연결하게되면 뭐가 달라지냐면,
예를들어 제가 "이 문제 작업해줘"라고 하면, AI가 알아서 이슈를 읽고, 정확한 지점과 그와 연관된 부분을 제대로 파악하고, 테스트를 돌리고, PR(풀 리퀘스트)을 올리고, 설정에 따라 슬랙으로 "작업 완료됐습니다" 알림을 보내거나, 브라우져 팝업으로 로컬팝업을 만들어서 브라우져팝업에 자동으로 알림이 뜨게 합니다.

한 문장으로 지시해도, 매우 많은 단계의 워크플로우가 자동으로 돌아갑니다.
