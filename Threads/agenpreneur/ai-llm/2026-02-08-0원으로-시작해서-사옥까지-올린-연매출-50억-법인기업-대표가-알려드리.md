# 0원으로 시작해서 사옥까지 올린, 연매출 50억 법인기업 대표가 알려드리는, 바이브코딩 팁

0원으로 시작해서 사옥까지 올린, 연매출 50억 법인기업 대표가 알려드리는, 바이브코딩 팁.

1. AI가 답해준 코드를 파악하려고 하지마라.
어느정도 읽어보는건 좋다. 근데 확인 기준을 항상 AI답변만을 기준으로 찾으려 하지마라. 확인은 실제 결과물로 확인해야 한다. 초보자가 저지르는 가장 큰 실수는, AI가 응답해준걸 바탕으로 파악하려는거다. 애초에 불가능하다. 직접 코딩을 해보면, 단 5줄마저도 굉장히 오랫동안 머리를 쓰고 생각해야하는 경우도 상당히 많다. 수십줄, 수백줄 코드 보는건 의미가 없다. 예를들어 a라는 텍스트를 자동으로 b라고 바뀌고 그 액션이 이뤄지는 동시에 c는 d로 바꾸게 하고 싶다고 치자. 그리고 이 작업이 완료되면, 이를 원상복구 안되게 고정시키고 싶게 만들고 싶다고 해보자. 그러면 코드만 보면 매우 간단하다. 하지만 이걸 직접 구현해내려하면, 몇줄도 안되는게, 생각도 많이해야하고 시간도 꽤 걸린다. 즉 코드만 보고 파악할 수 있다는 오만을 버리도록하라.

2. AI가 답해준 내용을 파악하려고 하지마라.
이 역시 어느정도 읽어보는건 좋다. 하지만 AI가 답해준걸 읽어보는것은 생각보다 파악이 느리다. 그것만으로 진짜 문제를 찾을 순 없다. AI는 그저 그부분만 작업해준걸 디테일하게 설명해줬을뿐이다. 진짜 해야할일은 당신이 실제로 프로덕트에 접속해서 살펴보면서 생긴다. 그러니 AI를 보면서 코딩하겠다는 마음은 버려라. AI기능이 강화될수록 억지로라도 실제 결과물을 직접 검토하는 시간을 늘려야한다. MCP나 스킬을 달아서 스크린샷찍게하고 그걸 연속으로 찍어서 에이전트 루프로 만들어서, 그에 대한 프롬프트 설계후 이런것까지 뭘해야할지 자동화하겠다고? 아니면 게이트웨이단으로 던져서 알아서 모든걸 파악하게 해보겠다고? 어느정도는 잘해줄것이다. 하지만 결국은 사용할 수 없는 제품이 될것이다. 바이브코딩에서 차별화는 당신이 얼마나 실제 결과물을 오래보면서 정확한 추가 명령을 내려줬느냐이다.

3. 절대 딸깍이라 생각하지말아라.
바이브코딩 강사들이 사기꾼이라 생각치는 않는다. 단지 실전경험이 없거나 너무 적어서 현실을 모르는것뿐이라 생각한다. 절대 딸깍이란 없다. 아무리 사전기획을 잘해놓아도 결국 중요한건 무한 테스트이다. 테스트를 하면서, 기획자가 방향을 제대로 잡아가야하는게 바이브코딩의 진짜 힘이다. 그렇다고 무작정 테스트만 해서도 안된다. AI가 준 응답을 읽어보고 부족한 부분을 반드시 자기가 직접 설계하고 말해줘야한다. 안그러면 과장좀 보태서 일주일을 허비할수도 있다. AI에게 모든걸 다 맡기려는 유혹을 항상 참아야한다. "너가 알아서 해줘", "~~이렇게 하려고해. 어떻게 해야할지 깊게 고민해서 처리해줘", 이런 명령들...... 습관되면, 쓰레기 프로덕트 만들기 딱 좋은 습관이 자리잡는다고 생각하면 될거같다.

4. AI의 텍스트 답변 마지막에는 항상 "사실 저는 일단 처리해야할것만 처리한것뿐입니다. 여기엔 모자란게 너무 많을겁니다. 하지만 제가 모든걸 파악할 순 없습니다. 그래서 이부분만 제 생각에 최선을 다했습니다. 그러므로 여기에서 더 확장하고 더 좋은 서비스나 유용한 서비스가 되기위해선 사용자님이 실제로 결과물을 확인하시고, 더욱 깊게 고민하여 저에게 피드백을 반드시 줘야합니다" 라고 응답하게 하라. 그러면 오히려 실제 사용되는 프로덕트까지 매우 잘 이끌어갈 수 있다.
