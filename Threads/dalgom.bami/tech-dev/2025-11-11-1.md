# 일단 비즈니스 요구사항에 맞춰 개판으로 파일 몇개에 몰아서 짠다

1. 일단 비즈니스 요구사항에 맞춰 개판으로 파일 몇개에 몰아서 짠다.
2. 이 수천줄 코드가 잘 돌아가나 검증할 수 있는 테스트 코드를 어느정도 만든다.(클코 활용하면 잘 만듦)
3. 만들면서 구조가 사업이 커지면서 어떻게 확장될지 고민을 계속 해둔다.
4. 이젠 한 눈에 코드가 들어오지 않고 천줄을 돌파하며 나조차도 파악이 어려울 때 클로드 코드에 리팩토링을 요청한다.
5. 이 때 정확한 디자인 패턴과 요구사항을 전달하는게 중요하다. plan.md파일을 하나 만들어서 여기에 지금까지 내가 고민한 리팩토링 방향을 최대한 자세히 입력한다.
6. 클로드 코드 plan모드에 위 파일을 맥락으로 주고, 리팩토링 요청. 기존 테스트 케이스 돌릴 수 있는 명령어 주면서 테스트 코드 돌려가며 자가검증하도록 요청
7. 클코의 엄청난 실력 구경하기

어제 이 방식으로 2500줄짜리 핵심 비즈니스 로직 코드를 정말 깔끔하고 확장성 있는 아키텍쳐로 단 1시간만에 변경했습니다.
