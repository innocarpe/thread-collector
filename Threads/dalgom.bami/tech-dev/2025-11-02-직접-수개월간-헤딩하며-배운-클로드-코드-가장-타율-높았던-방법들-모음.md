# 직접 수개월간 헤딩하며 배운 클로드 코드 가장 타율 높았던 방법들 모음

직접 수개월간 헤딩하며 배운 클로드 코드 가장 타율 높았던 방법들 모음

클로드코드의 성능이 과거대비 좋아진 것도 있겠지만, 나는 과거에는 이 친구가 들고온 코드를 다 revert하고 몇 번 다시 반복하면서 날리는 시간이 많았는데, 요새는 바로 잘 동작하면서도 퀄리티 좋은 코드를 아주 높은 타율로 얻어내게 되었어. 이게 꼭 정답이거나 그런건 아니겠지만 꽤 좋았던 방법들을 공유해볼게

1. 바로 뭐뭐뭐 만들어줘! 이거 해줘! 프롬프트 쳐서 작업하지 않기. 어떤 단위의 일을 한 번에 맡겨야 얘가 헷갈리지 않고 잘할지 모호한 부분을 첫 프롬프트에서 줄여주는게 가장 중요함. 어떤 기능을 구현해야한다고 했을 때, 전체 기능 중 일부를 어떻게 잘라서 요청할지, 테스트케이스는 어떤걸 통과시키라고 할지, 그냥 한 번에 구현해보라고 할지 제대로 고민해보면 좋음. 

나는 주니어 개발자에게 업무를 할당할 때 정리해서 보내주던 짧은 문서 초안 정도를 넣어주니까 정말 잘하더라.

2. 첫 요청은 plan mode + ultrathink로 요청

플랜모드에 나한테 모호한 부분들을 몇가지 되묻는 기능이 최근 추가되었어. 이 질문들이 정말 날카롭고 예리해. 내가 주니어들한테 기대한 부분이 이런 역량인데 이 능력을 클코가 갖추게 되었음. 

그래서 나는 이 플랜을 내가 마음에 들 때까지 충분히 내 의견도 더하고, 이친구 의견도 받아들이고 하면서 논의를 하고, 최종 플랜을 꼭 정독하고 있어. 여기까지 잘 되었으면 어지간하면 코드가 내 생각과 엇나갈일이 드라마틱하게 줄어듦

ultrathink는 솔직히 진짜 효과있나 싶은 생각이 있긴한데 약간 미신 개념 + 무지개 나오는거 이뻐서 넣고있음

3. 여러 프로젝트를 함께 작업하고 있다면(ex. 백엔드,프론트,앱 각각 프로젝트)
root폴더를 하나 만들고 이 아래에 저 프로젝트들 다 옮겨오고 root폴더에서 claude를 실행하면 압도적으로 효율 좋아짐. 

프론트 구현하다가도 백엔드의 의도도 조금은 파악하면서 진행하더라니까

4. 서버 기능 api로 먼저 만들고, 해당 api를 통해 나올 수 있는 sample response를 만들어달라고함. 이 reponse json을 가져와서 프론트에 주면서 dummy data로 ui구현 먼저 제대로 해둠. 다 됐으면 이제 dummy data를 실제 서버 호출로 연결하는 작업을 지시. 박수 나올정도로 잘함.

5. 이 기능이 잘 된다고 쳤을 때 test 코드 몇 개 정도를 만들라고 시킴. 해당 테스트 실행할 수 있는 스크립트 명령어 준비. 클코한테 작업 시킬 때, 이 테스트 호출 스크립트 실행해가며 에러 로그 보면서 니가 계속 고쳐라 지시. 테스트 코드 자체를 수정하면 커서로 갈아타겠다 문구 넣을것. 꽤 오래 걸리지만 제대로된 결과가 나올 확률 압도적으로 상승.

다만 몰래 테스트코드 고쳐두는 경우도 있으니 테스트 코드 수정 했나 안했나 git diff로 꼭 체크

6. 이래도 이상한 짓을 하기 때문에 언제든지 revert할 수 있도록 잘된것까진 commit꼭 하기

7. 일단 한 파일에 줄이 길어지더라도 쭉 구현해두고, 다 되면 내가 원하는 디자인패턴으로 리팩토링 시키면 진짜 잘함. 5번에서 만들어둔 테스트코드도 있기에 정말 잘함. 코드 엄청 깔끔해짐. 리팩토링은 원래 스펙 다 구현되고 하는게 젤 효율적임 귀찮아서 그렇지. 좋은 코드나 구조에 대한 죄책감없이 일단 다 구현하고, 마지막에 클린업 작업을 해주면 좋은 품질의 코드로 뚝딱 바뀜. 다만 현재 구조에 적합한 설계를 지시할 수 있는 지식과 경험이 있어야하는 것은 변함 없음

8. 새로 시작 하는거면 /clear 꼭 하기. 사실 기본적인건데 의외로 모르는 사람들이 많더라고..?

총평: 클로드 코드는 정말 일 잘하는 주니어다. 경기침체 여파도 있겠으나 얘네 때문에 주니어 채용이 어려워진건 맞다고 생각한다. 다만 시니어들은 항상 건강 및 노안 이슈가 있기에 AI 끼고 겨우 버티고 있는 시니어들 퍼지면 결국 현 주니어들에게도 기회 많이 올거다. 육아 하면서 매번 느끼는건데 하루하루 참 체력이..
