# @hon_coding 님이 외국 트위터를 정리해주신 내용을 계속 벼르고 있다가 오늘 개발할 앱 디자인 만드는

@hon_coding 님이 외국 트위터를 정리해주신 내용을 계속 벼르고 있다가 오늘 개발할 앱 디자인 만드는데에 한번 사용해봤어. 

사용해 본 소감은 미쳤다는 소리밖에 나오지 않는 것 같아.

여러 ai 디자인 생성툴을 사용해봤지만 크레딧도 너무 비싸고 결과물도 솔직히 찍어내기식이라서 진짜 초기 틀 잡는데에만 썼는데 

이 방법을 이용하면 앱 아이디어와 컨셉을 설명하면 그에 맞는 화면 "기획"까지 해주면서도 그에 맞는 UI 설계, 그리고 재사용 가능한 UI 컴포넌트들이 생겨서 진짜로 로직만 붙이면 되는 수준이야. 

이 방법이 사기인 이유는 나만의 컴포넌트들이 생기기 때문에 새 화면을 기획하거나 내 화면 ui를 수정할 때 고질적인 문제였던 엄청 긴 소스코드를 뱉느라 claude가 죽어버리는 일이 많은데 컴포넌트들로만 구성해서 새 화면을 만들기 때문에 엄청 간결하게 결과물이 나오고 토큰 낭비가 적어서 결과물이 정말 잘 나옴.
