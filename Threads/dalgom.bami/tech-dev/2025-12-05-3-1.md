3. Next/Image
최적화의 일등공신. script 외에 또 하나의 병목이 되는 이미지 처리를 굉장히 스무스하게 처리해줌.

기본적으로 lazy loading이 적용되어 있어서 뷰포트에 이미지가 들어올 때 로딩을 시작함. 즉 사용자가 스크롤해서 해당 이미지가 보일 시점에 가서야 로드되기 때문에 초기 로딩 시 불필요한 네트워크 요청을 줄일 수 있음.

브라우저가 지원하면 WebP나 AVIF 같은 최신 포맷으로 자동 변환해서 제공함. 같은 품질 대비 파일 크기가 훨씬 작아서 로딩 속도에 직접적인 이점이 있음.

반응형 이미지 처리도 자동임. srcset을 알아서 생성해줘서 디바이스 화면 크기에 맞는 적절한 해상도의 이미지를 제공함.

width와 height를 필수로 지정하게 해서 CLS 문제도 방지함. 이미지가 로드되기 전에 이미 공간이 확보되어 있어서 로딩 중에 레이아웃이 밀리는 현상이 없음. 이건 점수에도 직접적인 영향을 줌.

priority설정으로 FCP 대응도 쉬움
