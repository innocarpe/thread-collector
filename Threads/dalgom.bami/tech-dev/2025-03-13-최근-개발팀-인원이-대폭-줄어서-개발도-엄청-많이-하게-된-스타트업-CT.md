# 최근 개발팀 인원이 대폭 줄어서 개발도 엄청 많이 하게 된 스타트업 CTO가 알려주는 CursorAI 꿀팁

최근 개발팀 인원이 대폭 줄어서 개발도 엄청 많이 하게 된 스타트업 CTO가 알려주는 CursorAI 꿀팁

프로덕트의 크기는 엄청 커졌고 한동안 매니징과 사업개발만 하다가 다시 하루 5시간넘게 코딩을 하고 있어. 처음에는 조금 막막했는데 Cursor덕에 그래도 꽤 많은 feature들을 개발해낼 수 있었어. 

세 달간 사용하면서 커서 사용하면서 느낀 점을 순서대로 설명해볼게. 이 과정을 이해하고 잘 쓰면 무조건 커서가 날아다닐거야.

1. Cursor는 codebase로 검색 시 여러 파일을 참고하긴 하지만 별 필요없는 파일들이 context로 들어가 성능저하가 되기도 한다.
2. 그렇다고 codebase 기반으로 하거나 composer를 쓰지 않으면 내 파일 전체적인 구조 이해를 잘 못한다.
3. 이를 해결하기 위해 선행되어야 하는 작업은 커서에게 내 긴 코드를 적당한 단위로 모듈화하고 여러 파일로 쪼개달라고 요청해. composer가 잘함

(이어서)

4. 그 다음 쪼개진 파일들이 잘 쪼개졌는지 확인하고 어떤 파일들인지 대략적으로 알아둠
5. 이걸 말하려고 빌드업을 한건데 이렇게 쪼개두고, 내가 특정 기능을 만든다고 쳤을 때 그 기능에 영향 줄 것 같은 파일들만 태그해서 context로 활용하도록 한다. 
6. 이상한 파일 들쑤시지 않고 상당히 정확하게 코딩하는 커서를 흐뭇하게 바라본다.

결국 커서를 잘 쓰려면 프로젝트를 잘 구조화해서 내 머리속에 잘 담아두는게 중요하고, 그래 이 기능은 이것들이 수정되어야지 하는 파일들만 내가 정확하게 선정하는게 성능향상에 굉장히 중요해! 요새 context 최대 길이가 꽤 길어져 다 넣어도 상관없지만 그래도 쓸데없는 정보 없이 강력하게 연관된 코드만 컨텍스트를 넣어주는게 가장 성능 향상에 좋아.
