# 우선 supabase는 백엔드 어플리케이션 레이어를 하나 없이 직접 클라이언트랑 연결되어 작업하다보니 일반적

우선 supabase는 백엔드 어플리케이션 레이어를 하나 없이 직접 클라이언트랑 연결되어 작업하다보니 일반적인 CRUD(행 생성,조회,수정,삭제)를 넘어서 복잡한 로직을 구현하는게 원칙적으로는 어려워. 특히 사용자가 어떤 액션을 했을 때, 해당 사용자가 건들면 안되는 table을 동시에 수정해야하는 경우 문제가 커지지.

근데 실제로 서비스는 하나의 액션이 일어났을 때, 여러 테이블의 데이터를 동시 수정해야하는 케이스가 많은데 이걸 AI 코딩 툴에 시키면 얘네는 갑자기 Trigger라는걸 정의해서 쓰거나 edge function 또는 RPC를 만들어서 쓸거야. 

우선 Trigger는 현대 백엔드 어플리케이션에선 거의 쓰지 않는 방식이야.

아니 거의 쓰지 않는다는 너무 순화된 표현같아. 특수한 경우가 아니면 남발하면 팀원에게 심한 욕먹는 방식이야.

간단히 설명하자면 db의 특정 테이블이 insert, update, delete 될 때, 자동 실행되는 쿼리를 정의하는 거야.
