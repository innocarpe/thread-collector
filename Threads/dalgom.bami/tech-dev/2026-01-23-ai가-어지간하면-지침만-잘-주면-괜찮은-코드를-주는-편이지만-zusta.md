# ai가 어지간하면 지침만 잘 주면 괜찮은 코드를 주는 편이지만 zustand store 가져올 땐 rule로 만들어둬도 자꾸 아래 처럼 하는 경...

ai가 어지간하면 지침만 잘 주면 괜찮은 코드를 주는 편이지만 zustand store 가져올 땐 rule로 만들어둬도 자꾸 아래 처럼 하는 경우가 많다.

// AI가 선호 - 짧고 읽기 쉬움
const { a, b, c } = useMyStore()

// 해야할 코드
const a = useMyStore((state) => state.a)
const b = useMyStore((state) => state.b)
const c = useMyStore((state) => state.c)

아무래도 학습 코드 자체가 성능 최적화에 좀 덜 신경쓴 코드에 weight가 많이 갔거나 아니면 그냥 짧아서 저렇게 하고 싶어서 그런건지 여튼 말을 징그럽게 안들어서 이 부분은 항상 손코딩 하고 있다. 

store크기에 따라서 편차가 좀 있겠지만 두 코드의 성능은 컴포넌트와 데이터 구조에 따라 별 차이가 없을 수도, 말도안되는 성능 저하를 초래할 수도 있다.
