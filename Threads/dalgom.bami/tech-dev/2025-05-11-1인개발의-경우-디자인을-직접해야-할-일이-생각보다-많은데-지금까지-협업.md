# 1인개발의 경우 디자인을 직접해야 할 일이 생각보다 많은데, 지금까지 협업 경험상 시니어 디자이너분들은 확실

1인개발의 경우 디자인을 직접해야 할 일이 생각보다 많은데, 지금까지 협업 경험상 시니어 디자이너분들은 확실히 명확한 규칙이 느껴지는 margin 조합을 잘 쓰더라. margin에서 오는 짬바차이가 확실히 있다.

margin이 중구난방인 경우 전체적인 안정도가 많이 떨어지게 되고 뭔가 말로는 설명할 수 없는 이상함이 많이 느껴지는데 이 margin과 컴포넌트들의 크기, 적당한 color palette만 있어도 꽤 그럴싸한 디자인 결과물을 기대해볼 수 있다

특히 요새는 cursor로 디자인을 하는 경우가 많은데 이런 Rule들을 cursor rule로 세팅해두고, 주요 컴포넌트들을 미리 이 Rule에 기반해서 디자인시스템처럼 만들어 둔 뒤, rule에서 이 규칙들과 design system에 있는 컴포넌트들만 사용해서 디자인을 하라고 하면 꽤 안정적인 디자인이 나옴

추가팁을 주자면 그냥 내가 막 짜두고 미리 정의한 디자인 규칙에 맞게 리팩토링 하라고 요청해도 엄청난 효과가 있다
