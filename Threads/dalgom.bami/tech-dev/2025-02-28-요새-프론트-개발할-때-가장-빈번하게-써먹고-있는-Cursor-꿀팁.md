# 요새 프론트 개발할 때 가장 빈번하게 써먹고 있는 Cursor 꿀팁

요새 프론트 개발할 때 가장 빈번하게 써먹고 있는 Cursor 꿀팁

1. 일단 좋은 코드 구조 생각하지 말고 하나의 거대한 UI 트리로 만들고 스타일도 다 함께 정의해서 지저분한 코드로 빠르게 만든다. 로직도 다 넣는다.
2. cursor에 중간중간 사용한 스타일 전부다 하단의 const style로 정리/연결해달라고 한다.
3. export하는 메인 컴포넌트의 길이가 너무 기니까 적당한 단위로 쪼개달라고 한다 
4. UI와 코드의 비즈니스 로직 결합이 심각하다 이걸 hook 또는 zustand와 같은 store로 빼라고 지시
5. 최종 코드 리뷰 (거의 고칠 것 없긴 함)
6. cursor에게 이 컴포넌트를 보고 테스트 코드 작성해 지시(테스트 코드를 진짜 잘 짠다)
