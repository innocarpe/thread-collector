# App router

1. App router
Next 13.4 버전부터 지원하기 시작한 Next의 독특한 SSR방식. React 19부터 지원하기 시작한 Server component를 적극 지원함. 과거 pages를 쓰는게 아닌 app router를 쓰는 것만으로도 이미 꽤 엄청난 초기로딩 이점이 있음.

보통 Performance에 가장 영향 주는게 script 크기임. react 앱 특성상 프론트의 모든 코드를 빌드해서 몇개의 js파일로 압축해서 보통 배포하는데 실제 초기 로딩에는 필요 없는 것들도 함께 로딩됨. 

Server component를 이용하면 해당 화면이 만들어지는 과정에서 필요한 스크립트들은 서버에서 실행되고 프론트로는 전달되지 않음. 최종 ui와 약간의 js만 프론트로 전달 됨. 이 부분에서 js크기를 일단 크게 절약할 수 있음.

브라우저는 script를 만나면 일단 html파싱을 멈추게 되고 이게 최종 사용자에게 첫 화면이 빠르게 뜨는데에 병목이 됨.

2. Client Component
App Router에서 기본적으로 모든 컴포넌트는 Server Component임. 클라이언트 측 상호작용이 필요한 경우에만 파일 상단에 'use client' 지시어를 추가해서 Client Component로 명시함.
Client Component가 필요한 경우는 다음과 같음:

useState, useEffect 등 React hooks 사용 시
onClick, onChange 같은 이벤트 핸들러 사용 시
브라우저 전용 API(localStorage, window 객체 등) 접근 시 등등

주의할 점은 Client Component가 "클라이언트에서만 렌더링된다"는 의미가 아님. 서버에서 초기 HTML로 프리렌더링되고, 클라이언트에서 hydration을 통해 인터랙티브하게 됨. 다만 Server Component와 달리 해당 컴포넌트의 JavaScript 번들이 클라이언트로 전송됨. 그래서 꼭 필요한 곳만 클라이언트컴포넌트를 쓰는게 중요
