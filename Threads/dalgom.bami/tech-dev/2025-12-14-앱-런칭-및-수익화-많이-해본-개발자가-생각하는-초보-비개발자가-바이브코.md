# 앱 런칭 및 수익화 많이 해본 개발자가 생각하는 초보 비개발자가 바이브코딩으로 성공적으로 앱을 스토어까지 런칭하기 위해 선택하면 좋은 스택들 모...

앱 런칭 및 수익화 많이 해본 개발자가 생각하는 초보 비개발자가 바이브코딩으로 성공적으로 앱을 스토어까지 런칭하기 위해 선택하면 좋은 스택들 모음

(읽기 전 주의) 개발이건 사업이건 유일한 최고의 길 딱 하나만 존재한다고 생각하지 않습니다. 개인의 상황에 따라 잘 먹히는 수많은 방법이 있다고 생각하고 이 글에서 추천한 것들이 완전체라고 주장하다고 싶은 것이 아니니 분란 댓글은 받지 않습니다. 

이 글은 개발을 하나도 모르는 사람이 실제 앱 개발하고 런칭해서 운영하고 수익화하는데에 있어서 걸림돌이 되는 문제를 최대한 만나지 않도록 구성해본 것이고 개발에 익숙해지면 얼마든지 더 좋은 선택지들이 많습니다.

A to Z를 해주는 replit, createanything, rork, base44 같은 바이브코딩 툴들은 크레딧(현금)이 너무 많이 들어 솔직히 저도 제대로 런칭할 정도까지는 못만들어봐서 일단 제외하고 그냥 개발자처럼 맨땅에서 만든다고 쳤을때를 가정하고 써 봤습니다.

1. 맥 환경에서 개발
윈도우에서 시작 하면 셀프로 난이도 올려서 하는겁니다.
(개발자들중에서도 초반에 어려워하는 사람들 꽤 있습니다.)

윈도우도 환경 세팅 초반에 잘만 하면 실제 현업 개발자들도 많이 쓰는 환경이지만 아무 것도 모른다는 가정하에 무조건 헤매게 되어있습니다. 과거보다는 나아졌다고는 하나 여전히 꽤 진입장벽이 있습니다. 맥은 개발환경 세팅이 일단 윈도우대비 훨씬 수월합니다. 

놀랍게도 개발자들이 꼭 허세 때문에 맥을 사는게 아니긴 합니다. 개발자 컨퍼런스 가면 죄다 맥북들고 있는 이유가 다 있습니다.

2. Flutter
저는 React native, android, swift 모두 경험해보고 상황에따라 돌려쓰긴 하지만 개발 아예 모르는 초보한테 하나 추천하라면 무조건 flutter입니다.

우선 우리를 도와줄 코딩 AI가 헷갈릴 일이 가장 적습니다.
이건 프레임워크의 우열 이런 문제가 아니고 flutter 자체가 저 중에서 가장 최근 나오기도 했고, legacy가 쌓인게 없어서 코딩 AI들이 척척 잘 하는 편입니다.

React native를 선택하면 expo환경인지 아닌지에 따라 다르고, 무슨말인지 잘 이해안되실 수도 있지만 의존성 관리쪽에서 지옥이 열려서 여튼 빌드가 깨질 일이 정말 많습니다. 이건 React쪽의 특성때문이기도 한데 여기는 오픈소스 진영 의존도가 높습니다. 그래서 여러 라이브러리를 이것저것 가져다가 내 입맛대로 잘 조합해서 쓰는게 중요한데 이 과정에서 빌드가 깨지는 경우가 많습니다. 개발자들이야 헤딩하며 고치겠지만 보통 바이브코더들이 여기서 gg치고 관둡니다.

플러터는 React보다는 추가해야하는 라이브러리가 적은 편입니다. 애초에 flutter 프레임워크 자체에 꽤 이것저것 포함되어있어서 외부 라이브러리 추가가 훨씬 적습니다. 이게 바이브코딩에서 왜 장점이냐면 ai가 다른 라이브러리를 웹에서 리서치하면서 추가하지 않고도 대부분 기능을 구현해줄 수 있기 때문입니다. 

flutter로 구현 시 UI 스타일 구현도 AI가 꽤 잘하는 편입니다. 아무래도 flutter가 기본으로 들어가있는 ui 스타일이나 애니메이션 등등이 꽤 괜찮게 들어가있어서 ai와 함께라면 괜찮은 ui를 뽑아낼 수 있습니다.
