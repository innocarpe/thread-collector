# 폭탄 1번: RLS(Row Level Security)

폭탄 1번: RLS(Row Level Security)
일반적인 백엔드와 다르게 supabase는 RLS를 많이 쓰도록 권장되어있고 이게 큰 장점 중 하나임. 백엔드 서버를 별도로 하나 띄우지 않고도 백엔드 로직이 포함된 어플리케이션을 만들 수 있으니까. 

다만 이 클라이언트 중심 보안 체계는 잘 쓰면 이론상 보안 문제가 있을 정도는 아니지만, 잘 쓰는게 백엔드 지식 없이는 거의 불가능하다는게 문제임.

RLS를 쉽고 간단히 설명하자면 user_id = 4인 사람이 있다면, 이 user_id = 4가 접근가능한 Table Row를 정의해서 그 Row들만 클라이언트가 쓸 수 있게 하자가 컨셉임. 즉 이 사람은 어차피 다른 사람의 데이터를 조작하거나 조회하는게 막혀있으니 client token으로 백엔드 호출을 해도 이 토큰이 탈취되지 않는 이상 보안 이슈가 없는 것 + 탈취 되어도 이 사람 데이터 밖에 못건들기 때문에 비교적 안전이 컨셉임

문제는 여기서 부터 시작인데
