# 이런 할루시네이션이 나왔을 때 live서비스는 큰 타격을 입게 돼

이런 할루시네이션이 나왔을 때 live서비스는 큰 타격을 입게 돼.

이렇게 만들어서 잘 운영중인 Todo 앱이 있다고 칠게.
이번에 Todo 항목에 스케쥴기능을 넣고 싶어서 요청했어.
근데 이 AI놈이 db를 이리저리 건들다가 migration코드를 잘못짜서 기존 사용자 todo list 데이터가 다 날아간거야. 이거 감당가능할까?
이게 과장이 아니고 live 서비스에서는 현업 개발자도 자기가 짠 코드인데도 하는 실수라는거야..

엑셀에서 뭐 수정하다가 서식이나 함수 깨져서 혼나본 경험이 있다면 공감할거야.

LLM의 아키텍쳐, 현재처럼 내부 동작원리를 모르는 블랙박스 상태인 이상 다른 패러다임의 모델이 나오지 않는 이상 바이브코딩은 아직까진 개발자의 전유물이 될거야.(개발자에겐 유효함) 진지하게 서비스를 운영하고 싶다면 개발 공부를 하는 것이 더 빠를 것 같아.

혹시 이런 migration이슈나 유지보수까지도 완벽히 되는 Ai를 찾는다면 내가 제일 필요하니까 꼭 알려주고 😉
