# 개발 지식 없는 바이브코딩이 딱 프로토타입 수준에서 끝나야 하는 이유

개발 지식 없는 바이브코딩이 딱 프로토타입 수준에서 끝나야 하는 이유

라이브 서비스에서 적용 했다간 큰일날 액션을 과감하게 지를 때가 간혹 있는데 진짜 깜짝 놀랄때가 많음. 무조건 검토 해줘야함.

아래 인용 글도 보기만 해도 소름돋고, 나는 이런 일들이 있었음 

- 테스트 케이스 실패 시, 테스트케이스를 조작해서 우회
- 못하게하니까 테스트 코드 ci/cd에서 빼버림
- 이것도 못하게 하니까 하드코딩으로 테스트코드에서 원하는 값 나오게 통과하게 우회
- dependency문제 때문에 제대로된 라이브러리 버전으로 변경 해야되는건데 이 전 버전의 library 버전을 바꿔버려서 억지로 맞추거나 force install 강요
- n이 10000 이상 되면 서버 박살날 코드 추가
- db migration 파일을 만들 때 기존 데이터 보존 신경 안쓰고 alter table 시원하게 일발 장전
- 테이블 명 변경 한답시고 테이블 삭제 후, 테이블 재생성..

결론: 개발 배워서 쓰자
