# 수많은 앱을 개발하고 실제 운영하면서 느낀건 첫 버전 완성은 가장 쉽고 재밌는 부분이라는 것이다

수많은 앱을 개발하고 실제 운영하면서 느낀건 첫 버전 완성은 가장 쉽고 재밌는 부분이라는 것이다. 바이브 코딩 툴은 이 과정이 5분만에 되었다고 신나서 홍보한다.

나는 바이브 코딩 툴을 쓰지는 않지만, 간단한 prototype을 눈으로 보고 싶을 때, 그리고 실제 이런 poc 형태의 제품이 필요한 초기 창업자에게는 정말 용도에도 맞고 좋은 툴이라고 생각한다.

근데 한 가지 아쉬운 점은 앱 개발의 진짜 고통은 그 다음부터인데 이걸 툴들이 해결해주진 못하고 있다는 것이다. 

유저 늘어나면 예상 못한 버그 터지고 엣지케이스 폭발하고 OS 업데이트로 UI 깨지고 정책 업데이트되었다고 스토어에서 앱 업뎃안하면 앱 내린다고 협박하는거 대응해야하고 서버 비용 갑자기 감당안되어서 비용 최적화 코드 만들어야하고, 코드 히스토리 파악하고 뭘 건드리면 뭐가 깨질지 예측하고 데이터 마이그레이션 고민하고 옛날버전 이악물고 업데이트안하면서 뭐 안된다고 리뷰다는 사람들 대응하고 이게 진짜 일이다.
