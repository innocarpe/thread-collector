# 그래서 주로 나는 작은 불씨의 데이터 획득을 위해서라도 뻔한 결과가 예상되더라도 일단 작고 빠르게 실행 해보

그래서 주로 나는 작은 불씨의 데이터 획득을 위해서라도 뻔한 결과가 예상되더라도 일단 작고 빠르게 실행 해보는걸 주장하는 편이었어. 내 경험상 편향 가능성이 있는 적은 표본의 데이터라도 대부분 의사결정에 정말 큰 영향을 줬고 좋은 방향으로 이끌어줬던 것 같거든. 

어차피 이걸로 하루 아침에 성공하자는게 아니고 불씨나 가능성만 확인하기 위해서 구성하기 때문에 리소스도 최소화했고 검증에 필요한 예산도 정말 작게 잡는 편이었어.

그리고 위에 언급한 사람들은 아래와 같은 메시지들로 날 많이 괴롭혔던 것 같아

"솔직히 저는 이번에 이렇게 해보는거 잘 안 될 것 같아요. 저라면 안써요" 
"빠르게 테스트해보는건 좋긴 한데 더 잘 할 방법이 있지 않을까요? 이거에 리소스 쓰느니 조금 더 개선해서 한번에 잘 가는게 낫지 않아요?"
"아니 이건 이 개선안이 누가봐도 무조건 좋은데 왜 A/B테스트를 해야해요. 솔직히 시간 낭비같아요"
