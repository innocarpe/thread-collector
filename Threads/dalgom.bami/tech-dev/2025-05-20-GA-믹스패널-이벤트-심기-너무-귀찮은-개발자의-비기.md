# GA, 믹스패널 이벤트 심기 너무 귀찮은 개발자의 비기

GA, 믹스패널 이벤트 심기 너무 귀찮은 개발자의 비기

앱을 하나 만들면 기본적인 analytics이벤트들을 수십개 심어야하고 이게 생각보다 아주 귀찮은데 이 방법으로 광명을 찾았어. 

앱에 아예 이벤트가 하나도 없을 때 가장 효과적임

1. 커서에 이 앱의 핵심퍼널을 설명하고 필요한 주요 이벤트를 plan.md 파일을 생성해서 이벤트 생성 계획(어떤함수를 쓸 지 등) 하단에 이벤트명과 주요 이벤트 프로퍼티를 리스트로 만들어달라고 하기, 각 항목 앞에 [ ] 넣어달라고하기
2. plan.md에 정의된 이벤트명, 프로퍼티를 눈으로 체크(여기를 가장 신중하게 검토)
3. 커서에 plan.md를 context로 주고 이제 이벤트 호출 함수를 하나씩 실제로 적합한곳에 구현하고 [✅] 표시하고 다음으로 넘어가게 지시
4. 흐뭇하게 감상. 다 안했는데 끊기면 continue입력
5. 실행해보며 이벤트 로그보며 검증

지켜보면 이게 마누스야 커서야? 생각이 듦😁
