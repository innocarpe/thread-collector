# 서버기능 넣으려면 supabase로 하면 된다고해서 백엔드 개발 모르는데 서비스 일단 어찌어찌 바이브코딩으로

서버기능 넣으려면 supabase로 하면 된다고해서 백엔드 개발 모르는데 서비스 일단 어찌어찌 바이브코딩으로 만들었고 어찌됐건 유저 생겼는데 기능업데이트하다가 디비 데이터 날아가면 어떡하지?에 대해서 답변 당장 못하는 사람은 이 글을 꼭 저장해두고 바로 실천하세요.

docker로 local supabase띄우기, migration file 관리, staging supabase 따로 배치, supabase db push, supabase migrations up..

이 명령어들 뭔지 모르겠고 써본적 없으면 일단 아래처럼이라도 하세요. 그래야 나중에 개발자들이 여러분들 데이터 살려줄 수 있습니다. 언젠간 저한테 고마워 할겁니다..

1. 가장 쉬운 방법: 최소한 supabase pro플랜을 쓰세요, 자동으로 최근 7일에 대해서 db백업이 매일 돌아갑니다. 뭔가 데이터 날아가고 망했다 싶을때 백엔드 개발자를 수소문해서 구해서 supabase에서 백업은 해줬다는데.. 어떻게 살려줄 수 있어? 라고 물어봅니다.

2. 저거보단 어려운데 돈 안드는 방법:

우선 supabase client를 설치합니다. (https://supabase.com/docs/guides/local-development)

보통은 가이드대로 개발자라면 supabase link를 통해 프로젝트를 연결하지만 여러분들은 이상한 명령어 모르고 날려서 순식간에 나락으로 갈 수 있습니다. 정확하게 이게 뭐하는 건지 인지하기 전엔 절대 연결하지마세요. 참고로 클로드코드나 이런놈들 퍽하면 오 깔끔하게 supabase db reset 이거 한번 돌려보죠? 이러면서 제안하는데 이게 나락 지름길입니다. 데이터 다 날아감 저거 돌리면(계속)
