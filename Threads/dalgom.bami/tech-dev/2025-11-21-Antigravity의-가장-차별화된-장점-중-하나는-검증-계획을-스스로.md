# Antigravity의 가장 차별화된 장점 중 하나는 검증 계획을 스스로 세우고 진짜 자동으로 검증을 반복 

Antigravity의 가장 차별화된 장점 중 하나는 검증 계획을 스스로 세우고 진짜 자동으로 검증을 반복 한다는거다. 그리고 이게 개발할 때 말도안되는 효율 향상을 가져다주고 한 번의 프롬프트로 제대로된 결과물을 건질 확률을 엄청나게 늘려준다.

- 서버면 test script를 작성해서 돌려보거나 서버 실행해서 서버에 요청해서 터미널 로그 등을 확인해서 검증한다
- 웹이면 Antigravity안에서 크롬을 띄워서 실제 화면을 멀티모달로 자기가 보면서 실제 뭐 간격이 안맞는지 ui구현 제대로 했는지도 보고, 실제로 콘솔 로그도 보면서 디버깅한다.
- 만약 jest 테스트 환경이 구축 안되어있다면 그냥 스크립트 하나 짜서 돌려보고 지운다

위 상황이 모두 안되는 경우 수동으로 어떻게 정확히 검증하면 되는지 implementation plan에 적혀있는데 가끔 도움이 된다.

첫 버전이 이정도면 앞으로의 발전이 정말 기대되는 IDE인 것 같다. 
오늘도 클코를 켤 일이 없었다.
