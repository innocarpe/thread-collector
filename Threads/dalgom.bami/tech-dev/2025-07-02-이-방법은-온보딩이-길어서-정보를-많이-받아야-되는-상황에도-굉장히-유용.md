# 이 방법은 온보딩이 길어서 정보를 많이 받아야 되는 상황에도 굉장히 유용해

이 방법은 온보딩이 길어서 정보를 많이 받아야 되는 상황에도 굉장히 유용해.

정보 입력 전에 대뜸 로그인창 나오면 여기서 다 이탈하거든.

그래서 이 방법을 추천해.

anonymous 로그인을 자동으로 시킴 -> 사용자가 입력하면 서버에 다 저장하기 시작 -> 최종적으로 입력 끝났을 때 회원가입 창 보여주기 

supabase는 이렇게 했을 때 기존 anonymous 유저에 소셜로그인 정보를 자연스럽게 연결해줘. 

또 로그인 없이 계속 쓰더라도 데이터는 유지되다가 사용자가 데이터 저장이 안전하게 되는걸 원하는 시점에 선택해서 소셜 로그인 시키는 것도 엄청 쉽게 구현할 수 있겠지?
