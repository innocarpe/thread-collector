# SEO관련 글을 쓰다보면 자주 받는 질문이 "SEO 잘하려면 뭘 공부해야 하나요

SEO관련 글을 쓰다보면 자주 받는 질문이 "SEO 잘하려면 뭘 공부해야 하나요?"인데, 솔직히 이 질문에 간단하게 답하기가 어렵다. SEO는 단일 스킬이 아니기 때문이다.

SEO 전문가라고 하면 많은 사람들이 키워드 리서치나 메타태그 최적화 정도를 떠올린다. 물론 이것도 중요하지만 빙산의 일각에 불과하다. 실제로는 마케팅 전략을 수립할 줄 알아야 하고, 검색 의도를 파악해 컨텐츠를 기획할 수 있어야 하며, 고객이 어떤 여정을 거쳐 전환하는지 퍼널 관점에서 이해해야 한다. 여기까지는 그나마 마케터의 영역이라 할 수 있다.

문제는 여기서 끝이 아니라는 것이다(거의 생활의 달인급). 테크니컬 SEO를 제대로 하려면 웹사이트의 크롤링, 인덱싱, 렌더링 구조를 이해해야 하고, 이는 자연스럽게 백엔드와 프론트엔드 지식을 요구한다. 서버 응답 속도, API 구조, 자바스크립트 렌더링 방식, 구조화된 데이터 마크업 등 개발 영역의 이해 없이는 근본적인 문제 해결이 불가능한 부분도 많다.
