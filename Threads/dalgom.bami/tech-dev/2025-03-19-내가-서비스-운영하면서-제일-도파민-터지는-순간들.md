# 내가 서비스 운영하면서 제일 도파민 터지는 순간들

내가 서비스 운영하면서 제일 도파민 터지는 순간들

나랑 비슷한 연배라면 다들 어렸을 때 장래희망에 과학자라고 많이들 적었을 것 같은데 
실시간으로 실험해보는 재미가 있어서 뭔가 과학자가 된 느낌으로 재밌게 일하고 있어

현재 본업으로 운영중인 스타트업의 서비스는 MAU가 60만 정도 되는 웹페이지이다 보니 수많은 A/B테스트를 동시 진행 가능하고 배포 후 표본이 금방 쌓이다보니 결과도 금방 나오는 편이라서 번뜩이는 아이디어가 나왔을 때 실험을 해보고 얼마나 임팩트가 있었는지 구경하는 맛에 재밌게 하고 있어. 내 원래 스레드 아이디도 lets.abtest였던 이유도 최근 ab테스트 결과가 너무 재밌어서 저렇게 지었던 것 같아

A/B테스트를 수백개 해보면서 느끼는건 어느정도 자리잡은 서비스는 A/B테스트 해보면 원안을 이기지 못하는 경우가 90% 가까이 된다는 사실.. 이 사실을 아는 사람이 별로 없을거야. 실제로 optimizely에서 통계 발표하기도 했고.. (계속)
