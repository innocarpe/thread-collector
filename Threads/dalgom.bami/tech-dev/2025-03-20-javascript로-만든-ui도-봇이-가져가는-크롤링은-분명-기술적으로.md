# javascript로 만든 ui도 봇이 가져가는 크롤링은 분명 기술적으로는 가능하다

javascript로 만든 ui도 봇이 가져가는 크롤링은 분명 기술적으로는 가능하다.
문제는 이 javascript parser bot은 정말 중요한 페이지에 아주 조금의 시간만 할당해준다. 당연하게도 http 요청만으로 수집하는 일반 크롤링 봇 대비해서 브라우저를 돌리는 급의 큰 리소스가 들어가기 떄문에 모든 페이지에 이걸 돌리기 보다는 구글이 생각하기에 좋은 페이지에 추가적으로 해주는 정도다.

사이트 별로 이 크롤링 일/월별 크레딧 한도가 정해져있다. 초기 사이트일 수록 크레딧은 굉장히 적게 할당되며, 이 크레딧 소진을 막기위해 필요없는 페이지에는 no index를 걸거나 sitemap에서 뺴는 등 여러 노력을 해야 정상적으로 내 페이지들이 잘 인덱싱 된다.

내 모든 페이지가 js bot만 이해할 수 있다는 것은 대부분의 페이지는 봇이 수집 안하다는 것이다.

즉 CSR 페이지는 기술적으로 크롤링이 가능하다일 뿐이지 실전에서 그러면 안하느니만 못한것이다.
