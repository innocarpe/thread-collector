# [제대로 설명하는 AI에이전트(기초편)]

[제대로 설명하는 AI에이전트(기초편)]

클로드 코워크로 다음과 같이 파일을 정리할 수 있습니다.

저는 수업에서 다음과 같은 사례를 맨처음 다릅니다.

AI에이전트라고 말하지만 사실 뭔지도 모르겠고,
RAG, 스킬 등의 개념이 등장하면서
GPT는 별로네, 코덱스가 짱이네, 클로드 코드를 해야 하네,

이제는 공부해야 하는 게 기하급수적으로 많아지고 있죠.

오늘부터는 간단한 예시를 출발점으로
AI 에이전트가 무엇인지 차근차근 이야기해보겠습니다

방법은 매우 간단합니다 
1.정리안된 폴더에 파일을 모두 때려박는다
2.코워크를 실행시킨다.
3.파일을 종류별로 정리해달라는 프롬프트를 입력한다.

진짜 별거없죠? 우측사진처럼 3번클릭하고
폴더위치 지정해주고 그거정리해

첫영상은 이 3가지만 해서 한거에요 

그럼 왜 저는 이걸 에이전트의 예시라고하는걸 까요???

아니 대체 AI에이전트란 무엇일까요?

여기서 중요한 건 AI가 파일을 이해했다는 사실이 아니에요
이미 Chat gpt나 제미나이에 파일을 넣는건 너무나도 익숙한 일이죠?

그럼 뭐가다른걸까요?

우리가 해준건 파일에 대해 읽고 쓰는 행동을 할 수 있는 권한을 준다
이게 핵심이에요

일반적인 지금까지 쓰던 LLM 툴이 그저 읽고 출력을하는거였다면

이제 MCP 스킬 API등의 도구가 붙으면 파일을 읽고, 수정하고, 새로 만들고, 이름을 바꾸고, 필요하면 셸 명령까지 실행할 수 있다는거죠

공식 문서도 Claude Code가 도구를 가질 때 비로소 agentic하게 동작하며, 기본은 읽는다고 표현해요 

결국 주는 권한에 따라 달라지는다는거고 파일 수정 삭제, bash 실행같은 별도 승인 대상으로 둔다고 설명합니다.

클코를 사용하는데 1 1 1 1 1을 연속으로 누르는게 이런거죠

1980년대까지 올라가는 그렇고 

우리가 기억해야 하는건 REACT입니다.

ReAct(Reasoning + Acting) 추론과 결합의 합성어로 프롬프트 기법에서 행동까지 들어왔다 이해해도되요

“정리 안 된 폴더에 파일을 넣고, 분류해달라고 시켰다”

이건 이제 내가원하는걸 추론하고 일을했다.

왜이게 에이전트의 대표적인예시인지 이해가시나요?

그냥 글을 만들어주는 질의응답만 하는게 아니라

목표를 받고, 현재 상태를 읽고, 작업을 몇 단계로 쪼개고, 실행 결과를 다시 받아 다음 행동을 정하는 루프를 수행한 것이죠

Anthropic 공식 문서도 복잡한 작업에서 모델이 일을 단계로 나누고, 실행하고, 배운 것을 바탕으로 조정한다고 설명하는게 이예시죠
