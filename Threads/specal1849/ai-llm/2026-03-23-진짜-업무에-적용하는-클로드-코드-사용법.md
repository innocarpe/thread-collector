# 진짜 업무에 적용하는 클로드 코드 사용법

진짜 업무에 적용하는 클로드 코드 사용법

많은 분들이 비개발자가 사용하는 클로드 코드에 대해 묻습니다.
“왜 굳이 클로드 코드나 코덱스를 써야 하나요?”

이번에 클로드 코드 강의 기획을 잡고 있기도 하고,

공개할 수 있는 선에서 제가 쓰고 있는 
책 마케팅 기획을 잡는 데 사용한 방법을 써보려합니다.

안티그라비티 내에서 클로드 코드를 통해 
분석과 기획을 진행한 과정에 대해 이야기해 보죠

사용한 분석법
필립 코틀러의 4P, STP, AARRR 해적 지표,
SWOT 분석, 북극성 지표

자, 우선 먼저 기본적인 사전지식부터 짚고 가보겠습니다.

핵심만 말씀드리면

클로드 코드는 
1.사용자의 요청을 바탕으로 파일을 읽음
2.에이전트 루프 기반 필요한 내용을 정리, 수정, 명령을 실행

어렵죠? 직관적으로 말하면
정보를 수집하고, 작업을 수행한 뒤, 결과를 확인
이렇게 이해하시면 충분합니다.

즉, 단순히 코드를 쓰는 도구라고만 볼 필요는 없다. 
이게 핵심입니다. 

그렇다면요?

이 도구가 마크다운 파일처럼 문서를 작성할 수 있고,
규칙과 구조를 바탕으로 내용을 정리해나갈 수 있다면,

복잡한 코딩 문제를 단계적으로 풀어가듯이,
업무용 문서 작성이나 기획 작업에도 적용해보자 이거죠

문제를 구조화해서 풀거나
특정 방법론을 기준으로 분석 틀을 세우는 작업 에서
로직이나 작업 흐름까지 함께 정리해주기 때문에
중간중간 필요한 데이터만 잘 꺼내 활용하자는 거죠

일반 LLM앱 GPT 제미나이 같은애들은 
수백장을 하는데 한계가 있으니까요

그렇다면 왜 제가 클로드 코드나 코덱스를 사용하게 
되었는지부터 이야기해보겠습니다.

우선 먼저 짚고 넘어가야 할 것은 컨텍스트 윈도우입니다.

우리가 사용하는 원천 기술인 LLM은 한 번에 처리하고
참조할 수 있는 문맥의 범위가 유한합니다.

그렇다면 여기서 이런 생각을 해볼 수 있습니다.

필요한 정보를 중간중간 구조화해서 저장해두고,
원하는 시점에 원하는 방식으로 다시 참조하게 만들 수 있다면,
훨씬 더 효율적으로 작업할 수 있지 않을까요?

파일을 읽고, 수정하고, 필요한 명령을 실행하면서 작업을 진행하는
하네스 엔지니어링 컨텍스트 엔지니어링과 같은 접근이죠

그렇다면 실제로는 어떤 식으로 사용해야 할까요?

우선 여기서 가장 잘 활용한 것은 OMC였습니다.

이건 클로드코드를 직접 써보지 않은 분들께는 다소 낯설 수 있으니

일단은 워크플로우와 기획 정리를 도와주는 도구 
이렇게 이해하셔도 무방합니다.

기본적인 아이디어는 단순합니다.

제 책의 기획 파일과 관련 자료를 넣고
전체 마케팅 분석과 자료 분석 구조를 만들어보자

이런생각이죠

우선 보안상 일부 자료는 가려두었다는 점, 양해 부탁드립니다.
저는 먼저 관련 자료를 충분히 정리해서 넣었습니다.

공학적으로 정리하면 충분한 컨텍스트를 제공한 것이죠

그리고 전문가이신 윤찬님의 조언에 따라,

필립 코틀러 기반의 분석법과 몇 가지 
추가 분석 프레임을 함께 적용하고 싶다 명령 내렸죠

그렇다면 여기서 문제는 뭘까요?

이러한 분석법을 제대로 수행하려면
그에 맞는 데이터가 충분히 있어야 합니다.

그렇다면 이 문제는 어떻게 해결해야 할까요?
