# [제대로 알려주는 프롬프트 테크닉(마지막)]

[제대로 알려주는 프롬프트 테크닉(마지막)]

챗GPT 제미나이 클로드 부터 젠스파크 마누스까지
수많은 AI도구들의 시대에 우리는 살아가고 있습니다.

초기에는 프롬프트엔지니어가 뜬다 이런말도 분명있었지만
그러한 시대를 지나

COT, Few-shot Learning, TOT, ReAct, Reflexion등등 의
기술이나오며 하나의 기술스텍으로자리잡았습니다.

사실 수많은 기법들이 쏟아져 나왔지만 이게 일반인에게 
친숙한 이야기는 아닙니다.

다만 각각의 방법론은 특정 상황에서 나름의 효과를 입증해왔고
이제 AI에이전트의 이야기로 넘어가기전 마지막으로 정리를하려
이번글을쓰게 되었습니다.

그럼 가봅시다 제대로 알려주는 프롬프트 이야기입니다

[참고 좀많이어려움]

요즘 안쓰는 프롬프트들이 있습니다.

정확히는 CHAT GPT를 에이전틱하게 사용하거나
여러 도구를 에이전트 구성하며 사용하며

당신은 10년차엔지니어 입니다 이런것들은 사용하지않는데요
오히려 단어의 중요성이 훨씬더 강조되고 있습니다.

그렇기에 맨처음 할이야기는 결국 단어의 중요성입니다

The Butterfly Effect of Altering Prompts: How Small Changes and Jailbreaks Affect Large Language Model Performance
라는 논문은 제가 프롬프트이야기할떄 자주이야기하는 논문으로

공백 온점만으로도 다른결과를 낼수있고 이는 언어의 해석이 달라지기때문이다. 이런 설명을 자주 드리는데요 

여기서 한단계 더나아가 프롬프트라는 문장은 결국 LLM에게 주어지는 인풋이고  이를 해석해 가져오므로 어떤 단어를 쓰느냐에 따라 결과물이

드라마틱하게 달라집니다.

오늘할 이야기의 결론을 먼저 정리하고 가죠

같은 의도를 담은 문장을 비교해봅시다.
"코드 작성해" 
"REST API 엔드포인트 구현해"

두개는 언뜯보면 비슷해 보이지만 문제는 AI가 해석할떄 전혀다른 말로 받아드려 집니다.

벡터 임베딩이라는 기술적 메커니즘으로 깊이있게 고차원 벡터를 설명해야하지만 그런걸 떠나서

결국 AI가 응답을 찾는다는것은 그안에서 필수적인 내용을 뒤진다는것이고
한번에 응답에 도달할수있는 연산력이 제한되 있으니

그연산력을 효율적으로 쓰고 구조화하여 던지자는 제언입니다.
단순하게 같은업무를 처리할지라도 힌트가 있는것과 없는것

그리고 그러한 차리르 언급해주느것 만으로도 오류율을 줄이고
정확한 응답을 얻을수있죠

쉽게말하면 비싸고 여러번 일하기 싫으니까 AI에게 일을 잘시키려면
후자가 압도적으로 유리하다 이렇게 이해하셔도됩니다.

이전에도 설명드린적 있지만

AI에 텍스트를 입력하면 가장먼저 하는게 토큰화에요

인간이 읽는 문장을 AI가 처리할 수 있는
최소 단위로 분해하는 과정으로

뭐 크게 3가지로 나누고 BPE나 형태소 분석같은 이론이있지만
그것보다 중요한건 

한국어가 비효율적이라는겁니다.

영어: "I love programming" → ["I", " love", " programming"] (3토큰)
한국어: "나는 프로그래밍을 좋아해" → ["나는", " 프로그래밍", "을", " 좋아", "해"] (5토큰)
