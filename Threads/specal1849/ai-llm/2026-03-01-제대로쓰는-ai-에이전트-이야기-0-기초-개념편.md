# [제대로쓰는 AI 에이전트 이야기 0 기초 개념편]

[제대로쓰는 AI 에이전트 이야기 0 기초 개념편]
AI에이전트 이야기를 다루기전 핵심적인 개념이있습니다.

작년부터 엔트로픽 구글 OPEN AI같은 대형 회사들은
여러 개념을 제시했습니다. 요즘 유행하는 개념들을 리스트업하면
1.Scaffolding
2.Agent Loop
3.Context Engineering
4.Harness
5.Agent Swarm

어렵죠? 
이런 개념들은 AI를 하시는 분들에게는 익숙한 용어여도
하나하나 잡고가는게 중요합니다.

GPT나 제미나이 클로드코드사용자가 
알면 좋은 교양이야기 시작합니다.

대체 뭘 알아야하는 것일까요?

개념에 대해 이야기하기전 사실 에이전트 국문 대리자라는 개념은
1980년대부터 나온개념이니 꽤나 근본적인 이야기지만

AI를거치고 LLM시대를 넘으며 꽤나 많은 변화가 있었습니다.
이번글에서는 AI에이전트에 트렌드에 대해 논해 봅시다.

일단 chat gpt나 제미나이에 같은 질문을 해도 앱에서는 
일반적으로 한번 프롬프트에 응답을 하는 단일 응답방식입니다.

그런데 젠스파크, Claude Code, CODEX는
여러가지 일을 한번에하며 파일을 읽고 코드를 고치고 테스트를 돌리고 
실패하면 다시 수정하는 반복적인 일을 합니다.

분명 같은 llm에서 출발한 기술이지만 
왜 행동이 다른걸까요?

우선 두개의 행동양식이 다른이유는 간단합니다. 

모델을 감싸는 구조가 다르기 때문입니다. 

요즘말하는 AI서비스 즉 LLM은
텍스트를 넣으면 텍스트가 나오는 함수에 불과하지만

이 함수를  소프트웨어 구조로 감싸면 에이전트가 됩니다.

이중 가장 핵심이되는 개념이 Scaffold 한국말로는 비계라고 불리는 개념이에요 

이게 대체 뭘까요?

caffolding은 LLM을 감싸는 소프트웨어 아키텍처 전체를  말합니다.
이는 건설 현장의 비계에서 따온 용어로

쉽게말하면 LLM한테
AI를 업그레이드 하기위해 해 주변에 구축한 코드와 도구
이렇게 이해하셔도 무방합니다.

글도못쓰고 새로운 정보도 못받아들이네 그럼 도구를 달아!!

여러기지 LLM단일로는 부족하지만 각종 툴 
메모리 시스템 프롬프트를 줘서 강화하는 거에요

어느정도로 차이가 나냐면 

GPT-4기준 초기 
RAG 기반 Scaffold를 쓰면 2.7
CodeR이라는 Scaffold를 쓰면 28.3%

사람으로 치자면 여러 업무메뉴얼 도구를 세팅해주는 단계로 보셔도 무방합니다. 물론여기에 MCP나 SKLLIS같은 전문적 도구를 다뤄야 하지만

24년도 엔트로픽에서 내논 Building Effective Agents라는 아티클에 따라
LLM을 잘쓰기위한 뼈대를 만들었다 보셔도 무방합니다.
