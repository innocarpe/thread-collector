# 실무에서 사용하는 클로드코드 사용법

실무에서 사용하는 클로드코드 사용법

저는 요즘 모든실무를 클로드코드+코워크로 처리하고있습니다.

편한것도 사실이지만 
제 철학은 제가 100시간미만으로 사용한툴을
강의 하는것은 사기다 이렇게 생각해서 이기에

모든 업무를 죄다 클로드코드베이스로 처리해보고있습니다.

아래는 제가 실제로 출제데이터수집과 문제출제 과정을 
처리한 결과물을 보시고

어떤식으로 했는지 이야기해보죠

우선 제가 클로드코드에 자료를 넣기전 하는 루틴입니다.

1.문제 정의
상황 AI레벨테스트를 만들 필요성이대두
-목적 이프로젝트로 얻고싶은 결과물이뭐냐
-방법론 데이터가 필요하고 자료조사를 어떻게할것이냐

이 세가지를 먼저 정의합니다 

에이전트 설계나 세부 파이프라인은 나중문제에요 
세가지가 정의되지 않는다면 

아무리 좋은 파이프라인 스킬할아버지가 와도 
쓸만한 결과물을 내놓지 못합니다.

저는 레벨별 테스트를 빠른버전 중간버전 심화버전으로
만들자 기획했고

기조 데이터가 필요하다 생각했습니다.
역시 외부에서는 어떻게 하는지 조사할 필요성이있겠죠?

클로드코드 베이스로 깍아둔 딥리서치를 사용할 차례입니다.

단순하게 데이터조사를 해와가 아니라 어떤식으로 조사하고
리스크테이킹을 어떤식으로 할껀지 

세세한 지정은 기본적으로 설계되어야합니다.

어떤한 암묵지가 있을수있고 어떤문제가 있을수있는지등의 고민이 들어가야지요

기조조사이후 어떤식으로 데이터를 구성할껀지도
사실상 필수요소죠

자그럼이제 기초적인 데이터조사와 난이도 조사가끝낫다고 가정해봅시다.

휴먼인더 루프라고 하죠?

여기서 인간의 관찰을 한번넣습니다 어떤식으로 처리했는지
정리는 잘되었는지 할루시는 없었는지 체킹하는 과정이 필요합니다

이후에 이제 문제를 생성할차례입니다.

제가 원하는건 단순하게 레벨별로 체킹하는게아니라
실제 상황화 결합해 제대로 체킹이 가능한 문제를 출제하는거였죠

세세한 지표와 명령이 들어갓다 이해하셔도 문제없습니다.
