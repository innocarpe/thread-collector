# 바이브코딩 하수부터 쌉고수까지

바이브코딩 하수부터 쌉고수까지.
어떻게 하는지 딱 한 줄 정리 들어감.

하수

1. 프론트 개발 전 PRD 문서 작성
2. 위 문서 대로 개발 지시 후, 디자인 디테일 작업
3. 백엔드 개발 전 DB 스키마 및 API 기능 문서 작성
4. 위 문서를 바탕으로 서버 환경에 맞춰 작업 요청.
5. 디버깅하고 수정 진행.

고수

1. 위 순서대로 작업하는 MCP 제작
2. 이미지 리소스 생성 API 별도 제작 후 MCP에 연결
3. "어쩌구 저쩌구 웹사이트 만들어" 프롬프트 입력
4. 디버깅 및 디테일 작업 요청
5. 딱 1시간 작업 후 서비스 런칭.

참고로 이틀 전에 지인 만나서 보여드리고 충격 먹으심.
