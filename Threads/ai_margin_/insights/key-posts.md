## 2. 자동화 철학과 도구관

### 자동화의 올바른 순서

**원문**

자동화 먼저 하려는 사람은 100% 실패한다

내가 증명함

블로그 자동화? 3주 코딩했는데
정작 블로그를 수동으로 안써봤다?

그냥 직원관리 안하고 직원 시키는 셈

님들이 회사 대표인데 직원한테 관심이 없음
뭘 하든 상관안한다는거야. 
그럼 제대로 하겠니?

근데 잘 시키려면? 수동으로 해보고
그다음에 자동화다 잊지마 진짜야

수동 10번 → 패턴 → 자동화.
이 순서 틀리면 망함.

→ 자동화 과정 프로필에 정리 중.

**의미**

실행 일변도처럼 보이는 계정이 사실은 공정 이해를 꽤 중요하게 본다는 점을 보여준다. "무조건 자동화"가 아니라 "이해한 뒤 자동화"가 그의 일관된 프레임이다.

### 툴보다 상상력

**원문**

자동화 도구
make vs n8n vs 직접 코딩

비교해봄

make: 월 $9~, 드래그앤드롭, 초보 편함
n8n: 무료 셀프호스팅, 자유도 높음
직접 코딩: $0, 제한 없음, 내 맘대로

나는 전부 써봤는데
결론은 직접 코딩임

왜냐면
make/n8n은 "제공하는 것"만 됨
코딩은 "상상하는 것"이 다 됨

근데 코딩 모르잖아?
그래서 AI한테 시키는 거임

make 구독료로
클로드코드 쓰면
make보다 더 많은 게 됨

**의미**

이 글은 그의 도구 선택 기준을 잘 보여준다. 노코드 툴도 쓰지만, 궁극적으로는 AI를 매개로 직접 코딩 쪽 자유도를 선호한다.

### 반자동화의 실제 구조

**원문**

네이버 블로그 반자동화

## 3. SNS 성장과 데이터 운영

### 스레드가 안 터지는 이유

**원문**

스레드에서 안 터지는 글의 공통점.

1,600개 글 데이터 분석해서 뽑아냄:

1. 첫 줄 30자 넘김 → 스크롤 당함
2. 숫자 없음 → 구체성 0
3. "~해야 합니다" 체 → 교과서 느낌
4. 도구명 없이 "AI 시대" → 추상적
5. 질문하고 자기가 답함 → 참여 0

이 5개만 안 해도
바닥에서 벗어남.

→ 프로필에 더 정리해둠.

3번 진짜 많이 하는 실수. "~임/~됨" 체로 바꾸는 것만으로 달라짐.

**의미**

문체, 숫자, 첫 줄 구조까지 데이터로 설명하는 글이다. 그의 SNS론이 감이 아니라 실험과 누적 관찰에 기반한다는 점이 잘 보인다.

### 초반 성장 공식

**원문**

스레드 팔로워 0 → 1,000 가는 법.
내가 직접 한 방법.
진짜 다 알려줬다.

1. 매일 7개 올리기 (양으로 승부)
2. 같은 니치 계정 댓글 30개/일
3. 인증형 글 주 2회 (숫자+감정)
4. 절차형 글 주 2회 (저장 유도)
5. 도발형 글 주 1회 (팔로우 전환)
6. 팔로우하면 매일 팁 공유함

1,000까지 2개월 걸렸음.
근데 이 루프 돌리면 지금은 더 빠를 듯.

안 할 사람은 안 함.
근데 할 사람한테는 이거면 충분함.

2번이 귀찮은데 이게 초반에 제일 효과 좋음

**의미**

이 계정이 왜 "실행 루프"를 강조하는지 잘 보여준다. 콘텐츠 유형, 댓글 활동, 팔로우 전환까지 전형적인 퍼널 사고가 들어 있다.

### 시스템이 감을 이긴다

**원문**

스레드 자동화 시스템 만든 지 5개월째.

하루 7개 포스팅.
한 달이면 210개.
5개월이면 1,050개.

이 1,050개 중에
1만 넘긴 글이 40개 이상임.

비결?
글 하나하나 잘 쓰는 게 아니라
시스템으로 물량 돌리고
데이터 보고 패턴 뽑는 거임.

"이 글감은 터지고 이 글감은 죽는다"
이게 1,050개 데이터에서 나옴.

감으로 쓰는 시대 끝났음.

시스템 돌리고.
데이터 보고.
패턴 따라가면.

팔로워는 따라옴.

1,050개 올리면서 가장 크게 배운 거: "좋은 글"이랑 "터지는 글"은 다르다는 것.

**의미**

@ai_margin_의 콘텐츠 전략을 가장 대표하는 글이다. 좋은 글쓰기보다 실험 설계와 패턴 추출을 더 중시한다는 점이 명확하다.

### 양이 퀄을 만든다는 주장

**원문**

"자동화하면 퀄 떨어지잖아"

이 말 하는 사람한테
하나만 물어볼게

수동으로 하루에 3개 올리는 거랑
자동으로 하루에 30개 올리는 거

어디서 데이터가 더 나옴?

퀄은 양에서 나옴
양이 없으면 퀄을 판단할 기준도 없음

수동 3개 중에 1개 터지면 33%
자동 30개 중에 3개 터지면 10%

근데 터진 글 수는 3개 vs 1개

어떤 게 나음?

**의미**

물량, 테스트, 반응 데이터 중심의 사고가 가장 날것으로 드러난다. 창작을 장인 정신보다 실험 규모의 문제로 보는 시각이 분명하다.

### 외주와 자동화의 가격 차이

**원문**

스레드 글 외주 맡기면
1개에 3~5만원.

하루 7개면 월 630만원.

나는 이걸 클로드 코드
월 2.9만원으로 하고 있음.

630만원 vs 2.9만원.
이 차이 보고도 안 바꿀 거야?

자동화의 핵심은 비용 절감보다 물량 확보임. 물량이 알고리즘을 움직임.

**의미**

수익화와 콘텐츠 운영이 이 계정 안에서 어떻게 연결되는지 보여주는 글이다. 비용 절감보다 생산량 확보를 더 중요한 가치로 둔다는 점도 선명하다.

## 4. 자기서사와 1인 사업 철학

### 배경보다 반복

**원문**

조선소에서 용접하던 놈이
지금 AI로 사업하고 있음.

대학 안 갔음.

그냥 클로드한테 시켰음.

인생 뭐 어떻게 될지 모름 ㄹㅇ

1년 전의 나한테 "너 AI 사업가 된다" 하면 미친놈 소리 들었을 듯

**의미**

팔로워를 움직이게 만드는 가장 강한 서사 장치다. 전문성보다 반복 실행이 중요하다는 메시지가 자신의 이력과 결합해 설득력을 얻는다.

### 변화의 한 문장

**원문**

쓰친들아 솔직하게 얘기할게.

예전엔 최저시급 받고 일했음.
조선소. 매일 새벽 5시 출근.
"이게 맞나" 매일 생각했음.

스레드 1.7만 팔로워.

대기업에서 광고 문의 옴.

달라진 건 딱 하나.
"AI한테 시키면 되는 거 아니야?"

자기계발형 메시지와 실적 인증이 결합된 대표 글이다. 독자에게 "배경이 아니라 태도"라는 허가를 주는 방식이 잘 드러난다.

**원문**

어떻게 여기까지 왔는지 프로필에 정리해둠
