# 개인 앱 개발 프로젝트 런칭 원칙

개인 앱 개발 프로젝트 런칭 원칙

저는 개인 앱 개발자로서 다음과 같은 원칙을 세워 프로젝트를 진행하고 있습니다.

개발 단계
- 개발 기간이 2주를 초과할 것 같은 프로젝트는 시작하지 않습니다
- 복잡한 기능보다는 핵심 기능만 담은 MVP(최소기능제품)를 최우선으로 빠르게 완성합니다
- 완벽함보다는 출시 속도에 집중합니다

검증 단계
- 완성된 MVP를 즉시 스토어에 출시하여 실제 사용자 반응을 확인합니다(+광고세팅)
- 시장 검증에는 시간이 필요하므로, 검증 결과를 기다리는 동안 다음 프로젝트 개발을 병행합니다

의사결정 단계
- 시장에서 반응이 좋지 않은 앱은 과감히 포기하거나 방치합니다.(시간과 리소스 최소화)
- 검증에 성공한 앱은 추가 기능 개발과 적극적인 마케팅에 집중합니다

반복
- 위 과정을 지속적으로 반복하여 성공 확률과 개수를 높입니다

이 방식을 통해 한정된 개인 리소스로도 여러 프로젝트를 동시에 진행하며, 성공 가능성이 높은 앱에 집중할 수 있습니다.
