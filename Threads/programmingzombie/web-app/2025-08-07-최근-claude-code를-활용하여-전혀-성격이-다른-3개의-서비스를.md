# 최근 Claude Code를 활용하여 전혀 성격이 다른 3개의 서비스를 동시에 개발하면서, 예상치 못한 생산성 향상을 경험했습니다

최근 Claude Code를 활용하여 전혀 성격이 다른 3개의 서비스를 동시에 개발하면서, 예상치 못한 생산성 향상을 경험했습니다.

기존에는 하나의 작업을 요청하고 AI의 응답을 기다리는 시간이 불가피했습니다. 하지만 3개의 프로젝트를 동시에 진행하니 완전히 다른 차원의 워크플로우가 만들어졌습니다.

작업 사이클:
1. 3개 프로젝트에 동시에 작업 요청
2. 첫 번째 작업 완료 → 검증 시작
3. 검증하는 동안 두 번째 작업 완료
4. 첫 번째 검증 완료와 동시에 새로운 작업 요청
5. 두 번째 검증 중 세 번째 작업 완료

이미지는 예시 입니다. (댓글에 이어서)

이런 식으로 사이클이 돌아가니, AI 응답 대기 시간이 완전히 사라졌습니다. 항상 검증하거나 새로운 작업을 계획할 일이 있어서 유휴시간이 전혀 발생하지 않더군요.

당연히 토큰 소비량은 2~3배로 늘어났습니다. Max $200 플랜을 사용하는 입장에서는 구독 비용이 부담스러울 수도 있겠지만, 오히려 플랜의 한계치를 최대한 활용하는 셈이 되었습니다.

결과적으로 같은 비용으로 훨씬 더 많은 작업을 완료할 수 있게 되었고, 시간당 생산성은 기존 대비 압도적으로 향상되었습니다.

그렇게 시작한지 채 일주일이 되지 않아 전혀 다른 서비스 3개를 완성하였습니다. 내일은 해당 서비스의 스크린샷을 제작하고 앱 출시를 하는날이 될 것 같습니다. Flutter를 사용해서 만들었으니 iOS, Android 모두 출시한다면 6개의 앱이 출시되는 효과입니다.
