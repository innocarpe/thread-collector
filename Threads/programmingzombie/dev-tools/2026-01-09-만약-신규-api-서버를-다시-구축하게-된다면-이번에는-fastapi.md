# 만약 신규 API 서버를 다시 구축하게 된다면, 이번에는 FastAPI 또는 Go Fiber를 도입해 볼 생각입니다

만약 신규 API 서버를 다시 구축하게 된다면, 이번에는 FastAPI 또는 Go Fiber를 도입해 볼 생각입니다.

그동안은 실무에서 손발을 맞춰온 Spring Boot가 가장 편하고 확실한 도구였기에 관성적으로 선택해 왔습니다. 하지만 한정된 리소스로 운영되는 개인 서버 환경에서, JVM 기반의 무거운 Spring Boot는 다소 '오버스펙'이라는 느낌을 지울 수 없었습니다.

무엇보다 이제는 새로운 기술을 학습하고 적용하는 러닝커브가 거의 '제로'에 수렴하는 시대가 되었습니다. 익숙함에 머물기보다, 내 프로젝트에 딱 맞는 최적의 '핏(Fit)'을 빠르고 자유롭게 찾아 적용해 볼 생각입니다
