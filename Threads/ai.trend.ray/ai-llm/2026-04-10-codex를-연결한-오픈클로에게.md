# Codex를 연결한 오픈클로에게

Codex를 연결한 오픈클로에게
1개의 TASK를 쥐어주고
1개의 결과물을 받는 것은 아주 쉽다.

하지만 ,
TASK를 6개로 나누고
1차 TASK -> 중간보고 
-> 2차 TASK 진행 -> 중간보고
처럼 task 사이사이 보고를 넣으면
보고하는 순간 다음 task로 연결이 안되고 있다.

오픈클로 스스로 문제 진단한 결과
'보고를 하나의 끝점'으로 인식하기 때문에
이 부분을 강하게 다음 task를 밀수 있도록
수정하겠다고 했다.
설정 문서 구조도 스스로 박았으나,
여전히 다음 task를 못 밀고 있다.

크론으로 박아두면, heartbeat처럼
설정한 시간마다 task 상태를 살펴볼 수 있게
만들 수 있는건 알 수 있지만.
시간을 기반으로 진행하는게 아니라
task 처리 기반으로 진행해야
결과물을 빨리 받을 수 있다는 생각에
어떻게 하면 이 문제를 풀 수 있을까 고민 중.

일단,

1. 
일단 문제는 오픈클로가 혼자 일하면
1 task = 1 result 가 어쩔 수 없는 부분인 것 같고.
오픈클로에 코덱스 Cli, gemini cli, goose를 붙여줬으니
오케스트레이션으로 하면
검수 루프까지 돌린 후 결과를 가져다 주는지 테스트.

2. 
실제 루프를 돌리고
결과물까지 가져다주는 
시간은 대략 15~20분 정도 
소요 예상이라고 했으니
