## 핵심 주장 (≤5)
1. 안드로이드 앱은 기기 내부 기능만으로 한계가 있어 서버와 정보를 주고받는 구조가 필요하다.
2. 안드로이드 소켓 통신의 서버 쪽 구현은 기존 자바 소스로 처리할 수 있다.
3. 클라이언트가 서버 응답을 기다리며 블록킹되지 않도록 수신 처리는 별도 쓰레드로 백그라운드에서 돌리는 방식이 유효하다.
4. 안드로이드 클라이언트는 서버 소켓 연결, 송수신 스트림 설정, UI 반영을 분리해서 구현해야 한다.
5. 안드로이드에서 소켓 통신을 사용하려면 `AndroidManifest.xml`에 인터넷 권한을 추가해야 한다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- Java: 서버 소켓과 안드로이드 클라이언트 예제 코드를 구현하는 언어로 사용됨
- Android: 클라이언트 앱을 구현하고 소켓 통신 설정을 설명하는 플랫폼으로 사용됨
- Socket: 서버와 클라이언트가 데이터를 주고받는 통신 방식으로 사용됨
- Handler: 서버에서 받은 데이터를 UI 영역에 반영할 때 사용하는 안드로이드 구성요소로 언급됨
- Toast: 서버에서 받은 문자열을 화면에 출력하는 예제 수단으로 사용됨
- AndroidManifest.xml: 인터넷 권한을 선언하는 설정 파일로 사용됨
- XML: 입력창과 버튼이 있는 간단한 클라이언트 UI를 정의하는 데 사용됨

## 언급된 다른 creator·앱
- 트위터 서비스: 서버를 두고 정보를 주고받는 서비스의 예시로 언급됨
- Google Developer: 참고자료로 언급됨

## 복제 가능한 전술 (≤3)
1. 자바 서버와 안드로이드 클라이언트를 분리해 최소 기능의 TCP 소켓 예제를 먼저 만든다.
   - 구체적 스텝: 자바에서 `ServerSocket`으로 포트를 열고 `accept()`로 대기한다; 클라이언트 연결 후 입력 스트림으로 한 줄을 읽는다; 받은 문자열을 출력 스트림으로 다시 반환한다; 안드로이드에서는 서버 IP와 포트를 설정해 `Socket`으로 연결하고 송수신 스트림을 초기화한다.
   - 예상 리소스: 자바 개발 환경, 안드로이드 개발 환경, 서버 IP 및 포트 정보
   - 예상 효과: 서버와 클라이언트가 문자열을 주고받는 기본 소켓 통신 흐름을 확인할 수 있다.
2. 서버 수신 대기 때문에 클라이언트 UI가 멈추지 않도록 수신 루프를 백그라운드 쓰레드로 분리한다.
   - 구체적 스텝: 안드로이드 `onCreate()`에서 소켓 연결 뒤 별도 쓰레드를 시작한다; 쓰레드 내부에서 `while` 루프로 `readLine()`을 반복 호출한다; 읽은 값을 `Handler`로 메인 스레드에 전달해 `Toast` 등 UI로 출력한다.
   - 예상 리소스: 안드로이드 `Thread`, `Handler`
   - 예상 효과: 기존 클라이언트 동작은 유지하면서 서버에서 오는 정보를 지속적으로 받을 수 있다.
3. 안드로이드 쪽 네트워크 사용 조건을 사전에 설정한다.
   - 구체적 스텝: 레이아웃 XML에 텍스트 입력창과 전송 버튼을 추가한다; 버튼 클릭 시 입력 문자열을 서버로 전송하도록 연결한다; `AndroidManifest.xml`에 `android.permission.INTERNET` 권한을 선언한다.
   - 예상 리소스: 레이아웃 XML, `AndroidManifest.xml`
   - 예상 효과: 입력한 문자열을 서버로 보내고 응답을 수신하는 테스트 가능한 클라이언트 화면을 구성할 수 있다.

## 원문 요약 (≤5문장)
이 글은 안드로이드 앱에서 서버와 데이터를 주고받기 위한 서버/클라이언트 소켓 통신의 기초 구현을 설명한다. 서버는 자바 `ServerSocket`으로 클라이언트 연결을 받고, 문자열을 수신한 뒤 다시 반환하는 예제로 구성된다. 안드로이드 클라이언트는 소켓 연결 후 입력값을 서버로 보내고, 서버 응답은 별도 쓰레드에서 지속적으로 읽어 `Handler`를 통해 UI에 반영한다. UI는 간단한 `EditText`와 버튼으로 구성하며, 매니페스트에는 인터넷 권한을 추가해야 한다. 글의 핵심은 네트워크 대기 작업을 백그라운드로 분리해 앱의 기존 동작이 막히지 않게 만드는 것이다.

## 본문 포인트별 발췌
> 오늘은 서버, 클라이언트의 소켓(Soket) 통신에 대해서 알아보겠습니다.
> 그렇기 때문에 클라이언트가 계속 데이터를 기다리면서 블록킹 되어 있는 상태에서만 프로그래밍이 돌아갔습니다.
> 이러한 부분을 수정하여 쓰레드를 이용하여 백그라운드에서 돌아가게 하여, 기존의 클라이언트에서는 원래의 프로그램이 수행되고, 서버에서 오는 정보를 받는 부분은 쓰레드를 통해 해결했습니다.
> 메니페스트에는 소켓을 이용하기위해 인터넷을 사용해야 하므로, 인터넷을 사용하겠다는 퍼미션만 추가해주시면 됩니다.
> 안드로이드 클라이언트 부분에서도 현재 수행되고 있는 동작에 영향을 받지 않으면서 백그라운드에서 쓰레드가 돌면서 서버에서 오는 정보를 수시로 받을 수 있도록 해야합니다.
