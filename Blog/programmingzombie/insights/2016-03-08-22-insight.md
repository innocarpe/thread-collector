## 핵심 주장 (≤5)
1. ObjectMapper는 JSON 값을 다룰 때 직접 파싱하거나 GSON을 쓰는 방식보다 더 깔끔하고 간단하게 데이터를 가공할 수 있다.
2. 작성자는 앞으로 JSON 관련 파싱을 ObjectMapper를 바탕으로 진행할 생각이라고 밝힌다.
3. ObjectMapper의 핵심 사용법은 `writeValue(...)`, `writeValueAsString()`, `readValue(...)`로 Java 객체와 JSON 사이를 변환하는 것이다.
4. `Map`과 `List<Map>`도 `writeValueAsString`, `readValue`, `TypeReference`를 이용해 JSON과 상호 변환할 수 있다.
5. Jackson으로 매핑할 Java 클래스는 기본 생성자를 유지해야 한다는 점이 중요하다고 소개한다.

## 공개된 숫자·지표
- Jackson 1.x 상태: `maintenance project` (기간: 글에서 인용한 튜토리얼 기준, 출처: `Jackson 1.x is a maintenance project, please use Jackson 2.x instead.`)
- Jackson 의존성 버전: `1.9.13` (기간: 예제 기준, 출처: `compile 'org.codehaus.jackson:jackson-mapper-asl:1.9.13'`)
- `@JsonView` 지원 시작 버전: `1.4` (기간: 기능 소개 기준, 출처: ``@JsonView` has been supported in Jackson since version 1.4, it lets you control what fields to display.`)
- 한글 자료에서 제공한 JAR 버전: `2.2.3` (기간: 관련 문서 소개 기준, 출처: `jackson-annotations-2.2.3.jar`, `jackson-core-2.2.3.jar`, `jackson-databind-2.2.3.jar`)

## 언급된 도구·서비스
- ObjectMapper: Java 객체, `Map`, `List<Map>`를 JSON 문자열이나 파일로 변환하거나 다시 읽어오는 핵심 도구로 소개됨
- Jackson: ObjectMapper, `JsonParser`, `JsonGenerator`, `@JsonView` 등을 제공하는 JSON 처리 라이브러리로 언급됨
- GSON: ObjectMapper와 비교 대상으로 언급되며, 작성자는 ObjectMapper가 더 깔끔하고 간단했다고 평가함
- `writeValue(...)`: Java 객체를 JSON 파일로 저장하는 메서드로 소개됨
- `writeValueAsString()`: 객체나 맵을 JSON 문자열로 바꾸는 메서드로 소개됨
- `readValue(...)`: JSON 문자열이나 파일을 Java 객체, `Map`, `List<Map>`로 되돌리는 메서드로 소개됨
- `TypeReference`: `ArrayList<HashMap<String, String>>` 같은 제네릭 타입을 역직렬화할 때 타입 명시에 사용하는 도구로 소개됨
- `@JsonView`: 노출할 필드를 제어하는 Jackson 기능으로 소개됨
- Gradle: Jackson 의존성을 추가하는 예시 도구로 소개됨
- Color Scripter: 코드 예제 하단에 색상 표시 도구로 표기됨

## 복제 가능한 전술 (≤3)
1. 수동 JSON 파싱 코드를 ObjectMapper 기반 직렬화·역직렬화로 교체한다.
   - 구체적 스텝: JSON을 직접 파싱하거나 인터페이스를 수작업으로 맞추는 지점을 찾고, Java 객체에는 `writeValueAsString()` 또는 `writeValue(...)`, JSON 입력에는 `readValue(...)`를 적용해 변환 흐름을 단순화한다.
   - 예상 리소스: Java 프로젝트, Jackson 의존성, 기존 JSON 처리 코드 검토 시간
   - 예상 효과: 원문처럼 직접 파싱이나 GSON 대비 더 깔끔하고 간단하게 데이터를 가공할 수 있다.
2. `Map`과 `List<Map>`를 JSON API 응답용 모델로 바로 다룬다.
   - 구체적 스텝: 자바스크립트에 넘길 데이터를 `HashMap<String, String>` 또는 `ArrayList<HashMap<String, String>>`로 구성하고 `writeValueAsString()`으로 JSON 문자열로 변환한 뒤, 다시 읽을 때는 `readValue(...)`와 `TypeReference`로 타입을 명시한다.
   - 예상 리소스: `ObjectMapper`, `TypeReference`, Java 컬렉션 타입
   - 예상 효과: 원문처럼 `Map - JSON`, `List<Map>` 변환을 같은 메서드 체계로 처리할 수 있고, 자바스크립트에 JSON을 넘겨줄 때 유용하다.
3. Jackson 매핑용 클래스에 기본 생성자를 유지하고 뷰별 출력 필드를 분리한다.
   - 구체적 스텝: JSON 매핑 대상 클래스에 기본 생성자를 추가하고, 필드 공개 범위를 나눠야 하면 `@JsonView`와 `writerWithView()`를 적용해 필요한 필드만 출력되도록 구성한다.
   - 예상 리소스: Jackson 어노테이션, 매핑 대상 POJO 수정 시간
   - 예상 효과: 원문이 소개한 방식대로 Jackson이 객체 인스턴스를 생성할 수 있고, 출력 필드 제어도 함께 구현할 수 있다.

## 원문 요약 (≤5문장)
글은 작성자가 ObjectMapper를 처음 접한 뒤, JSON 처리에서 직접 파싱이나 GSON보다 더 간단하고 깔끔하다고 느낀 점을 기록한 자료 조사 메모다. 본문은 세 개의 참고 링크를 중심으로 ObjectMapper의 핵심 메서드와 `Map`, `List<Map>` 변환 예제를 정리한다. 특히 `writeValueAsString()`, `readValue(...)`, `TypeReference`를 통해 일반 객체와 제네릭 컬렉션을 JSON으로 주고받는 방식을 강조한다. 추가로 `@JsonView`, Jackson 버전, 기본 생성자 필요성 같은 실무 주의점도 함께 소개한다.

## 본문 포인트별 발췌
> 며칠전 아는형님의 소개로 Object Mapper에 대해 알게 되었다.
> 그동안 JSON값 활용을 위해 직접 파싱을 하고 인터페이스를 만드는등 번거로운 노가다(?) 작업이 필요했는데,사용해보니 직접 파싱 or GSON 보다 훨씬 ObjectMapper이 깔끔하고 간단하게 데이터를 가공할수 있었다.
> 앞으로 JSON관련 파싱은 이를 바탕으로 할 것같다.
> 맵 타입이 JSON 형식의 String 타입으로 변환된다. 자바스크립트에 JSON을 넘겨줄 때 유용하다.
> 다음은 view에 전달할 model이 List<map<?, ?>> 타입일 때 이를 JSON으로 변환하는 방법이다. 사용방법은 크게 다르지 않고 writeValueAsString, readValue 메서드를 사용하되 타입 명시만 달리한다.
