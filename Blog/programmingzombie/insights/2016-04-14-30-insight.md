## 핵심 주장 (≤5)
1. 안드로이드 프로젝트에서 SQLite 테이블 내용을 눈으로 확인하려면 에뮬레이터와 명령프롬프트를 통해 DB 파일을 직접 열어볼 수 있다.
2. SQLite는 다른 데이터베이스보다 자료형이 제한적이며, 원문은 지원 자료형을 5가지로 정리한다.
3. `execSQL(String sql);` 메서드로 `SELECT`를 제외한 대부분의 SQL 명령을 실행할 수 있다.
4. `getReadableDatabase();`와 `getWritableDatabase();` 호출 시 DB가 없으면 `onCreate();`, 버전이 바뀌면 `onUpgrade();`가 호출된다.
5. 저장된 DB는 `data/data/패키지명/databases` 경로에 있고, `sqlite3`, `.tables`, `select * from 테이블명;`로 구조와 데이터를 확인할 수 있다.

- 에뮬레이터: 루팅 없이 DB 파일 생성 여부를 확인하는 실행 환경으로 사용됨

## 복제 가능한 전술 (≤3)
1. SQLiteOpenHelper의 `onCreate()`에서 테이블 생성과 샘플 데이터 삽입을 함께 구성해 초기 상태를 바로 검증한다.
   - 구체적 스텝: `DBManager`에서 DB 이름과 버전을 상수로 정의한다 -> `onCreate(SQLiteDatabase db)`에서 `CREATE TABLE`을 실행한다 -> 이어서 `INSERT INTO` 구문으로 확인용 샘플 데이터를 넣는다
   - 예상 리소스: 안드로이드 프로젝트, `SQLiteOpenHelper`, SQL 작성 시간
   - 예상 효과: DB 생성 직후 조회 가능한 데이터가 들어 있어 테이블 구조와 저장 결과를 즉시 눈으로 확인할 수 있다
2. 액티비티 시작 시 `getReadableDatabase();` 또는 `getWritableDatabase();`를 호출해 DB 생성과 업그레이드 훅을 강제로 실행한다.
   - 구체적 스텝: 액티비티의 `onCreate()`에서 `DBManager` 인스턴스를 만든다 -> `getReadableDatabase();` 또는 `getWritableDatabase();`를 호출한다 -> 작업 후 `close();`로 DB를 닫는다
   - 예상 리소스: 테스트용 액티비티 1개, `DBManager` 클래스
   - 예상 효과: DB가 없을 때는 `onCreate();`, 버전 변경 시에는 `onUpgrade();`가 호출되어 DB 파일 생성과 스키마 반영을 확인할 수 있다
3. 에뮬레이터에서 생성된 DB 파일을 `sqlite3`로 열어 테이블 목록과 레코드를 직접 검증한다.
   - 구체적 스텝: 에뮬레이터로 앱을 실행한다 -> DDMS 등으로 DB 파일 생성 여부를 확인한다 -> `data/data/패키지명/databases` 경로로 이동한다 -> `sqlite3 DB명`을 실행한다 -> `.tables`로 테이블을 보고 `select * from 테이블명;`으로 데이터를 조회한다
   - 예상 리소스: 에뮬레이터, DDMS, 명령프롬프트, `sqlite3`
   - 예상 효과: 실제 저장된 테이블과 1번부터 6번까지의 입력 데이터가 출력되어 CRUD 결과를 눈으로 검증할 수 있다

## 원문 요약 (≤5문장)
글은 안드로이드에서 SQLite 테이블 내용을 직접 확인하는 방법을 정리한 튜토리얼이다.
SQLite의 기본 자료형과 `execSQL()`로 실행할 수 있는 주요 SQL 명령을 먼저 설명한다.
이후 `DB_Test.java`와 `DBManager.java` 예제로 DB 생성, 테이블 생성, 샘플 데이터 삽입 흐름을 보여준다.
DB는 `getReadableDatabase();` 또는 `getWritableDatabase();` 호출 시 생성되며, 실제 파일은 `data/data/패키지명/databases` 경로에 저장된다고 설명한다.
마지막으로 에뮬레이터와 명령프롬프트에서 `sqlite3`, `.tables`, `select * from 테이블명;` 명령으로 데이터를 확인하는 절차를 안내한다.

## 본문 포인트별 발췌
> SQLite는 다른 데이터베이스에 비해 사용 가능한 자료형이 많지않고 5가지를 지원한다.NULLINTEGER : 1,2,3,4,6,8bytes의 정수값REAL : 8bytes의 부동소수점값TEXT : UTF-8, UTF-16BE, UTE-16LE 인코딩의 문자열BLOB : 입력된 그대로 저장
> SELECT 명령을 제외한 대부분의 명령을 execSQL(String sql); 메서드로 실행 할 수 있다.
> 읽기전용으로 DB를 불러온다. 이 때 생성된 DB가 없으면 onCreate(); DB가 있지만 버전이 바뀌었다면 onUpgrade();를 호출한다.
> 실제 단말기에서 DB를 열어보려면 루팅을 해야 하고, 에뮬레이터로 실행시키면 DDMS를 통해 DB파일이 생성되었는지 확인이 가능하다.
> 데이터베이스는 data/data/패키지명/databases 경로에 저장되어있다.
