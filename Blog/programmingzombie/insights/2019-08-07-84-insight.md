## 핵심 주장 (≤5)
1. 혼자서 서비스를 전적으로 맡아 개발할 때는 기술 선택의 자유는 있었지만 코드 검증과 성장 체감이 어려웠다.
2. 서버팀 합류 이후 스크럼, 회고, 페어 프로그래밍, 오프라인 리뷰 같은 협업이 문제 해결 방식의 폭을 넓혀줬다.
3. 스터디에서 배우기만 하려는 태도보다 직접 정리하고 가르치려는 태도가 더 큰 학습 효과를 만든다.
4. 서버 개발로 커리어를 전환하면서 기존 경험이 초기화된 듯한 감각을 느꼈고, Spring Boot부터 Kubernetes까지 다시 학습해야 한다고 판단했다.
5. 운동과 수면 개선은 체력, 집중도, 학습 능력 회복에 직접적인 영향을 줬고, 책 읽기는 단기간에 빠르게 성장하는 수단으로 인식됐다.

## 공개된 숫자·지표
- 단독 서비스 담당 기간: 1년 6개월 가량 (기간: 입사 후 초기, 출처: "1년 6개월 가량은 거의 혼자서 하나의 서비스를 전적으로 맡아서 만들었는데, 안드로이드 앱 개발 + API 개발로 거의 대부분의 시간을 보냈다.")
- 서버팀 합류 후 기간: 약 7개월 (기간: 이후 시점부터 현재까지, 출처: "그리고 이후 시점 ~ 현재까지(약 7개월)는 서버팀에 합류하여 백앤드 관련일을 맡게 되었다.")
- 체중 감량: 7kg (기간: 2019년 상반기, 출처: "인생 최대 몸무게를 찍고 7kg를 감량하였다.")
- 운동 빈도: 매주 3~5회 (기간: 감량 진행 시기, 출처: "더 늦기전에 헬스와 PT를 같이 등록하고 매주 3~5회 운동을 나갔다.")
- 식단 빈도: 하루 1~2끼 (기간: 감량 진행 시기, 출처: "그리고 하루 1~2끼는 닭가슴살과 고구마 그리고 셀러드를 먹은것 같다.")
- 평균 수면시간: 4.5시간 (기간: 작년, 출처: "작년만해도 거의 평균 수면시간이 4.5시간쯤 이었던것 같다.")
- 커피 섭취량: 하루 3~4잔 (기간: 작년, 출처: "그래서 하루 3~4잔의 커피로 버텼으며")
- 평균 수면시간: 5.5~6.5시간 (기간: 올해, 출처: "올해 평균 수면 시간을 5.5~6.5 시간으로 늘렸다")
- 마이크로서비스 스터디 기간: 한달 정도 (기간: 사내 스터디, 출처: "한달 정도 진행한 마이크로서비스 스터디가 끝나버렸다.")
- 포커 책 학습 후 승률: 70% 이상 (기간: 1주일 학습 후 대결, 출처: "도서관에서 1주일동안 포커 관련 기술이랑 승리하는 방법이 나와있는 책을 통해 학습한 뒤 대결을 펼쳤는데, 이전과 비교했을때 승률이 70%를 넘었다고 한다.")
- 포커 선행 경험 차이: 5년 (기간: 비교 대상의 누적 경험, 출처: "자기보다 포커를 5년 먼저 시작하고 꾸준히 포커 클럽에서 모임도 갖고 사교 활동을한 형이 있는데")

## 언급된 도구·서비스
- Kotlin: 안드로이드와 서버 업무에서 공통으로 사용한 언어
- SpringBoot: 서버 업무를 위해 올해 초부터 학습한 기술로 언급됨
- Exposed: 서버 개발 학습 과정에서 사용해본 기술로 언급됨
- Protocol Buffers: 서버 개발 학습 과정에서 사용해본 기술로 언급됨
- Gradle Kotlin DSL: 서버 개발 업무를 위해 익혀야 하는 기술로 언급됨
- Docker: 서버 인프라 학습 과정에서 사용해본 기술로 언급됨
- AWS: 다양한 서비스를 학습 대상으로 언급함
- 쿠버네티스: 서버 인프라 학습 과정에서 대략적인 한 사이클을 돌려본 기술로 언급됨
- 스크럼: 서버팀 협업 방식으로 사용됨
- 회고: 서버팀 협업 방식으로 사용됨
- 페어 프로그래밍: 서버팀 협업 방식으로 사용됨
- 오프라인 리뷰: 서버팀 협업 방식으로 사용됨
- 헬스: 체중 감량을 위해 등록한 운동 수단
- PT: 체중 감량을 위해 헬스와 함께 등록한 개인 트레이닝

## 언급된 다른 creator·앱
- 동욱이 형: 매주 토요일 스터디에서 "가르치려고 해야 배운다"는 취지의 이야기를 반복해준 인물로 언급됨

## 복제 가능한 전술 (≤3)
1. 새 기술 스터디에서는 학습자보다 발표자 역할을 맡아 직접 정리하고 설명한다.
   - 구체적 스텝: 스터디 주제를 정한다 → 매 챕터별로 내용을 문서로 정리한다 → 스터디 시간에 직접 공유하고 설명한다 → 끝난 뒤 이해가 부족했던 부분을 다시 보완한다
   - 예상 리소스: 정리 문서 작성 시간, 스터디 모임, 발표 자료
   - 예상 효과: 배우기만 했을 때보다 더 많은 것을 얻고 성장했다는 체감을 만들 수 있다
2. 커리어 전환 초기에는 협업 루틴을 통해 문제 해결 방식과 코드 기준을 빠르게 흡수한다.
   - 구체적 스텝: 비슷한 종류의 일을 하는 팀에 합류한다 → 이미 잘 만들어진 코드를 찾아 읽는다 → 스크럼, 회고, 페어 프로그래밍, 리뷰에 반복 참여한다 → 리뷰받은 내용을 기준으로 해결 방식을 비교 학습한다
   - 예상 리소스: 협업 가능한 팀 환경, 코드베이스, 리뷰 시간
   - 예상 효과: 다양한 문제 해결 방법을 깨우치고, 잘한 부분을 공유하며 성장 감각을 회복할 수 있다
3. 체력과 집중력 회복이 필요하면 운동 빈도와 수면 시간을 먼저 관리한다.
   - 구체적 스텝: 헬스와 PT 같은 운동 루틴을 만든다 → 매주 3~5회 운동한다 → 식사 일부를 닭가슴살·고구마·샐러드 중심으로 조정한다 → 평균 수면시간을 5.5~6.5시간 수준으로 늘린다 → 커피 의존도와 피로 변화를 관찰한다
   - 예상 리소스: 헬스장과 PT 비용, 주간 운동 시간, 식단 준비, 수면 확보 시간
   - 예상 효과: 몸이 가벼워지고 자신감이 생기며, 피곤함이 줄고 업무 집중도와 학습 능력이 증가할 수 있다

## 원문 요약 (≤5문장)
작성자는 2019년 상반기에 클라이언트 개발자에서 서버 개발자로 커리어를 전환하며 기존 지식이 초기화된 듯한 감각 속에서 다시 학습을 시작했다. 혼자 개발하던 시기보다 서버팀에서 협업하는 현재가 코드 검증과 성장 측면에서 훨씬 도움이 된다고 평가했다. 마이크로서비스 스터디 경험을 통해 배우기만 하는 태도보다 직접 정리하고 가르치려는 태도가 더 중요하다고 결론 내렸다. 안드로이드는 잠시 내려놓고 서버 역량에 집중하기로 했으며, 운동과 수면 개선으로 체력과 집중력이 회복됐다고 적었다. 하반기에는 스프링부트, 리눅스, Gradle KTS, AWS, Docker, Kubernetes 등을 계속 학습하고 블로그에 정리할 계획을 세웠다.

## 본문 포인트별 발췌
> 클라이언트 개발자 > 서버 개발자로 커리어 전환을 했다.
> 장점은 마음 편하게 내가 사용하고 싶은 기술들을 마음껏 해볼 수 있다는 것이었고, 단점은 내가 작성한 코드에 대한 검증이 힘들고 자랑도 할 수가 없다는 것이었다.
> 스터디가 끝나고 난 뒤에 내가 배운것은 거의 없었다.
> "스터디를 정말 많이 그리고 열심히 해봤지만 배우려고 간 스터디에서 무엇인가를 얻은 기억은 하나도 없었지만, 가르치려고 했던 스터디는 정말 많은 것들을 얻었고 그속에서 성장했다."
> 수면은 중요한 것이라는 것을 깨닳았고 현재 수면의 질을 향상시키는 것은 진행중이다.
