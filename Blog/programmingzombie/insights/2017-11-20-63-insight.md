## 핵심 주장 (≤5)
1. Android Architecture Components는 안드로이드 생명주기 관리의 복잡성을 줄이고 데이터 유지에 도움을 준다.
2. Fabric Branch의 Deferred Deep Link는 설치 이후 가입·결제·이탈 지점을 포함한 사용자 유입 경로 분석에 유용하다.
3. 데이터의 중요성을 이해하고 적절한 도구를 선택해 적용하는 태도가 중요하다.
4. 디자인 패턴은 반복되는 문제를 빠르게 해결하고 생산성과 유지보수성을 높이는 방법이다.
5. 세미나를 통해 AAC, MVP, MVVM, 디자인 패턴 등 기본기를 더 체계적으로 학습해야 한다는 동기를 얻었다.

## 복제 가능한 전술 (≤3)
1. 앱 설치 이후까지 이어지는 유입 분석이 필요하면 Deferred Deep Link 기반 가입 경로 추적을 설계한다.
   - 구체적 스텝: Branch를 설치하고, 마켓 링크에 레퍼러를 붙인 스토어 링크를 배포한 뒤, `onNewIntent`와 `onStart`에서 Deep Link 파라미터를 받아 저장하고, 로그인·가입 같은 행동 이벤트를 전송한다.
   - 예상 리소스: Branch, 앱 코드 수정, SharedPreference 저장 처리
   - 예상 효과: 다운로드 이후 가입 여부, 이탈 지점, 특정 초대 링크 이후 행동까지 추적할 수 있다.
2. 안드로이드 생명주기 복잡도를 줄이기 위해 AAC 코드랩을 직접 빌드하며 학습한다.
   - 구체적 스텝: 코드랩 예제를 내려받아 Android Studio에서 빌드하고, 생명주기 문제를 어떤 방식으로 해결하는지 따라가며 ViewModel과 LiveData 동작을 확인한다.
   - 예상 리소스: Android Studio, AAC 코드랩 예제, 학습 시간
   - 예상 효과: 생명주기 처리 부담을 줄이고 화면 회전 시 데이터 유실 문제를 줄이는 구조를 이해할 수 있다.
3. 반복적으로 겪는 유지보수 문제를 디자인 패턴 관점에서 분류해 해결책을 적용한다.
   - 구체적 스텝: 데이터 변경 전파, 생성자 파라미터 과다, 기존 API 인터페이스 부적합 같은 문제를 식별하고 각각 Observer, Builder, Adapter 같은 패턴 후보로 대체 설계를 검토한다.
   - 예상 리소스: 기존 코드 리뷰 시간, 디자인 패턴 레퍼런스
   - 예상 효과: 확장성과 유지보수성을 높이고 코드 재사용과 팀 내 의사소통을 개선할 수 있다.

## 원문 요약 (≤5문장)
글쓴이는 DEVFEST 2017에서 들은 네 개 세션을 중심으로 학습 포인트와 느낀 점을 정리했다. AAC 세션에서는 안드로이드 생명주기 관리 복잡성을 줄이고 ViewModel과 LiveData로 데이터 유실을 줄이는 방식을 인상적으로 받아들였다. Fabric Branch 세션에서는 Deferred Deep Link를 통해 설치 이후 가입·결제까지 이어지는 사용자 경로 분석이 가능하다는 점을 핵심으로 정리했다. UI Test 세션은 이해가 어려웠지만 CI, Espresso, Dagger2, Mockito, Firebase Test Lab 등 더 공부해야 할 영역을 확인하는 계기가 되었다. 마지막으로 디자인 패턴 세션을 통해 좋은 코드는 생산성과 유지보수성에 직결되며, 기초를 더 탄탄히 다져야겠다는 동기를 얻었다.

## 본문 포인트별 발췌
> - Activity의 생명주기들을 자동으로 관리가능.
> ViewModel을 사용하면 앱을 종료하지 않는이상(Foreground 상태) 데이터가 유지된다.
> Fabric에서는 Deferred Deep Link를 쉽게 구현할 수 있도록 Branch라는 라이브러리를 제공하고 있다.
> - 데이터의 중요성을 이해하고 적절한 수단을 찾아 적용하려는 마음가짐이 중요하다.
> - 좋은 코드를 고민하는 개발자들에게 디자인 패턴은 하나의 방법이 될 수 있다.
