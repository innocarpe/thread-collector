## 핵심 주장 (≤5)
1. 안드로이드 단말에 설치된 앱의 APK 파일은 추출할 수 있다.
2. APK 추출 방법은 유틸성 앱을 사용하는 방법과 ADB 명령어를 사용하는 방법, 크게 두 가지로 제시된다.
3. 추출 대상 APK를 찾으려면 먼저 `adb shell pm list packages -f`로 설치된 패키지 목록을 확인해야 한다.
4. 원하는 패키지를 빠르게 찾기 위해 `grep`으로 검색 범위를 좁히는 방식이 유용하다.
5. 원하는 패키지 경로를 찾은 뒤 `adb pull {패키지 경로.apk} {apk가 저장될 경로}`로 APK를 저장 위치에 추출할 수 있다.

## 언급된 도구·서비스
- MyAppSharer: APK 추출을 도와주는 유틸성 앱의 예시로 언급됨
- ADB: 설치된 패키지 목록 조회와 APK 파일 추출에 사용하는 명령줄 도구로 사용됨
- 안드로이드 스튜디오 터미널: USB로 폰을 연결한 뒤 명령어를 실행하는 터미널 예시로 언급됨
- grep: 패키지 목록에서 원하는 앱 관련 항목만 좁혀 찾는 데 사용됨

## 언급된 다른 creator·앱

## 복제 가능한 전술 (≤3)
1. ADB로 설치된 앱의 APK 경로를 조회한 뒤 로컬로 복사한다.
   - 구체적 스텝: 폰을 USB로 컴퓨터에 연결한다. 터미널을 연다. `adb shell pm list packages -f`로 패키지 목록을 확인한다. 필요하면 `adb shell pm list packages -f | grep photo`처럼 검색어를 붙여 원하는 패키지명을 찾는다. 확인한 경로를 `adb pull {패키지 경로.apk} {apk가 저장될 경로}`에 넣어 실행한다.
   - 예상 리소스: USB 케이블, 안드로이드 단말, ADB가 실행 가능한 터미널 환경
   - 예상 효과: 폰에 설치된 특정 앱의 APK 파일을 원하는 로컬 경로로 추출할 수 있다.
2. 유틸성 앱을 이용해 APK 추출 작업을 단순화한다.
   - 구체적 스텝: APK 추출용 유틸성 앱을 설치한다. 글에서 예시로 든 `MyAppSharer` 같은 앱을 사용해 설치된 앱의 APK를 내보낸다.
   - 예상 리소스: APK 추출용 유틸성 앱
   - 예상 효과: ADB를 쓰지 않고도 APK 추출을 수행하는 대안이 된다.

## 원문 요약 (≤5문장)
글은 안드로이드 단말에 설치된 앱의 APK 파일을 추출하는 방법을 소개한다. 작성자는 유틸성 앱을 쓰는 방법과 ADB 명령어를 쓰는 방법, 두 가지를 제시한 뒤 개발자이기 때문에 ADB 방식을 선택했다고 설명한다. 절차는 USB 연결 후 터미널을 열고, `adb shell pm list packages -f`로 패키지 목록을 확인하는 것부터 시작한다. 원하는 앱은 `grep`으로 검색 범위를 좁혀 찾고, 마지막으로 `adb pull` 명령어로 APK를 원하는 위치에 복사한다.

## 본문 포인트별 발췌

> 크게 두가지 방법이 있는데,
> 2. **간지나게 ADB 명령어로 APK파일을 원하는 장소에 추출한다.**

> adb pull {패키지 경로.apk} {apk가 저장될 경로}
