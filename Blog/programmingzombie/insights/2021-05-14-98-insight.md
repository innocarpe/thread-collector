## 핵심 주장 (≤5)
1. Mac 환경에서는 Raspberry Pi 공식 홈페이지에서 Raspberry Pi Imager를 내려받아 SD 카드에 OS를 설치할 수 있다.
2. Desktop이 필요 없다면 Raspberry Pi OS (other) 메뉴의 Desktop 없는 라이트 버전을 선택하는 것이 적합하다.
3. 설치 후 기본 로그인 정보는 `ID: pi`, `PW: raspberry`다.
4. 모니터 없이도 SD 카드에 `ssh` 파일과 `wpa_supplicant.conf`를 넣어두면 Wifi 설정과 SSH 접속 준비를 할 수 있다.
5. `ping raspberrypi.local`로 IP를 확인한 뒤 `ssh pi@{IP}`로 접속하면 헤드리스 상태에서도 라즈베리파이를 확인할 수 있다.

## 공개된 숫자·지표
- 발견한 라즈베리파이 수: 2대 (기간: 없음, 출처: "간만에 집 구석구석 정리를 하다가 문득 발견한 라즈베리파이 3 2대")
- UI 포함 추천 OS 용량: 1.1 GB (기간: 없음, 출처: "반대로 UI 요소가 포함된 OS를 설치를 원하시면 상단에 추천하고 있는 라즈베리파이 1.1 GB를 설치하시면 됩니다.")
- 예시로 확인한 라즈베리파이 IP 주소: 192.168.0.39 (기간: 설치 후 네트워크 확인 시점, 출처: "PING raspberrypi.local (192.168.0.39): 56 data bytes")

## 언급된 도구·서비스
- Raspberry Pi 공식 홈페이지: macOS용 설치 파일을 받기 위해 접속한 다운로드 페이지로 사용됨.
- Raspberry Pi Imager: 설치할 OS와 Storage를 선택하고 SD 카드에 이미지를 쓰는 도구로 사용됨.
- Raspberry Pi OS: SD 카드에 설치하는 운영체제로 사용됨.
- Mac/MacBook: 설치 작업과 SD 카드 파일 편집을 진행한 환경으로 사용됨.
- Terminal: `vim`, `ping`, `ssh` 명령을 실행해 설정과 접속 확인을 하는 도구로 사용됨.
- vim: `ssh`, `wpa_supplicant.conf` 파일을 생성하는 편집기로 사용됨.
- ssh: 모니터 없이 라즈베리파이에 원격 접속하는 방식으로 사용됨.
- ping: `raspberrypi.local`과 `www.google.com` 응답을 확인해 IP 탐색 및 네트워크 연결 점검에 사용됨.
- Google: 라즈베리파이 내부에서 인터넷 연결 상태를 점검하는 대상으로 사용됨.

## 복제 가능한 전술 (≤3)
1. Mac에서 Raspberry Pi Imager로 필요한 형태의 Raspberry Pi OS를 SD 카드에 설치한다.
   - 구체적 스텝: Raspberry Pi 공식 홈페이지의 macOS 다운로드 페이지로 이동해 설치 파일을 받는다. 다운로드한 파일을 Mac에 설치한 뒤 Raspberry Pi Imager를 실행한다. 설치할 OS와 Storage를 선택하고, Desktop이 필요 없으면 `Raspberry Pi OS (other)`에서 Desktop 없는 라이트 버전을 고른다. 선택이 끝나면 `WRITE` 버튼을 눌러 SD 카드에 설치한다.
   - 예상 리소스: Mac, SD 카드, Raspberry Pi Imager, Raspberry Pi OS 이미지
   - 예상 효과: SD 카드에 Raspberry Pi OS가 설치되고, 라즈베리파이에 꽂아 전원을 연결하면 설치 여부를 확인할 수 있다.
2. 모니터 없이 부팅할 수 있도록 SD 카드에 SSH 활성화 파일과 Wifi 설정 파일을 미리 넣는다.
   - 구체적 스텝: Mac에서 SD 카드에 접근한 뒤 빈 `ssh` 파일을 `vim ssh`로 생성하고 저장한다. 이어 `vim wpa_supplicant.conf`로 파일을 만든 뒤 원문의 형식대로 `country`, `ctrl_interface`, `update_config`, `network`, `ssid`, `psk`, `scan_ssid`, `key_mgmt` 값을 입력해 저장한다. 설정이 끝난 SD 카드를 라즈베리파이에 꽂고 전원을 켠다.
   - 예상 리소스: Mac 터미널, SD 카드, Wifi 이름과 비밀번호
   - 예상 효과: 모니터를 매번 연결하지 않아도 SSH로 접속할 준비를 할 수 있다.
3. 로컬 네트워크에서 라즈베리파이 IP를 찾고 SSH와 인터넷 연결까지 확인한다.
   - 구체적 스텝: 터미널에서 `ping raspberrypi.local`을 실행해 라즈베리파이의 IP 주소를 확인한다. 확인한 IP로 `ssh pi@{IP 주소}`를 입력해 접속하고 초기 비밀번호 `raspberry`를 넣는다. 접속 후 라즈베리파이 내부에서 `ping www.google.com`을 실행해 응답이 돌아오는지 확인한다.
   - 예상 리소스: 터미널, 로컬 네트워크, SSH 접속 가능한 Raspberry Pi
   - 예상 효과: 모니터 없이도 라즈베리파이에 접속하고 Wifi 연결 상태까지 점검할 수 있다.

## 원문 요약 (≤5문장)
이 글은 Mac 환경에서 Raspberry Pi 3용 OS를 SD 카드에 설치하는 과정을 정리한 설치 기록이다. 작성자는 Raspberry Pi 공식 홈페이지에서 Raspberry Pi Imager를 내려받아 OS와 Storage를 선택한 뒤 SD 카드에 이미지를 쓰는 방법을 설명한다. Desktop이 필요 없을 경우 라이트 버전을 고를 수 있고, 설치 후 기본 로그인 정보로 접속할 수 있다고 안내한다. 또 모니터 없이 사용하기 위해 SD 카드에 `ssh` 파일과 `wpa_supplicant.conf`를 넣어 Wifi와 SSH를 미리 설정하는 방법을 소개한다. 마지막으로 `ping`과 `ssh`를 이용해 라즈베리파이의 IP를 찾고 인터넷 연결까지 확인하는 흐름을 보여준다.
