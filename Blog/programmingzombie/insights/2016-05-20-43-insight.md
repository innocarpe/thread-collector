## 핵심 주장 (≤5)
1. 액션바 중앙에 이미지를 배치하려면 `setCustomView`로 커스텀 레이아웃을 적용하고 기본 타이틀 표시를 끄는 방식이 유효하다.
2. 액션바 아이콘을 숨기려면 `setDisplayUseLogoEnabled(false)`와 `setDisplayShowHomeEnabled(false)`를 함께 설정할 수 있다.
3. 액션바 사용 시 API 버전에 따라 `< 11`에서는 `getSupportActionBar()`, `> 11`에서는 `getActionBar()`를 사용해야 한다.
4. `Toolbar`에서도 중앙 정렬된 `ImageView`를 넣는 방식으로 유사한 커스텀 구성이 가능하다.

## 언급된 도구·서비스
- ActionBar: 커스텀 뷰를 적용하고 제목, 로고, 홈 아이콘 노출을 제어하는 데 사용됨
- Toolbar: 액션바 대안으로 중앙 정렬 `ImageView`를 넣는 예시가 제시됨
- ImageView: 액션바 또는 툴바 중앙에 로고 이미지를 표시하는 데 사용됨
- `getActionBar()`: 기본 ActionBar 인스턴스를 가져오는 데 사용됨
- `getSupportActionBar()`: 낮은 API 대응을 위해 ActionBar 대체 접근으로 사용됨

## 복제 가능한 전술 (≤3)
1. 액션바 중앙 로고 레이아웃을 커스텀 뷰로 삽입한다.
   - 구체적 스텝: `actionbar_custom_view_home.xml`에 `android:gravity="center"`인 `LinearLayout`과 `ImageView`를 만든다; 액티비티 `onCreate`에서 `actionBar.setCustomView(...)`를 호출한다; `setDisplayShowTitleEnabled(false)`와 `setDisplayShowCustomEnabled(true)`를 적용한다.
   - 예상 리소스: 안드로이드 XML 레이아웃 편집, 액티비티 코드 수정, `ActionBar` 또는 `SupportActionBar`
   - 예상 효과: 액션바 중앙에 이미지를 배치한 커스텀 헤더를 구성할 수 있다.
2. 기본 액션바 아이콘과 홈 표시를 비활성화해 커스텀 헤더만 남긴다.
   - 구체적 스텝: 커스텀 뷰 적용 이후 `setDisplayUseLogoEnabled(false)`와 `setDisplayShowHomeEnabled(false)`를 추가한다; 기본 타이틀은 계속 비활성화한다.
   - 예상 리소스: 기존 액션바 설정 코드 수정
   - 예상 효과: 기본 아이콘이 제거되어 커스텀 뷰 중심의 액션바 구성이 가능하다.
3. `Toolbar`로 전환해 동일한 중앙 로고 구성을 구현한다.
   - 구체적 스텝: 레이아웃에 `android.support.v7.widget.Toolbar`를 추가한다; 내부에 `android:layout_gravity="center"`를 가진 `ImageView`를 배치한다; 툴바 스타일과 배경색을 지정한다.
   - 예상 리소스: `Toolbar` 레이아웃 정의, 앱 테마 또는 스타일 설정
   - 예상 효과: `Toolbar` 기반 UI에서도 중앙 로고 형태의 상단 바를 구현할 수 있다.

## 원문 요약 (≤5문장)
글은 액션바를 커스텀하는 간단한 안드로이드 예시 코드를 소개한다.
핵심은 커스텀 XML 레이아웃을 액션바에 적용하고 기본 타이틀을 숨겨 중앙 이미지 형태로 구성하는 것이다.
추가로 기본 액션바 아이콘과 홈 버튼을 숨기는 설정도 함께 제시한다.
API 버전에 따라 `getActionBar()`와 `getSupportActionBar()` 사용 기준을 구분해야 한다고 설명한다.
마지막에는 `Toolbar`로 같은 구성을 구현하는 예시 XML도 덧붙였다.
