## 핵심 주장 (≤5)
1. `split`으로 특수문자를 구분자로 사용해 문자열을 나눌 때 의도대로 분리되지 않는 경우가 있다.
2. 문제 원인은 일부 특수문자가 `split`에 곧바로 적용되지 않는 데 있다.
3. 예제에서는 `&`, `^`가 문제의 원인이었고, 해당 문자 앞에 `\\`를 붙여 해결했다.
4. `.` , `*` 등 다른 문자도 바로 적용되지 않을 수 있다.
5. 결론적으로 바로 동작하지 않는 특수문자 앞에는 `\\`를 붙여야 한다.

## 복제 가능한 전술 (≤3)
1. Java `split` 구분자에 특수문자가 포함될 때 문제 문자를 이스케이프해서 분리 오류를 해결한다.
   - 구체적 스텝: 재현용 문자열과 구분자를 만든다 -> `data.split(delimiter)` 결과가 기대와 다른지 확인한다 -> 구분자 안에서 문제가 되는 특수문자 앞에 `\\`를 붙인다 -> 다시 실행해 분리 결과를 검증한다
   - 예상 리소스: Java 실행 환경, 간단한 테스트 코드 작성 및 실행 시간
   - 예상 효과: 원문 사례처럼 특수문자 구분자로 인해 나뉘지 않던 문자열을 정상적으로 분리할 수 있다
2. 특수문자 구분자 사용 시 디버깅으로 어떤 문자가 원인인지 먼저 특정한다.
   - 구체적 스텝: 구분자 문자열을 기준으로 `split` 결과를 확인한다 -> 특수문자를 하나씩 점검하며 어떤 문자가 바로 적용되지 않는지 확인한다 -> 해당 문자만 우선 이스케이프 처리한다
   - 예상 리소스: 디버거 또는 반복 실행 가능한 개발 환경
   - 예상 효과: 문제 문자를 빠르게 특정해 불필요한 수정 없이 원인을 해결할 수 있다

## 원문 요약 (≤5문장)
글은 Java에서 `split`으로 특수문자를 구분자로 사용했을 때 문자열이 제대로 나뉘지 않는 문제를 다룬다. 작성자는 디버깅을 통해 일부 특수문자가 `split`에 바로 적용되지 않는다는 점을 확인했다. 예제에서는 `&`, `^`가 문제였고, 구분자 문자열에서 해당 문자 앞에 `\\`를 붙여 해결했다. 추가로 `.`, `*` 등도 같은 성격의 문제를 일으킬 수 있다고 언급한다. 결론은 동작하지 않는 특수문자 앞에 `\\`를 붙이라는 것이다.
