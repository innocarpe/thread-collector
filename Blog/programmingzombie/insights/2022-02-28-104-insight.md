## 핵심 주장 (≤5)
1. ElasticBeanstalk에서 RDS용 데이터베이스를 추가할 때 웹콘솔에서 에러가 발생할 수 있다.
2. 글쓴이는 원인을 MySQL 5.6이 deprecated 처리되었지만 웹콘솔에는 반영되지 않은 점으로 추정한다.
3. 해결을 위해 AWS ElasticBeanstalk CLI를 설치하고 `eb init`으로 기본 환경 설정을 해야 한다.
4. `eb config`에서 `aws:rds:dbinstance`의 `DBEngineVersion`을 `'8.0'`으로 추가하면 정상적으로 DB 연결이 동작한다.

## 공개된 숫자·지표
- MySQL 엔진 버전: 5.6 (기간: 문제 발생 시점, 출처: `Configuration validation exception: Invalid option value: '5.6'`)
- MySQL 엔진 버전: 8.0 (기간: 해결 설정 적용 시점, 출처: `DBEngineVersion: '8.0'`)

## 언급된 도구·서비스
- AWS ElasticBeanstalk: 애플리케이션 환경을 만들고 RDS 연결을 시도한 서비스
- RDS: ElasticBeanstalk에서 사용하려고 추가한 데이터베이스 서비스
- AWS 웹콘솔: MySQL 5.6 관련 설정이 반영되지 않아 에러가 나는 맥락으로 언급됨
- AWS ElasticBeanstalk CLI: `eb init`, `eb config`로 환경 설정과 config 수정을 수행하는 도구
- MySQL: RDS 엔진 버전 문제와 해결 대상인 데이터베이스 엔진

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. ElasticBeanstalk 웹콘솔에서 RDS 추가 시 MySQL 버전 오류가 나면 EB CLI로 엔진 버전을 직접 고정한다.
   - 구체적 스텝: AWS ElasticBeanstalk CLI를 설치한다 -> `eb init`을 실행해 region과 application을 선택한다 -> `eb config`를 열어 `aws:rds:dbinstance` 항목에 `DBEngineVersion: '8.0'`을 추가한다 -> 설정 반영이 끝난 뒤 다시 DB 연결을 시도한다
   - 예상 리소스: EB CLI, ElasticBeanstalk 환경 접근 권한, 설정 수정 시간
   - 예상 효과: MySQL 8.0이 적용되어 다시 DB 연결을 눌렀을 때 정상적으로 동작한다

## 원문 요약 (≤5문장)
글은 ElasticBeanstalk에서 RDS 데이터베이스를 추가할 때 발생하는 에러와 그 해결 방법을 다룬다. 작성자는 일부 리전에서는 되고 일부 리전에서는 같은 에러가 발생한다고 설명한다. 원인을 MySQL 5.6 deprecated 처리와 웹콘솔 미반영으로 추정한다. 해결책으로 AWS ElasticBeanstalk CLI를 설치하고 `eb init` 후 `eb config`에서 `DBEngineVersion`을 `8.0`으로 추가하라고 안내한다. 이렇게 설정하면 잠시 후 MySQL 8.0이 적용되어 DB 연결이 정상 동작한다고 정리한다.

## 본문 포인트별 발췌
> ElasticBeanstalk을 설정하고 RDS 사용을 위해 데이터베이스를 추가하면 위의 스크린샷과 같이 에러가 발생합니다.
> AWS의 버그(?)같긴한데, 다른 리전으로 변경하며 테스트 해보면 되는 곳도 있고 위와같이 에러가 발생하며 되지 않는 곳도 있습니다.
> 원인은 Mysql 5.6버전이 deprecated처리를 하고 웹콘솔에는 적용이 안되서 인 것 같습니다 😩
> eb config를 입력하여 가장 하단에 있는 **aws:rds:dbinstance 값에 DBEngineVersion: '8.0' 으로 추가**해줍니다.
> AWS ElasticBeanstalk이 수정중으로 변경되고, 잠시 기다리면 Mysql 8.0으로 적용되어 다시 DB연결을 눌렀을때 정상적으로 동작하는 것을 확인 하실 수 있습니다.
