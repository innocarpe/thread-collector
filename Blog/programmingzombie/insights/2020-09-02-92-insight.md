## 핵심 주장 (≤5)
1. `chunked`는 리스트를 "3개의 큰 덩어리"로 나누는 함수라기보다, 지정한 크기의 작은 청크 단위로 분할한다.
2. 리스트 크기가 커지면 `chunked(3)`는 작성자가 의도한 "3등분"과 다르게 동작할 수 있다.
3. 원하는 등분 단위로 리스트를 나누려면 리스트 크기를 나눈 값을 `chunked` 또는 `windowed`에 넣어야 한다.
4. `windowed`는 기본적으로 `partialWindows = false`라서 나머지 값을 버릴 수 있으므로, 남는 값을 포함하려면 `true`를 명시해야 한다.
5. 분할한 두 개의 `List<List<Int>>`를 교차 삽입한 뒤 `flatten()`으로 평탄화하면 원하는 순서의 결과 리스트를 만들 수 있다.

## 공개된 숫자·지표
- A List 원소 범위: 1~9 (기간: 해당 예시, 출처: "val aList = (1..9).chunked(3).toMutableList()")
- B List 원소 범위: 100~900, 간격 100 (기간: 해당 예시, 출처: "val bList = (100..900 step 100).chunked(3)")
- 분할 기준: 3 (기간: 해당 예시, 출처: "List를 3등분으로 분할 하기위해 chunked(3)를 사용")
- 테스트 결과 리스트 개수: 34개 (기간: 테스트 사례, 출처: "테스트를 돌려본 결과 3의 결과를 기대 했지만 무려 34개의 리스트로 잘게 쪼개졌다.")
- 마지막값 오차: 예상 100, 실제 98 (기간: 추가 테스트 사례, 출처: "마지막값이 100이라고 예상했는데, 98이 찍혀있었다.")

## 언급된 도구·서비스
- Kotlin `chunked`: 리스트를 지정한 크기의 청크로 분할하는 데 사용
- Kotlin `windowed`: `chunked` 내부 동작을 확인하고, 원하는 등분과 `partialWindows` 제어를 위해 직접 사용하는 함수로 언급
- Kotlin `flatten`: `List<List<Int>>`를 `List<Int>`로 평탄화하는 데 사용
- 테스트: `chunked`와 `windowed`의 실제 동작 및 실패 여부를 확인하는 용도로 사용

## 복제 가능한 전술 (≤3)
1. 리스트를 "N개의 큰 덩어리"로 나누려면 전체 크기를 기준으로 분할 크기를 계산해 `chunked` 또는 `windowed`에 넣는다.
   - 구체적 스텝: 전체 리스트 길이를 기준으로 나눌 크기(`divideSize`)를 구한다; `testList.chunked(divideSize)` 또는 `testList.windowed(divideSize, divideSize, true)`를 적용한다; 결과가 의도한 등분인지 테스트로 확인한다.
   - 예상 리소스: Kotlin 컬렉션 함수, 간단한 테스트 코드
   - 예상 효과: `chunked(3)`처럼 청크 개수와 등분 개수를 혼동해 잘못 분할하는 문제를 줄일 수 있다.
2. 두 개의 분할 리스트를 교차 병합할 때는 `forEachIndexed`로 삽입 위치를 계산한다.
   - 구체적 스텝: 두 리스트를 동일한 단위로 분할한다; 기준 리스트를 `MutableList`로 만든다; 다른 리스트를 순회하면서 `index * 2` 위치에 각 청크를 삽입한다.
   - 예상 리소스: `toMutableList()`, `forEachIndexed`
   - 예상 효과: 각 청크를 원하는 순서로 끼워 넣어 교차 배치된 중간 구조를 만들 수 있다.
3. 중첩 리스트를 최종 결과 리스트로 바꿀 때는 `flatten()`으로 마무리한다.
   - 구체적 스텝: 병합 결과가 `List<List<Int>>` 형태인지 확인한다; `flatten()`을 호출해 단일 리스트로 변환한다; 결과 순서를 검증한다.
   - 예상 리소스: Kotlin `flatten`
   - 예상 효과: 교차 삽입 후 생긴 중첩 구조를 평평하게 만들어 최종 출력 형식으로 정리할 수 있다.

## 원문 요약 (≤5문장)
글은 두 개의 숫자 리스트를 일정 단위로 나눈 뒤 교차로 병합해 원하는 순서의 결과 리스트를 만드는 방법을 설명한다. 작성자는 처음에 `chunked(3)`으로 3등분을 시도했지만, 이 함수가 "3개의 큰 덩어리"가 아니라 "크기 3의 청크"를 만든다는 점을 테스트로 확인했다. 이후 원하는 등분을 만들기 위해 리스트 크기를 나눈 값을 `chunked`나 `windowed`에 넣는 방식이 필요하다고 정리한다. 특히 `windowed`는 기본적으로 남는 값을 버릴 수 있어 `partialWindows = true`를 알아야 한다고 강조한다. 마지막으로 분할된 리스트를 인덱스 기준으로 끼워 넣고 `flatten()`으로 평탄화해 최종 리스트를 만든다.

## 본문 포인트별 발췌
> chucked의 경우 List를 3개의 큰 덩어리로 만들어주는게 아닌, 작은 3개의 청크 단위로 만들어주는 것임을 알 수 있다.
> 그렇다면 내가 원하는 List를 3개의 큰 덩어리로 만드려면 windowed에 List Size를 3으로 나눈 값을 넣어주면 될 것 같았다.
> 마지막값이 100이라고 예상했는데, 98이 찍혀있었다. 나눠지고 남은 값들은 버려버린 것이다.
> 결론 적으로 List를 내가 원하는 등분으로 나누려면 두가지 방법중 원하는 것을 사용하면 되는 것을 알 수 있었다.
> List<List<Int> => List<Int>로 flatten은 말그대로 평평하게 만드는 작업을 한다.
