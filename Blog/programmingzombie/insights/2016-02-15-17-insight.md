## 핵심 주장 (≤5)
1. 가상머신에 우분투를 설치한 뒤 `openssh-server`를 추가 설치하면 Windows 10의 `Xshell5`에서 우분투 SSH로 접근할 수 있다.
2. 글의 목적은 SSH 접근 이후에도 다시 참고할 수 있도록 리눅스 서버의 부가 설정을 메모해 두는 것이다.
3. `/etc/network/interfaces`에서 `eth0`를 `static`으로 두고 주소, 게이트웨이, DNS를 명시하는 방식으로 네트워크를 수동 설정할 수 있다.
4. `/etc/ssh/sshd_config`에서 포트, 인증, PAM, DNS 같은 SSH 서버 동작 옵션을 직접 조정할 수 있다.

## 공개된 숫자·지표
- SSH 포트: `22` (기간: 설정 시점, 출처: `Port 22`)
- SSH 프로토콜 버전: `2` (기간: 설정 시점, 출처: `Protocol 2`)
- 서버 키 재생성 간격: `3600` (기간: 설정 시점, 출처: `KeyRegenerationInterval 3600`)
- 서버 키 비트 수: `1024` (기간: 설정 시점, 출처: `ServerKeyBits 1024`)
- 로그인 유예 시간: `120` (기간: 설정 시점, 출처: `LoginGraceTime 120`)
- X11 디스플레이 오프셋: `10` (기간: 설정 시점, 출처: `X11DisplayOffset 10`)
- DNS 서버 수: `2개` (기간: 네트워크 설정 시점, 출처: `dns-nameservers 168.126.63.1 168.126.63.2`)

## 언급된 도구·서비스
- Ubuntu: 가상머신에 설치한 서버 운영체제로 사용됨
- `sudo apt-get install openssh-server`: 우분투에 SSH 서버를 설치하는 명령으로 사용됨
- Windows 10: SSH 클라이언트에서 접속을 시도한 호스트 OS로 언급됨
- Xshell5: Windows 10에서 우분투 SSH에 접속하는 클라이언트로 사용됨
- `vi`: `/etc/network/interfaces`, `/etc/ssh/sshd_config`를 수정하는 편집기로 사용됨
- SSH: Windows 10에서 우분투에 원격 접근하는 방식으로 사용됨
- PAM: SSH 설정에서 인증과 세션 처리 옵션으로 언급됨
- SFTP: `Subsystem sftp /usr/lib/openssh/sftp-server` 설정으로 언급됨

## 복제 가능한 전술 (≤3)
1. 우분투 가상머신에 SSH 서버를 설치하고 외부 클라이언트로 접속을 검증한다.
   - 구체적 스텝: 가상머신에 우분투를 설치한 뒤 `sudo apt-get install openssh-server`를 실행하고, 설치 후 Windows 10에서 `Xshell5`로 우분투의 SSH에 접속해 정상 접근 여부를 확인한다.
   - 예상 리소스: 우분투 가상머신, Windows 10 PC, `Xshell5`, `openssh-server`
   - 예상 효과: 원문처럼 Windows 10 환경에서 우분투 SSH에 정상적으로 접근할 수 있다.
2. `/etc/network/interfaces`에 정적 네트워크 설정을 기록해 서버 주소를 고정한다.
   - 구체적 스텝: `vi /etc/network/interfaces`를 열고 `auto eth0`, `iface eth0 inet static`, `address`, `netmask`, `network`, `gateway`, `dns-nameservers` 항목을 환경에 맞게 입력한다.
   - 예상 리소스: `vi`, 네트워크 정보(address, netmask, gateway, DNS)
   - 예상 효과: 원문처럼 `eth0` 인터페이스에 수동(static) 네트워크 구성을 적용할 수 있다.
3. `/etc/ssh/sshd_config`에서 인증·접속 관련 SSH 옵션을 명시한다.
   - 구체적 스텝: `vi /etc/ssh/sshd_config`를 열고 `Port 22`, `Protocol 2`, `PermitRootLogin without-password`, `PermitEmptyPasswords no`, `ChallengeResponseAuthentication no`, `UsePAM yes`, `UseDns no` 등 필요한 값을 점검·수정한다.
   - 예상 리소스: `vi`, SSH 서버 설정 파일 접근 권한
   - 예상 효과: 원문에 적힌 방식대로 SSH 서버의 로그인 정책과 동작 옵션을 한곳에서 관리할 수 있다.

## 원문 요약 (≤5문장)
글은 우분투 가상머신에 `openssh-server`를 설치한 뒤 Windows 10의 `Xshell5`로 SSH 접속에 성공한 경험을 바탕으로, 잊지 않기 위한 서버 설정 메모를 남긴 것이다. 본문은 설명형 서술보다 `/etc/network/interfaces`와 `/etc/ssh/sshd_config`에 넣은 실제 설정값을 그대로 기록하는 데 집중한다. 네트워크 설정에서는 `eth0`를 정적 IP 방식으로 두고 주소, 게이트웨이, DNS를 명시했다. SSH 설정에서는 포트, 프로토콜, 인증, X11, PAM, DNS 관련 옵션을 나열했다.

## 본문 포인트별 발췌
> 가상머신에 우분투를 설치하고,
> 명령어 "sudo apt-get install openssh-server" 를 입력한 다음 설치가 이루어지고 난 뒤
> 다시 본래의 Windows10에서 Xshell5를 통하여 설치한 우분투의 ssh로 접근을 하였다.
> 정상적으로 잘 접근이 되는 것을 확인 하였고, 부가적으로 필요한 셋팅에 대해 잊어먹지 않기 위해
> 메모해둔다.
