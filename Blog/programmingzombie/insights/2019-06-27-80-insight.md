## 핵심 주장 (≤5)
1. IntelliJ에서 하단바와 우측바의 내용이 보이지 않을 때 원인은 `tool window bar`가 숨겨진 상태일 수 있다.
2. `Build, Message, Java Enterprise, Version Controll, Spring, Run, TODO, Gradle, Maven` 같은 항목은 `tool window bar`를 다시 표시하면 나타난다.
3. 해결 방법은 설정에서 `Show tool window bars`를 체크하는 것이다.

## 복제 가능한 전술 (≤3)
1. IntelliJ에서 사라진 tool window bar를 설정 메뉴에서 복구한다.
   - 구체적 스텝: 설정을 연다 -> `tool window`를 검색한다 -> `Appearance & Behavior > Appearance`로 이동한다 -> `Show tool window bars`를 체크한다.
   - 예상 리소스: IntelliJ, 설정 메뉴 접근 시간
   - 예상 효과: 원문 기준으로 안 보이던 툴바들이 다시 나타난다.

## 원문 요약 (≤5문장)
글은 IntelliJ에서 프로젝트 생성 후 하단바와 우측바의 주요 창들이 보이지 않을 때의 해결 방법을 설명한다.
작성자는 해당 UI 요소의 명칭을 몰라 한참 헤맸고, 이름이 `tool window bar`임을 찾았다고 말한다.
해결 방법은 설정에서 `tool window`를 검색한 뒤 `Appearance & Behavior > Appearance > Show tool window bars`를 체크하는 것이다.
이 설정을 적용하면 보이지 않던 툴바들이 다시 나타난다고 정리한다.
