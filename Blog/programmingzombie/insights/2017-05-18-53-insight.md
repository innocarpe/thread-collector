## 핵심 주장 (≤5)
1. RecyclerView 내부 CheckBox는 스크롤 후 재등장할 때 체크값이 초기화되는 문제가 생길 수 있다.
2. `holder.cbSelect.setChecked(objIncome.isSelected());`로 데이터 객체의 선택 상태를 CheckBox에 다시 반영해야 한다.
3. `setChecked` 전에 `holder.cbSelect.setOnCheckedChangeListener(null);`로 리스너를 초기화해야 스크롤 후 체크값 초기화 문제를 막을 수 있다.
4. CheckBox 변경 이벤트에서 `objIncome.setSelected(isChecked);`를 호출해 데이터 객체에 마지막 체크 상태를 저장해야 한다.

- CheckBox: RecyclerView 각 아이템 내부에서 선택 상태를 표시하고 변경하는 UI 요소로 사용됨

- XML: RecyclerView 아이템 레이아웃 구조를 설명하는 예시 파일 형식

## 복제 가능한 전술 (≤3)
1. RecyclerView 아이템의 체크 상태를 뷰가 아니라 데이터 객체에 저장한다.
   - 구체적 스텝: 아이템 모델에 선택 상태 필드를 둔다; `onBindViewHolder`에서 `holder.cbSelect.setChecked(objIncome.isSelected());`로 상태를 반영한다; `OnCheckedChangeListener`에서 `objIncome.setSelected(isChecked);`로 변경값을 저장한다.
   - 예상 리소스: RecyclerView Adapter 수정, 아이템 모델 필드 추가, Android Java 코드 편집
   - 예상 효과: 사용자가 체크한 값을 객체에 저장해 재바인딩 시 다시 표시할 수 있다.
2. `setChecked` 전에 CheckBox 리스너를 초기화해 재바인딩 중 불필요한 체크 이벤트를 막는다.
   - 구체적 스텝: `onBindViewHolder`에서 먼저 `holder.cbSelect.setOnCheckedChangeListener(null);`를 호출한다; 그 다음 `setChecked(objIncome.isSelected())`를 호출한다; 마지막으로 새 `OnCheckedChangeListener`를 다시 연결한다.
   - 예상 리소스: Adapter 바인딩 로직 수정, Android Java 코드 편집
   - 예상 효과: 글 작성자 기준으로 스크롤 후 다시 올렸을 때 체크값이 초기화되는 문제가 해결된다.

## 원문 요약 (≤5문장)
글 작성자는 개인 개발 중 RecyclerView 내부에 CheckBox를 넣었을 때 스크롤 후 체크 상태가 초기화되는 문제를 겪었다. 검색을 통해 Stack Overflow 예시 코드를 참고해 데이터 객체의 선택 상태를 `setChecked`와 `setSelected`로 연결하는 방식을 사용했다. 하지만 이 방식만으로는 RecyclerView를 아래로 스크롤했다가 다시 올렸을 때 체크값이 초기화되는 문제가 남아 있었다. 해결책은 `setChecked` 전에 `setOnCheckedChangeListener(null)`을 호출해 리스너를 초기화한 뒤 다시 리스너를 연결하는 것이다. 이렇게 하면 기존 코드가 정상 작동한다고 정리한다.

## 본문 포인트별 발췌
> 체크박스에 체크를 하고 스크롤을 내렸다가 다시 올리면 체크해놓은 아이템의 체크값이 초기화 되는 문제점이 있었다.
> 이를 어떻게 해결할까 구글링 해본결과 역시 해결책을 찾을 수 있었다.
> holder.cbSelect.setChecked(objIncome.isSelected());
> holder.cbSelect.setOnCheckedChangeListener(null);
> 기존 의 setChecked와 리스너를 사용하게되면 정상 작동함을 알 수 있다.
