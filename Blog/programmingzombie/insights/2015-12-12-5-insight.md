## 핵심 주장 (≤5)
1. `java.util.Timer`와 `java.util.TimerTask`를 사용하면 특정 시간 또는 일정 주기로 반복되는 작업을 손쉽게 구현할 수 있다.
2. `schedule()`은 fixed-delay 방식이라 선행 작업이 지연되면 다음 작업 시작도 그만큼 지연되므로, 정확한 실행 시각이 중요하지 않은 작업에 적합하다.
3. 정확한 시간 간격으로 작업을 실행해야 할 때는 fixed-rate 방식의 `scheduleAtFixedRate()`를 사용하는 것이 적합하다.
4. `cancel()`은 Timer를 중지하고 예정된 작업을 취소하지만, 이미 실행 중인 `TimerTask`에는 영향을 주지 않는다.
5. 기본 생성자로 만든 `Timer`는 데몬 스레드가 아니므로 `cancel()` 없이 두면 애플리케이션이 종료되지 않을 수 있고, `Timer(true)`를 사용하면 데몬 스레드로 실행된다.

## 공개된 숫자·지표
- JDK 버전: 1.3 (기간: 기능 추가 시점, 출처: "JDK1.3에 새롭게 추가된 java.util.Timer 클래스를 사용하여 이벤트의 실행을 제어하는 것에 대해서 알아본다.")
- `schedule()` 형태 수: 4가지 (기간: 해당 API 설명, 출처: "위에서 schedule() 메소드는 특정한 시간에 원하는 작업을 수행하고자 할 때 사용하는 메소드로 다음 표와 같이 4가지 형태가 존재한다.")
- 예제 시작 지연: 1000ms (기간: 예제 실행 시점, 출처: "jobScheduler.scheduleAtFixedRate(job, 1000, 3000);")
- 예제 반복 간격: 3000ms (기간: 예제 실행 시점, 출처: "jobScheduler.scheduleAtFixedRate(job, 1000, 3000);")
- 메인 스레드 대기 시간: 20000ms (기간: 예제 실행 시점, 출처: "Thread.sleep(20000);")

## 언급된 도구·서비스
- `java.util.Timer`: 백그라운드에서 특정 시각 또는 일정 주기로 작업을 실행하는 스케줄링 클래스
- `java.util.TimerTask`: `Timer`가 수행할 작업을 나타내는 클래스
- `cron`: 유닉스/리눅스에서 특정 시간에 프로세스를 실행하는 명령어의 예시
- `at`: 유닉스/리눅스에서 특정 시간에 프로세스를 실행하는 명령어의 예시

## 복제 가능한 전술 (≤3)
1. 반복 실행 작업은 `TimerTask`로 분리하고 `scheduleAtFixedRate()`로 주기 실행한다.
   - 구체적 스텝: `TimerTask`를 상속한 작업 클래스를 만들고 `run()`에 실행 로직을 넣은 뒤 `Timer` 인스턴스를 생성해 `scheduleAtFixedRate(task, delay, period)`로 등록한다.
   - 예상 리소스: Java 개발 환경, `java.util.Timer`/`TimerTask`
   - 예상 효과: 원문 예제처럼 일정 시간 후 시작해 고정 간격으로 작업을 반복 실행할 수 있다.
2. 실행 시각의 정확도가 덜 중요한 배치성 작업은 `schedule()`로 구현한다.
   - 구체적 스텝: 일회성 또는 반복 작업에 맞는 `schedule()` 오버로드를 선택하고, 선행 작업 지연 시 다음 실행도 밀릴 수 있음을 감안해 삭제·백업 같은 작업에 적용한다.
   - 예상 리소스: Java 개발 환경, 작업 로직이 담긴 `TimerTask`
   - 예상 효과: 스레드를 직접 구현하지 않고도 특정 시각 또는 일정 지연 후 작업을 실행할 수 있다.
3. 애플리케이션 종료를 막지 않아야 하는 보조 스케줄러는 `Timer(true)`로 생성한다.
   - 구체적 스텝: `new Timer(true)`로 데몬 타이머를 만들고 작업을 등록한 뒤, 메인 로직 종료 후 타이머 스레드가 프로세스를 붙잡지 않도록 구성한다.
   - 예상 리소스: Java 개발 환경, 데몬 스레드 동작 이해
   - 예상 효과: 원문 설명처럼 `cancel()`을 호출하지 않아도 메인 실행이 끝나면 애플리케이션이 종료된다.

## 원문 요약 (≤5문장)
이 글은 JDK 1.3에서 추가된 `java.util.Timer`와 `java.util.TimerTask`를 이용해 자바에서 작업 스케줄링을 구현하는 방법을 설명한다. `schedule()`과 `scheduleAtFixedRate()`의 차이를 fixed-delay와 fixed-rate 관점에서 비교하며, 실행 정확도에 따라 적절한 메소드를 선택해야 한다고 말한다. 예제로는 1초 후 시작해 3초마다 현재 시간을 출력하는 프로그램을 제시한다. 또한 기본 `Timer`는 데몬 스레드가 아니어서 `cancel()`이 필요할 수 있으며, `Timer(true)`를 사용하면 데몬 스레드로 실행된다고 설명한다.

## 본문 포인트별 발췌
> JDK1.3에 새롭게 추가된 java.util.Timer 클래스를 사용하여 이벤트의 실행을 제어하는 것에 대해서 알아본다.
> java.util.Timer 클래스는 백그라운드에서 특정한 시간 또는 일정 시간을 주기로 반복적으로 특정 작업을 실행할 수 있도록 해 준다.
> 따라서, 정확하게 특정 시간 마다 실행해야 하는 작업의 경우에는 schedule() 메소드가 아닌 scheduleAtFixedRate() 메소드를 사용하는 것이 좋다.
> cancel() 메소드는 Timer를 중지시키며, 실행될 예정인 모든 작업을 취소한다.
> 이 때, boolean 값이 true이면 Timer 클래스와 관련된 쓰레드들은 데몬 쓰레드로 설정된다.
