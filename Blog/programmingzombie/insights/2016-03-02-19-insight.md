## 핵심 주장 (≤5)
1. RelativeLayout은 자식 뷰들을 서로의 상대적 관계에 따라 배치하는 레이아웃이다.
2. RelativeLayout의 핵심은 부모 기준 정렬 속성과 다른 뷰 ID를 기준으로 한 상대 배치 속성을 조합해 화면을 구성하는 데 있다.
3. 예제에서는 TextView, EditText, Button 간의 상대 위치 지정만으로 입력 폼 레이아웃을 만들 수 있음을 보여준다.

## 복제 가능한 전술 (≤3)
1. 기준 뷰 ID를 중심으로 입력 폼을 상대 배치로 구성한다.
   - 구체적 스텝: 라벨 `TextView`에 ID를 부여하고, 입력창 `EditText`에 `android:layout_toRightOf`와 `android:layout_alignBaseline`을 적용해 라벨 오른쪽에 정렬한다.
   - 예상 리소스: Android XML 레이아웃 파일, 기본 위젯 2종(TextView, EditText), 개발 시간 소량
   - 예상 효과: 라벨과 입력창의 수평 관계를 명시적으로 고정해 단순한 입력 UI를 빠르게 구성할 수 있다.
2. 액션 버튼을 입력창 하단 기준으로 정렬한다.
   - 구체적 스텝: `OK` 버튼에 `android:layout_below`와 `android:layout_alignRight`를 적용해 입력창 아래와 오른쪽 가장자리에 붙이고, `Cancel` 버튼은 `android:layout_toLeftOf`와 `android:layout_alignTop`으로 `OK` 버튼 왼쪽 상단에 맞춘다.
   - 예상 리소스: Android XML 레이아웃 파일, Button 2개, 기준 뷰 ID
   - 예상 효과: 버튼 간 상대 위치를 명시해 입력 폼 하단 액션 영역을 일관되게 배치할 수 있다.
3. 부모 기준 속성과 형제 기준 속성을 구분해 레이아웃 규칙을 설계한다.
   - 구체적 스텝: 부모 가장자리나 중앙 배치는 `android:layout_alignParentTop`, `android:layout_centerHorizontal` 같은 부모 기준 속성으로 잡고, 뷰 간 관계는 `android:layout_below`, `android:layout_toRightOf` 같은 형제 기준 속성으로 나눈다.
   - 예상 리소스: RelativeLayout 속성 목록 참조, Android XML 편집기
   - 예상 효과: 어떤 속성이 부모 기준인지 대상 뷰 기준인지 혼동을 줄여 레이아웃 작성과 수정이 쉬워진다.

## 원문 요약 (≤5문장)
이 글은 LinearLayout 설명에 이어 RelativeLayout의 개념과 주요 속성을 정리한다. RelativeLayout은 자식 뷰를 부모나 다른 자식 뷰와의 상대적 관계로 배치하는 레이아웃이라고 설명한다. 본문은 부모 기준 정렬, 중앙 배치, 특정 뷰의 위·아래·좌·우 배치 등 주요 XML 속성을 표 형태로 나열한다. 이어서 URL 입력 폼 예제로 TextView, EditText, OK/Cancel 버튼을 상대 배치하는 샘플 코드를 제시한다. 마지막으로 각 뷰가 어떤 기준 뷰에 맞춰 배치되는지 간단히 설명한다.

## 본문 포인트별 발췌
> 앞에서  LinearLayout 설명에 이어서 이번에는 RelativeLayout에 대해서 설명해 보겠다.
> 의미상 그대로 자식 뷰들을 상대적인 관계에 따라서 배치가 되는 Layout이다.
> 예를 들어 자식 A는 자식 B의 오른쪽, 자식 C는 자식 B의 왼쪽 등등으로 상대적인 위치를 설정해서 화면을 구성하는 방식이다.
> - EditText를 TextView의 오른쪽에 위치시키고, 아래 라인은 TextView에 맞춘다. 
> - Cancel Button을OK Button의 왼쪽에 위치시키고, 상단 라인은 OK Button에 맞춘다.
