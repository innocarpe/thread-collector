## 핵심 주장 (≤5)
1. 안드로이드 개인 앱개발에서 파싱한 데이터를 SQLite에 저장할 필요가 있었고, 참고 자료를 바탕으로 저장이 잘 되는 것을 확인했다.
2. `SQLiteOpenHelper`를 싱글턴 패턴으로 구현하면 SQLite 객체를 한 번만 생성하고 열어서 사용할 수 있다.
3. 다른 액티비티에서 `Insert`, `delete`, `select`가 가능하도록 메서드를 구성할 수 있다.
4. 글쓴이의 구현에서는 필요한 기능이 아니라서 `Update`는 제외했다.
5. DB를 사용할 때마다 여는 구조라면 메모리 누수를 막기 위해 사용 후 닫아줘야 한다.

## 공개된 숫자·지표
- DB 버전: 1 (기간: 없음, 출처: `public static final int DB_VERSION = 1;`)
- 컬럼 수: 12개 데이터 컬럼 + 1개 기본키 컬럼 (기간: 없음, 출처: `IDX`와 `COL_0`부터 `COL_11`까지 정의한 코드)

## 언급된 도구·서비스
- Android SQLite: 파싱한 데이터를 앱 내부 DB에 저장하고 조회·삭제하는 용도로 사용했다.
- SQLiteOpenHelper: DB 생성, 업그레이드, CRUD 메서드 구성을 위해 사용했다.
- Cursor: 조회 결과 존재 여부 확인과 레코드 읽기에 사용했다.
- ContentValues: `insertData`에서 객체 필드를 DB 컬럼에 매핑해 주입하는 데 사용했다.

## 언급된 다른 creator·앱

## 복제 가능한 전술 (≤3)
1. 앱 전역에서 재사용할 SQLite 헬퍼를 싱글턴 `SQLiteOpenHelper`로 감싼다.
   - 구체적 스텝: DB 이름, 테이블명, 컬럼 상수를 정의한 뒤 `getInstance(Context)`로 단일 인스턴스를 반환하게 만들고, 생성자에서 `getWritableDatabase()`를 호출해 연결을 연다.
   - 예상 리소스: 안드로이드 스튜디오, 자바 코드 작성 시간, `SQLiteOpenHelper`
   - 예상 효과: SQLite 객체를 한 번만 생성·오픈하는 구조로 여러 액티비티에서 같은 DB 헬퍼를 재사용할 수 있다.
2. 도메인 객체를 `ContentValues`로 매핑해 삽입하고, 삽입 성공 여부를 반환한다.
   - 구체적 스텝: `NumberData` 같은 모델 객체의 필드를 `contentValues.put()`으로 컬럼별 매핑하고 `db.insert()` 결과가 `-1`인지 검사해 `boolean`으로 반환한다.
   - 예상 리소스: 모델 클래스 1개, `ContentValues`, insert 메서드 구현 시간
   - 예상 효과: 파싱한 데이터를 구조적으로 SQLite에 저장하고 저장 성공/실패를 호출부에서 바로 처리할 수 있다.
3. 조회 목적별로 존재 확인, 단건 조회, 전체 조회, 최신값 조회 메서드를 분리한다.
   - 구체적 스텝: `rawQuery`나 `query`를 사용해 존재 여부는 `moveToFirst()`로 판단하고, 단건 조회는 모델 객체로 역매핑하며, 전체 조회는 리스트에 누적하고, 최신 회차는 `DESC` 정렬과 `LIMIT 1`로 가져온다.
   - 예상 리소스: `Cursor`, 리스트 자료구조, SQL 문자열 작성 시간
   - 예상 효과: 다른 액티비티에서 `select`, 전체 목록 조회, 최신 데이터 확인을 목적별 메서드로 바로 호출할 수 있다.

## 원문 요약 (≤5문장)
글쓴이는 안드로이드 개인 앱에서 파싱한 데이터를 저장하기 위해 SQLite를 적용했다. `SQLiteOpenHelper`를 상속한 헬퍼 클래스를 싱글턴으로 구현하고, 테이블 생성과 삽입·존재 확인·단건 조회·전체 조회·삭제 메서드를 만들었다. 이 구조를 통해 다른 액티비티에서도 `Insert`, `delete`, `select`를 사용할 수 있다고 설명한다. 참고한 외부 예제는 싱글턴이 아니고 `close()`를 자주 호출하지만, 글쓴이는 싱글턴 구조에서는 DB를 한 번만 열어 재사용하므로 굳이 닫지 않아도 메모리 누수가 발생하지 않는다고 본다. 반대로 사용할 때마다 DB를 여는 구조라면 사용 후 닫아야 한다고 덧붙인다.

## 본문 포인트별 발췌
> 안드로이드 개인 앱개발을 진행하면서, 파싱한 데이터를 SQLite에 저장할 필요가 생기게 되었는데
> 해당 내용을 참조하여 데이터가 잘 들어간 것을 확인하였다.
> public static SQLiteHelper getInstance(Context context){ // 싱글턴 패턴으로 구현하였다.
> 해당 메서드를 구성하고 다른 액티비티에서 Insert, delete, select가 가능하다 Update의 경우는 구현하고자 하는 내용에 필요가 없어서
> 만일 DB를 사용할때마다 DB를 열도록 구현했다면 메모리 누수가 발생하지 않도록 사용후 닫아 줘야 한다.
