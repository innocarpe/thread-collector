## 핵심 주장 (≤5)
1. `@Transactional` 기반 선언적 트랜잭션을 쓰려면 `transactionManager` 빈을 등록하고 `<tx:annotation-driven transaction-manager="transactionManager" />` 설정이 필요하다.
2. `@Transactional`은 `isolation`, `propagation`, `readOnly`, `rollbackFor`, `noRollbackFor`, `timeout` 같은 속성으로 트랜잭션 동작을 세밀하게 제어할 수 있다.
3. 트랜잭션 격리 수준은 다른 트랜잭션의 간섭을 얼마나 허용할지 정하는 기준이며, 별도 지정이 없으면 DB의 기본 격리 수준을 따른다.
4. 전파 속성은 기존 트랜잭션이 있을 때 참여할지, 새로 만들지, 예외를 낼지 같은 실행 방식을 결정한다.
5. 트랜잭션은 여러 처리 과정을 하나의 묶음으로 보장하고, 중간 오류가 나면 전체를 이전 상태로 되돌리는 개념이다.

## 언급된 도구·서비스
- Spring `@Transactional`: 선언적 트랜잭션을 적용하고 속성으로 동작을 제어하는 데 사용됨
- `DataSourceTransactionManager`: `transactionManager` 빈으로 등록해 트랜잭션 매니저로 사용됨
- `<tx:annotation-driven>`: 애노테이션 기반 트랜잭션 처리를 활성화하는 설정으로 사용됨
- `mvc-dispatcher-servlet.xml`: `component-scan`으로 패키지를 명시해야 하는 설정 파일로 언급됨
- 데이터베이스(DB): 기본 격리 수준 제공 주체로 언급됨

## 복제 가능한 전술 (≤3)
1. Spring 프로젝트에 선언적 트랜잭션을 먼저 활성화한다.
   - 구체적 스텝: `DataSourceTransactionManager` 빈을 등록하고 `<tx:annotation-driven transaction-manager="transactionManager" />`를 설정한 뒤, 트랜잭션이 필요한 서비스 메서드에 `@Transactional`을 부여한다.
   - 예상 리소스: Spring XML 설정 파일, 데이터소스 연결 정보, 서비스 코드 수정 시간
   - 예상 효과: 애노테이션 기반으로 트랜잭션 경계를 선언해 일련의 처리를 하나의 묶음으로 관리할 수 있다.
2. 메서드 성격에 맞춰 `@Transactional` 속성을 명시한다.
   - 구체적 스텝: 읽기 전용 조회 메서드에는 `readOnly=true`, 특정 예외에서 롤백이 필요하면 `rollbackFor`, 롤백 제외가 필요하면 `noRollbackFor`, 수행 시간 제약이 필요하면 `timeout`을 지정한다.
   - 예상 리소스: 예외 정책 검토, 서비스 메서드별 속성 정의
   - 예상 효과: 메서드별로 읽기 전용 처리, 롤백 정책, 시간 제한을 분리해 트랜잭션 동작을 명확히 제어할 수 있다.
3. 데이터 충돌 가능성에 따라 격리 수준과 전파 정책을 선택한다.
   - 구체적 스텝: DB 기본값을 따를지 검토하고, 필요 시 `isolation`을 지정한다. 기존 트랜잭션 참여가 필요한지, 항상 새 트랜잭션이 필요한지, 트랜잭션 없이는 실행되면 안 되는지를 기준으로 `propagation` 값을 정한다.
   - 예상 리소스: 트랜잭션 흐름 분석, 서비스 호출 관계 검토
   - 예상 효과: 다른 트랜잭션 간 간섭 수준과 메서드 호출 시 트랜잭션 참여 방식을 의도대로 맞출 수 있다.

## 원문 요약 (≤5문장)
글은 Spring의 `@Transactional`을 이용한 선언적 트랜잭션 설정과 주요 속성을 정리한 기술 메모다. 먼저 `transactionManager` 빈과 `<tx:annotation-driven>` 설정 예시를 제시하고, `isolation`, `propagation`, `readOnly`, `rollbackFor`, `noRollbackFor`, `timeout` 속성을 설명한다. 이어서 격리 수준과 전파 옵션별 의미를 표 형태로 요약한다. 후반부에서는 ATM 출금 예시로 트랜잭션, 커밋, 롤백 개념을 직관적으로 설명한다. 전체적으로 실무 설정 참고용 요약과 개념 정리에 초점이 있다.
