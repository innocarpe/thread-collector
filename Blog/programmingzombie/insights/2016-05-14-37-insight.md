## 핵심 주장 (≤5)
1. SourceTree가 갑자기 실행되지 않을 때 일반적인 재설치·관리자 권한 실행·CMD 실행·설치 폴더 삭제만으로는 해결되지 않을 수 있다.
2. `C:\Users\name\AppData\Local\Atlassian\SourceTree.exe_Url_xx..xx` 경로를 삭제하니 이후에는 SourceTree가 실행됐다.
3. 이 방법은 SourceTree의 초기 설정값과 Local 저장소 데이터가 날아갈 수 있는 단점이 있다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- SourceTree: 갑자기 실행되지 않는 문제를 겪은 Git GUI 도구로 언급됨
- CMD: SourceTree를 실행해보는 대안 경로로 언급됨
- 프로그램 추가제거: SourceTree를 삭제 후 재설치하는 방법으로 언급됨

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. SourceTree가 실행되지 않을 때 사용자 로컬 AppData 아래의 SourceTree 실행 관련 캐시/설정 폴더를 삭제해 복구를 시도한다.
   - 구체적 스텝: SourceTree 실행 불가 상태에서 먼저 재설치, 관리자 권한 실행, CMD 실행, `C:\Program Files (x86)\Atlassian` 폴더 삭제를 시도한 뒤에도 해결되지 않으면 `C:\Users\name\AppData\Local\Atlassian\SourceTree.exe_Url_xx..xx` 경로를 삭제하고 다시 실행한다.
   - 예상 리소스: Windows PC, SourceTree, 파일 탐색기, 수분 내 처리 가능
   - 예상 효과: 작성자 사례에서는 해당 경로 삭제 이후 SourceTree가 다시 실행됐다.
2. SourceTree 복구 작업 전에 로컬 설정 및 저장소 데이터 손실 가능성을 감안해 삭제 대상을 신중히 정한다.
   - 구체적 스텝: `AppData\Local\Atlassian` 아래 경로 삭제 전 SourceTree 초기 설정값과 Local 저장소 데이터가 초기화될 수 있음을 확인하고 필요한 데이터는 별도로 보존한 뒤 작업한다.
   - 예상 리소스: Windows PC, 파일 확인 시간
   - 예상 효과: 원문에서 언급한 설정값·Local 저장소 데이터 손실 위험을 줄일 수 있다.

## 원문 요약 (≤5문장)
작성자는 어느 날 SourceTree가 실행되지 않는 문제를 겪었다. 재설치, 관리자 권한 실행, CMD 실행, `C:\Program Files (x86)\Atlassian` 폴더 삭제를 시도했지만 해결되지 않았다고 적었다. 이후 `C:\Users\name\AppData\Local\Atlassian\SourceTree.exe_Url_xx..xx` 경로를 삭제하자 실행이 됐다고 정리한다. 다만 이 방법은 SourceTree의 초기 설정값과 Local 저장소 데이터가 사라질 수 있다고 덧붙였다.

## 본문 포인트별 발췌
> 어느날 소스트리를 켰는데 실행이 안되는 것이었다.
> 1. 프로그램 추가제거에서 삭제 후 재설치
> **그래도 결론은 실행이 안된다!**
> 혹시나 하는 마음에 아래의 경로를 삭제해보니 이후로는 실행이 된다.
> 다만 단점이라면 소스트리의 초기 설정값들과 Local 저장소데이터가 다 날아간다는점이 있을 수 있겠다.
