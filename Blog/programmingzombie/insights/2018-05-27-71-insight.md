## 핵심 주장 (≤5)
1. 코틀린의 `forEach`에서 `return@forEach`는 루프 전체를 빠져나가지 않고 `continue`처럼 동작한다.
2. `forEach`는 인덱스를 기본 제공하지 않기 때문에 인덱스 기반 조건이 필요하면 별도 변수를 두거나 `forEachIndexed`를 써야 한다.
3. `forEachIndexed` 안에서 조건만 추가하는 방식은 원하는 결과를 만들 수 있어도 전체 아이템을 끝까지 순회하는 비효율이 생길 수 있다.
4. `run`에 라벨을 붙이고 `return@loop`를 사용하면 `forEach`/`forEachIndexed`에서도 원하는 시점에 순회를 중단할 수 있다.
5. 루프 탈출이 필요한 상황에서는 `for`, `forEach`, `forEachIndexed`의 동작 차이를 정확히 이해하고 선택해야 한다.

## 공개된 숫자·지표
- 예시 리스트에서 보여줄 항목 수: 3번째 데이터까지 (기간: 예시 코드 설명, 출처: "리스트 중 3번째 데이터 까지만 보여주고 나머지는 생략을 하고싶은 경우는 어떻게 할까?")
- 예시 대용량 항목 수: 10만개 (기간: 성능 문제 예시, 출처: "지금처럼 아이템 갯수가 적으면 상관없지만 10만개라면?")
- 검증된 문자열 위치: 14 (기간: 테스트 실행 결과 검증, 출처: "Assert.assertThat(result.lastIndexOf(\" .. 생략\"), CoreMatchers.`is`(14))")

## 언급된 도구·서비스
- Kotlin 공식 홈페이지: `return`과 `forEach`의 동작 및 우회 방법을 확인하는 참고 자료로 사용됨
- JUnit `@Test`/`@Before`: 예제 코드를 테스트 메서드와 초기화 메서드로 실행하는 데 사용됨

## 복제 가능한 전술 (≤3)
1. `forEach`에서 조기 종료가 필요하면 외부 `run` 블록에 라벨을 붙이고 `return@loop`로 빠져나간다.
   - 구체적 스텝: `run loop@ { ... }` 블록을 만든다 -> `forEach` 내부에서 종료 조건을 검사한다 -> 조건 만족 시 결과를 한 번 추가한 뒤 `return@loop`를 호출한다
   - 예상 리소스: 코틀린 코드 수정, Kotlin 공식 문서 참고
   - 예상 효과: `return@forEach`가 `continue`처럼 동작하는 문제를 피하고 원하는 시점에 전체 순회를 중단할 수 있다
2. 인덱스 조건이 필요한 순회는 `forEach` 대신 `forEachIndexed`로 바꿔 별도 인덱스 변수를 제거한다.
   - 구체적 스텝: `forEach` 호출부를 `forEachIndexed { index, item -> ... }`로 변경한다 -> 기존 수동 `index` 변수와 증가 로직을 제거한다 -> 인덱스 조건을 `index <= 2`처럼 직접 적용한다
   - 예상 리소스: 코틀린 컬렉션 순회 코드 수정
   - 예상 효과: 인덱스 관리 코드가 줄어들고 조건식이 더 직접적으로 표현된다
3. 순회 중단이 불가능한 우회 로직 대신 실제 종료 여부를 테스트로 검증한다.
   - 구체적 스텝: 결과 문자열에 생략 표시가 추가되는 지점을 정의한다 -> 순회 함수 실행 후 `result.lastIndexOf(" .. 생략")`를 검증한다 -> 조기 종료가 필요한 함수마다 같은 패턴으로 회귀 테스트를 둔다
   - 예상 리소스: 테스트 코드, `Assert.assertThat`
   - 예상 효과: 겉으로는 같은 출력이 나와도 내부적으로 불필요한 순회가 계속되는 구현을 구분해낼 수 있다

## 원문 요약 (≤5문장)
글은 리스트에서 앞의 일부 항목만 출력하고 나머지를 생략하는 예제로 `for`, `forEach`, `forEachIndexed`의 루프 탈출 차이를 비교한다. `for`는 `break`로 쉽게 중단되지만, `forEach`에서 `return@forEach`는 전체 탈출이 아니라 `continue`처럼 동작해 의도와 다르게 계속 순회한다. `forEachIndexed`로 인덱스 처리는 단순해질 수 있지만, 조건만으로 우회하면 전체 아이템을 계속 검사하는 비효율이 남는다. 작성자는 Kotlin 공식 문서를 참고해 `run loop@ { ... }`와 `return@loop`를 사용하면 `forEach`/`forEachIndexed`에서도 원하는 시점에 순회를 멈출 수 있음을 확인했다. 결론적으로 각 루프의 제어 흐름 차이를 이해하고 목적에 맞는 방식을 선택해야 한다는 학습 내용을 정리한다.

## 본문 포인트별 발췌
> break 직접적으로 할 순 없지만 방법이 있었다. 그리고 return 을 했을때, continue처럼 동작하는거라고 설명되어있다.
> 마치 **continue**처럼 동작하는 것을 알 수 있다.
> 따라서 다른 방법이 필요했다.
> 결과는 만족스럽게 잘 동작한다. :)
> for, forEach, forEachIndexed를 좀 더 제대로 알고 사용할 수 있게 된 것 같다. :)
