## 핵심 주장 (≤5)
1. Handlebars는 "semantic templates"를 효과적으로 만들 수 있게 해주는 템플릿 도구다.
2. Handlebars는 Mustache 템플릿과 대체로 호환되어 기존 템플릿을 유지한 채 교체할 수 있는 경우가 많다.
3. 브라우저에서 템플릿을 컴파일하는 대신 사전 컴파일하면 런타임 라이브러리 크기를 줄이고 컴파일 비용을 아낄 수 있다.
4. 기본 `{{expression}}`은 HTML escape를 수행하며, escape하지 않으려면 `{{{` 또는 `Handlebars.SafeString`을 사용해야 한다.
5. Handlebars는 block helper, nested path, helper, partial, built-in helper를 통해 재사용성과 표현력을 확장한다.

## 언급된 도구·서비스
- Handlebars: HTML 안에 표현식을 넣어 템플릿을 작성하고 `Handlebars.compile`로 컴파일하는 템플릿 엔진
- Mustache: Handlebars와 대체로 호환되는 템플릿 시스템으로 비교 대상으로 언급됨
- 브라우저: `<script>` 태그로 템플릿을 전달하고 브라우저 내 컴파일이 일어나는 실행 환경으로 언급됨
- 모바일 디바이스: 사전 컴파일의 이점이 특히 중요할 수 있는 환경으로 언급됨
- jQuery: `$("#entry-template").html();` 예제에서 템플릿 소스를 읽는 데 사용됨

## 복제 가능한 전술 (≤3)
1. 브라우저 템플릿 렌더링 흐름을 Handlebars로 표준화한다.
   - 구체적 스텝: HTML 템플릿을 `<script type="text/x-handlebars-template">`에 넣고, JavaScript에서 `Handlebars.compile`로 컴파일한 뒤, context 객체를 전달해 최종 HTML을 생성한다.
   - 예상 리소스: Handlebars, 브라우저 환경, 템플릿 마크업, context 데이터
   - 예상 효과: HTML 안에 데이터를 주입하는 템플릿 렌더링을 분리해 의미 중심 템플릿을 구성할 수 있다.
2. 모바일 또는 성능 민감 환경에서는 템플릿을 사전 컴파일한다.
   - 구체적 스텝: 브라우저에서 매번 `compile`하지 않고 템플릿을 미리 precompile해서 배포하고, 더 작은 runtime library로 실행되게 구성한다.
   - 예상 리소스: Handlebars precompilation 도구, 빌드 과정
   - 예상 효과: 브라우저 내 템플릿 컴파일이 줄어들고 required runtime library가 더 작아진다.
3. 반복 UI와 HTML 생성 규칙은 helper와 partial로 분리한다.
   - 구체적 스텝: 리스트 렌더링은 `registerHelper`로 block helper를 등록하고, 공통 출력 조각은 `registerPartial`로 등록하며, HTML 문자열을 반환할 때는 필요한 경우 `Handlebars.SafeString`을 사용한다.
   - 예상 리소스: Handlebars helper/partial API, 재사용할 템플릿 조각
   - 예상 효과: 템플릿 재사용이 쉬워지고, 공통 마크업과 로직을 중앙에서 관리할 수 있다.

## 원문 요약 (≤5문장)
이 글은 Handlebars의 공식 소개와 주요 사용법을 예제 중심으로 정리한 글이다. 기본 템플릿 작성 방식, 브라우저 전달, `Handlebars.compile`을 통한 컴파일, context 실행 흐름을 설명한다. 또한 HTML escaping, block expression, nested path, comments, helper, partial, built-in helper 같은 핵심 기능을 순서대로 소개한다. 특히 precompilation은 더 작은 runtime library와 브라우저 컴파일 절감 효과를 준다고 강조한다.

## 본문 포인트별 발췌
> Handlebars provides the power necessary to let you build semantic templates effectively with no frustration.
> Handlebars is largely compatible with Mustache templates.
> It is also possible to precompile your templates.
> This will result in a smaller required runtime library and significant savings from not having to compile the template in the browser.
> Handlebars HTML-escapes values returned by a `{{expression}}`.
