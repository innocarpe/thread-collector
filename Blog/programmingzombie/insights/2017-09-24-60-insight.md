## 핵심 주장 (≤5)
1. 커스텀 DialogFragment에서 원치 않는 액션바 타이틀이 표시되는 문제가 있었고, 이는 롤리팝(5.0) 이하에서 확인되었다.
2. 스타일에서 `windowNoTitle` 값을 주거나 `actionBar.hide()`를 호출하는 방식은 이 문제를 해결하지 못했다.
3. DialogFragment에는 ActionBar를 붙일 수 없고 `Dialog.getActionBar()`는 항상 `null`을 반환한다고 정리했다.
4. `onCreateDialog()`에서 `dialog.window.requestFeature(Window.FEATURE_NO_TITLE)`를 호출하면 타이틀바를 제거할 수 있다.

## 복제 가능한 전술 (≤3)
1. DialogFragment의 기본 타이틀바가 남는 구버전 이슈를 `onCreateDialog()`에서 직접 제거한다.
   - 구체적 스텝: `DialogFragment`를 상속한 클래스에 `onCreateDialog(savedInstanceState: Bundle?)`를 오버라이드하고, `super.onCreateDialog(savedInstanceState)`로 받은 `Dialog`의 `window.requestFeature(Window.FEATURE_NO_TITLE)`를 호출한 뒤 반환한다.
   - 예상 리소스: Android 프로젝트, `DialogFragment` 수정 1회, Kotlin 또는 Java 코드 편집
   - 예상 효과: 롤리팝 5.0 이하에서 표시되던 빈 타이틀바가 더이상 보이지 않는다.
2. UI 테마 설정이나 `actionBar.hide()` 호출이 실패할 때는 Dialog 생성 지점에서 해결하도록 접근을 바꾼다.
   - 구체적 스텝: `windowNoTitle` 스타일 적용과 `newFragment.activity.actionBar.hide()` 같은 우회 시도를 중단하고, 다이얼로그가 생성되는 `onCreateDialog()`에서 창 feature를 지정한다.
   - 예상 리소스: 기존 시도 제거 또는 유지 비교 테스트, Android 에뮬레이터/실기기 확인
   - 예상 효과: 원문에서 실패한 우회 방법 대신 실제로 동작하는 해결 경로로 바로 전환할 수 있다.

## 원문 요약 (≤5문장)
작성자는 커스텀 DialogFragment를 띄울 때 원하지 않는 액션바 타이틀이 계속 표시되는 문제를 겪었다. 이 현상은 마시멜로우와 누가에서는 보이지 않았지만 롤리팝 5.0 이하에서는 확인되었다. 스타일의 `windowNoTitle` 적용이나 `actionBar.hide()` 호출은 효과가 없었다. 대신 `onCreateDialog()`에서 `dialog.window.requestFeature(Window.FEATURE_NO_TITLE)`를 호출하는 방식으로 타이틀바를 제거했다.

## 본문 포인트별 발췌
> 커스텀 DialogFragment를 만들고 간단한 Alert 창을 띄울 일이 있었는데 내가 원하지 않는 액션바 타이틀이 계속 나오게되었다.
> 상위 버전에서는 (마시멜로우, 누가 버전)출력되지 않는 것을 확인 했으나 롤리팝(5.0) 버전 이하에서는 해당 증상이 확인되었다.
> DialogFragment에 ActionBar 등록 자체가 되지 않고, Dialog.getActionBar()호출하면 항상 리턴값이 null을 가진다.
> dialog.window.requestFeature(Window.FEATURE_NO_TITLE)
> 파라메터 값으로 Window.FEATRURE_NO_TITLE 을 넣었으므로 타이틀이 없는 창으로 확장되었으므로, **타이틀바가 더이상 보이지 않는다.**
