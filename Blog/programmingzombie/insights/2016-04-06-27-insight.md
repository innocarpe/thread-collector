## 핵심 주장 (≤5)
1. 요일 문자열을 받아 현재 날짜 기준으로 과거를 제외한 해당 요일의 날짜를 구할 수 있다.
2. `Date.compareTo`는 milliseconds 단위까지 비교해 같은 날짜도 다르게 판정되는 문제가 생길 수 있다.
3. 시분초가 아니라 날짜만 비교할 목적이라면 `yyyy-MM-dd` 형식으로 포맷한 문자열의 `compareTo`를 사용하는 방식이 적합하다.
4. 비교 결과 입력된 날짜가 오늘보다 과거이면 7일을 더해 다음 주 같은 요일 날짜로 보정할 수 있다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- Java `SimpleDateFormat`: 날짜를 `yyyy-MM-dd` 문자열로 변환해 날짜 비교에 사용됨
- Java `Calendar`: 현재 날짜를 가져오고 `DAY_OF_WEEK` 설정 및 7일 가산 처리에 사용됨
- Java `Date`: 요일 계산 결과와 현재 날짜 비교 대상 타입으로 사용됨
- `compareTo`: `Date`와 `String` 각각의 비교 방식 차이를 설명하는 데 사용됨

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. 요일 입력을 다음 도래 날짜로 변환하는 헬퍼 메서드를 만든다.
   - 구체적 스텝: `sun`~`sat` 문자열 배열을 두고 입력값과 매칭해 `Calendar.DAY_OF_WEEK` 숫자를 구한 뒤, `Calendar`에 설정해 해당 요일의 `Date`를 만든다.
   - 예상 리소스: Java 개발 환경, 날짜 유틸 메서드 작성 시간
   - 예상 효과: 원문처럼 현재 날짜 기준으로 월·수·금 등 특정 요일들의 실제 날짜 문자열을 얻을 수 있다.
2. 날짜 비교는 문자열 포맷으로 정규화한 뒤 수행한다.
   - 구체적 스텝: 비교할 두 `Date`를 모두 `yyyy-MM-dd`로 포맷하고, 문자열 `compareTo` 결과가 음수면 과거 날짜로 판정한다.
   - 예상 리소스: `SimpleDateFormat`, 비교 로직 수정 시간
   - 예상 효과: 원문에서 겪은 milliseconds 단위 오차 없이 날짜만 기준으로 비교할 수 있다.
3. 과거로 판정된 요일 날짜는 7일 뒤로 이동시킨다.
   - 구체적 스텝: 문자열 비교 결과 과거이면 `Calendar`에 대상 날짜를 세팅하고 `DATE`에 `+7`을 더한 뒤 반환한다.
   - 예상 리소스: `Calendar.add` 사용 코드
   - 예상 효과: 현재 주에서 이미 지난 요일이 입력되어도 다음 주 동일 요일 날짜를 일관되게 반환할 수 있다.

## 원문 요약 (≤5문장)
작성자는 요일 문자열을 입력받아 현재 날짜 기준으로 지나간 날짜를 제외한 해당 요일의 날짜를 구하는 Java 코드를 정리했다. 예시로 2016-04-06 수요일에 월, 수, 금을 입력받으면 2016-04-06, 2016-04-08, 2016-04-11을 얻는 방식이다. 처음에는 `Date.compareTo`를 사용했지만 milliseconds 단위 비교 때문에 같은 날짜도 다르게 판정되는 문제가 있었다고 설명한다. 그래서 날짜를 `yyyy-MM-dd` 문자열로 변환한 뒤 `String.compareTo`로 비교하고, 과거면 7일을 더하는 방식으로 해결했다.

## 본문 포인트별 발췌
> 현재 날짜를 기준으로 과거의 날짜를 제외한, 해당 요일의 날짜를 알아야 하는 상황이 있었다.
> 위와 같이 2016-04-06, 2016-04-08, 2016-04-11 의 데이터를 얻어오는 것이 목적이었다.
> 날짜를 비교하는데 Date 클래스에서 제공하는 compareTo와, String 클래스에서 제공하는 compareTo 두 가지가 있어 무엇을 사용해 해결할지 고민하다가 String에 compareTo를 사용하기로 결정하였다.
> 이렇게 했을때 compareTo에서는 비교를 milliseconds 단위로 비교를 하기 때문에 Calendar 객체의 getTime()을 호출했을때,
> 따라서 굳이 시분초단위 까지 비교하진 않고 단순 날짜만 비교하기에는 String compareTo가 적합하다고 판단되었고 아래와 같이 수정하여 해결하였다.
