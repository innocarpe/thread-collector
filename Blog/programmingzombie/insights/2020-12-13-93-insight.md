## 핵심 주장 (≤5)
1. 하나의 루트 프로젝트 안에 여러 학습 주제를 모듈로 추가하는 구조가 새 프로젝트를 계속 만드는 것보다 관리하기 좋다.
2. Intellij에서는 루트 프로젝트에서 `new > module`로 주제별 모듈을 추가할 수 있다.
3. 하위 모듈의 기본 Gradle 파일 내용은 루트 Gradle 설정과 중복되면 비우고 `dependencies {}` 정도만 남겨도 된다.
4. `settings.gradle` 또는 `settings.gradle.kts`에서 추가된 모듈명이 자동 반영되었는지와 중복·오류 여부를 확인해야 한다.
5. Project Structure에서 중복 생성되거나 불필요한 모듈, 그리고 자동 생성된 `main`, `test` 모듈을 정리하면 모듈 구성이 깔끔해진다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- Intellij: 하나의 프로젝트 아래에 여러 스터디 모듈을 생성하고 정리하는 IDE로 사용됨
- Gradle: 루트 설정을 하위 모듈에 일괄 적용하는 빌드 설정 도구로 사용됨
- Kotlin: 생성할 모듈 유형 예시이자 `settings.gradle.kts` 언급 맥락에서 사용됨
- gist: 루트 Gradle 설정 예시를 공유해둔 참고 링크로 언급됨

## 복제 가능한 전술 (≤3)
1. 학습용 작업을 주제별 모듈 구조로 통합한다.
   - 구체적 스텝: 루트 프로젝트를 하나 만든 뒤 Intellij에서 `root 프로젝트 우클릭 > new > module`로 새 모듈을 추가하고, 주제가 생길 때마다 같은 방식으로 누적한다.
   - 예상 리소스: Intellij, Gradle 기반 프로젝트, 모듈 생성 및 이름 지정에 드는 짧은 설정 시간
   - 예상 효과: 새 주제가 생길 때마다 별도 프로젝트를 만들지 않고 하나의 `study` 프로젝트 아래에서 주제별로 관리할 수 있다.
2. 하위 모듈 Gradle 파일을 최소화하고 루트 설정을 공통 적용한다.
   - 구체적 스텝: 모듈 생성 후 기본 생성된 Gradle 파일에서 루트 설정과 중복되는 내용을 제거하고 `dependencies {}`만 남긴 뒤, 공통 설정은 루트 Gradle에서 관리한다.
   - 예상 리소스: Gradle 파일 편집, 루트 Gradle 공통 설정
   - 예상 효과: 하위 모듈마다 중복 설정을 반복하지 않고 루트 설정으로 일괄 적용할 수 있다.
3. 모듈 생성 후 `settings.gradle`과 Project Structure를 함께 점검한다.
   - 구체적 스텝: `settings.gradle` 또는 `settings.gradle.kts`에서 모듈명이 올바르게 추가됐는지 확인하고, `Project Settings > Modules`에서 중복 생성된 항목이나 불필요한 `main`, `test` 모듈을 제거한 뒤 `Apply > OK`로 마무리한다.
   - 예상 리소스: Intellij Project Structure 화면, 설정 점검 시간
   - 예상 효과: 중복 경로나 불필요한 모듈을 제거해 원하는 모듈 구조로 정리할 수 있다.

## 원문 요약 (≤5문장)
작성자는 학습 주제가 늘어날 때마다 새 프로젝트를 만드는 대신 하나의 `study` 루트 프로젝트 아래에 여러 모듈을 두는 방식을 선택했다. 글은 Intellij에서 루트 프로젝트에 새 모듈을 만드는 방법과 Gradle/Kotlin 모듈 생성 흐름을 간단히 정리한다. 또한 하위 모듈의 기본 Gradle 파일은 루트 설정과 중복되므로 최소화하고 루트 Gradle로 공통 적용하는 방식을 소개한다. 이어서 `settings.gradle`의 모듈 등록 상태와 Project Structure의 중복·불필요 모듈 정리 과정을 설명한다. 마지막으로 `src > main > java or kotlin` 폴더가 자동 생성되지 않을 때 Sources 지정으로 해결하는 팁을 덧붙였다.

## 본문 포인트별 발췌
> 고민을 하다가 스터디라는 프로젝트를 만들고 _하나의 프로젝트 안에서 여러개의 주제가 생겨날때마다 추가하도록 하자_라는  
> 생각이 들었다.
> dependencies를 아무것도 받지 않고, root에 있는 gradle 설정을 통해 하위 모듈들에게 모두 일괄 적용할 수 있다.
> settings.gradle(kts를 사용하시는분은 settings.gradle.kts)를 살펴보면 자동적으로 내가 추가한 모듈네임이 추가된 것을 확인할 수 있는데, 혹시나 중복되어 추가되었거나, 잘못들어간 내용이 있는지 확인하도록 하자.
> 그리고 내가 생성하지 않은 main, test 모듈도 같이 생성된 것을 볼 수 있다. 모두 정리해주도록 하자.
> 원하는 형태로 study 하위에 algorithm 모듈이 생성되었다.
