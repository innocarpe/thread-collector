## 핵심 주장 (≤5)
1. 프로젝트 규모가 커질수록 사용하지 않는 메소드나 변수가 늘어난다.
2. 미사용 메소드나 변수를 수작업으로 찾아 지우는 데에는 적지 않은 시간과 노력이 든다.
3. IntelliJ와 Android Studio의 `Analyze > Run Inspection by Name` 기능으로 미사용 항목을 쉽게 찾을 수 있다.
4. 중간중간 리팩토링과 불필요한 코드 제거가 번거로웠다면 inspection 기능으로 프로젝트를 정리해볼 만하다.

## 공개된 숫자·지표
- 사용되지 않는 symbol 발견 건수: 42건 (기간: 개인프로젝트 검색 시점, 출처: "3. 검색하면 아래와 같이 결과가 나온다. 개인프로젝트에서 찾아진 건수는 총 42건! 아래 내용을 클릭해보자.")

## 언급된 도구·서비스
- IntelliJ: `Analyze > Run Inspection by Name` 메뉴에서 미사용 메소드·변수·심볼을 찾는 도구로 언급됨.
- Android Studio: IntelliJ와 함께 안 쓰는 메소드와 변수를 찾는 IDE로 제목과 본문에서 언급됨.
- Kotlin: 작성자가 symbol 검색 대상을 고를 때 사용한 언어로 언급됨.

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. IDE의 inspection 검색으로 미사용 코드 목록을 한 번에 추출한다.
   - 구체적 스텝: IntelliJ 또는 Android Studio에서 상단 메뉴 `Analyze > Run Inspection by Name`를 연다. 검색어에 `Unused`를 입력한다. 언어·변수·함수·파라미터·라이브러리 등 목록 중 점검할 항목을 선택한다. 검색 결과에서 표시된 미사용 symbol을 클릭해 위치를 확인하고 정리한다.
   - 예상 리소스: IntelliJ 또는 Android Studio, inspection 실행 시간, 결과 검토 시간
   - 예상 효과: 수작업 탐색보다 적은 시간과 노력으로 사용되지 않는 메소드·변수·심볼을 찾아 프로젝트를 정리할 수 있다.

## 원문 요약 (≤5문장)
글은 프로젝트 규모가 커질수록 미사용 메소드와 변수가 늘어난다는 문제를 다룬다. 작성자는 이런 코드를 눈으로 직접 찾아 지우는 일이 시간과 노력이 많이 든다고 설명한다. 해결책으로 IntelliJ와 Android Studio의 `Run Inspection by Name` 기능에서 `Unused`를 검색해 미사용 항목을 찾는 방법을 제시한다. 예시로 개인 프로젝트에서 사용되지 않는 symbol 42건이 검색됐다고 밝힌다. 마지막으로 이 기능을 활용해 리팩토링과 불필요한 코드 제거를 해보자고 권한다.
