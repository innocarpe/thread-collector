## 핵심 주장 (≤5)
1. 다국어 지원 안드로이드 앱에서 `strings.xml`에 문구 하나를 추가하는 일도 변수명 작성, 언어별 번역, 각 폴더 순회 및 반영까지 수작업이 많이 든다.
2. 작성한 파이썬 스크립트를 사용하면 여러 언어용 `strings.xml` 번역 추가 작업을 10초 이내로 처리할 수 있다.
3. 번역 문자열 생성은 ChatGPT에 위임하고, 스크립트는 각 언어 폴더를 순회하며 `strings.xml`을 자동 생성 또는 업데이트하는 역할을 맡기면 된다.
4. 프롬프트 템플릿을 재사용하면 매번 마지막 번역 대상 문구만 바꿔 ChatGPT로 실행 가능한 명령어를 빠르게 만들 수 있다.
5. 작성자는 추후 Python 스크립트 내부에 ChatGPT API를 붙이면 키워드 하나만으로 번역과 자동 주입까지 가능할 것으로 본다.

## 공개된 숫자·지표
- 처리 시간: 10초 이내 (기간: 스크립트 1회 실행 기준, 출처: "위 모든 번거로움을 단 10초 이내에 작업할 수 있게 파이썬 스크립트를 작성하였다.")
- 지원 언어 수: 10개 (기간: 스크립트 설정 기준, 출처: "languages = ['en', 'ar', 'de', 'es', 'fr', 'it', 'ja', 'ko', 'zh-rTW', 'zh']")

## 언급된 도구·서비스
- ChatGPT: 번역 문자열과 실행 명령어를 생성하는 데 사용
- Python: `add_translation.py` 스크립트를 실행하는 데 사용
- Android `strings.xml`: 다국어 문자열을 저장하고 자동으로 갱신하는 대상 파일
- 파파고: 언어별 번역을 수작업으로 할 때 사용하던 번역기 예시
- 구글 번역기: 언어별 번역을 수작업으로 할 때 사용하던 번역기 예시

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. Android 리소스 폴더를 순회하는 `add_translation.py`를 프로젝트 루트에 두고 언어별 `strings.xml`을 일괄 갱신한다.
   - 구체적 스텝: 프로젝트 최상단에 `add_translation.py` 생성 → `languages` 배열을 실제 지원 언어 순서로 수정 → 터미널에서 `python3 add_translation.py {key} "{번역1|번역2|...}"` 형식으로 실행
   - 예상 리소스: Python 3, Android 프로젝트, 언어별 번역 문자열
   - 예상 효과: `strings.xml` 값 추가 시 각 언어 폴더를 찾아 직접 수정하는 반복 작업을 줄이고, 원문 기준 10초 이내 처리가 가능해진다
2. ChatGPT 프롬프트에 스크립트와 사용 예시를 함께 넣고 마지막 번역 대상 문구만 바꿔 실행 명령어를 생성한다.
   - 구체적 스텝: 프롬프트에 현재 스크립트 코드와 예시 명령어를 붙여넣기 → `languages` 순서를 명시 → 마지막에 번역할 문구만 입력 → ChatGPT가 반환한 `python3 add_translation.py ...` 명령을 그대로 실행
   - 예상 리소스: ChatGPT, 기존 프롬프트 템플릿, 터미널
   - 예상 효과: 번역본과 구분자 `|`가 포함된 긴 명령어를 직접 조합하지 않아도 되어 작업 준비 시간이 줄어든다

## 원문 요약 (≤5문장)
작성자는 안드로이드 앱의 다국어 지원 과정에서 `strings.xml`에 문구 하나를 추가할 때도 변수명 작성, 번역, 언어별 파일 수정이 반복되는 점을 문제로 봤다. 이를 해결하기 위해 언어 폴더를 순회하며 `strings.xml`을 생성하거나 갱신하는 `add_translation.py` 스크립트를 만들었다. 번역문과 실행 명령어는 ChatGPT에 맡기고, 개발자는 결과 명령만 실행하는 흐름을 제안했다. 이 방식으로 작성자는 ChatGPT 응답시간과 스크립트 실행시간만 들고, 전체 작업을 10초 내외로 줄일 수 있다고 설명한다. 추가로 Python 스크립트 안에 ChatGPT API를 붙이면 키워드만 넣어도 번역과 자동 주입까지 가능할 것이라고 전망했다.
