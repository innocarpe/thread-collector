## 핵심 주장 (≤5)
1. 변수 선언과 사용 시 데이터 타입이 보이지 않으면 개발자가 답답함을 느낄 수 있다.
2. 변수 타입이 화면에 보이면 개발 중 "이 타입이 뭐지?"라는 고민을 줄일 수 있다.
3. IntelliJ에서는 버전에 따라 Kotlin 변수 타입 힌트를 표시하는 설정 경로가 다르다.

## 공개된 숫자·지표
- IntelliJ 버전 기준: 2019.3.1 미만 / 2019.3.1 이상 (기간: 없음, 출처: "Intellij 2019.3.1 버전 미만", "Intellij 2019.3.1 버전 이상")

## 언급된 도구·서비스
- IntelliJ: Kotlin 코드에서 변수 타입 힌트를 보이도록 설정하는 IDE로 언급됨.
- Kotlin: IntelliJ 설정에서 타입 힌트를 활성화하는 대상 언어로 언급됨.

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. IntelliJ 버전에 맞는 Inlay/힌트 설정을 켜서 Kotlin 로컬 변수 타입을 IDE에 표시한다.
   - 구체적 스텝: IntelliJ 버전을 확인한 뒤, 2019.3.1 미만이면 `Settings -> Editor -> General -> Appearance -> "Show parameter name hints" -> Configure... -> Language: Kotlin -> Options -> "Show local variable type hints"`로 이동해 옵션을 켠다. 2019.3.1 이상이면 `Settings -> Editor -> Inlay Hints -> Kotlin -> Parameter hints -> "Show parameter hints"`와 `Settings -> Editor -> Inlay Hints -> Kotlin -> Types -> "Show hints for: "`를 체크한다.
   - 예상 리소스: IntelliJ, Kotlin 프로젝트, 설정 변경에 필요한 짧은 작업 시간
   - 예상 효과: 개발 중 변수의 타입을 바로 확인할 수 있어 "이 타입이 뭐지?"에 대한 고민을 덜 수 있다.

## 원문 요약 (≤5문장)
글은 IntelliJ에서 Kotlin 변수 타입 힌트를 표시하는 방법을 설명한다. 작성자는 변수 타입이 보이지 않으면 답답할 수 있고, 타입이 보이면 개발 중 혼란을 줄일 수 있다고 말한다. 이어서 IntelliJ 2019.3.1 미만과 이상 버전으로 나눠 각각의 설정 경로를 제시한다. 핵심은 Inlay Hints 관련 옵션을 켜서 로컬 변수 타입을 IDE에 노출하는 것이다.

## 본문 포인트별 발췌
> 변수 선언하고 사용할때, 해당 데이터가 어떤 타입을 가지는지 안보이면 답답한 경우가 많다.
> 위의 형태보다는 아래처럼 타입이 보인다면 개발할때 "이 타입이 뭐지?"에 대한 고민을 덜어줄 수 있다.
> Settings -> Editor -> General -> Appearance -> "Show parameter name hints" -> Configure... -> Language: Kotlin -> Options -> "Show local variable type hints".
> Settings -> Editor -> Inlay Hints -> Kotlin -> Parameter hints -> "Show parameter hints" 체크
> Settings -> Editor -> Inlay Hints -> Kotlin -> Types -> "Show hints for: " 체크
