## 핵심 주장 (≤5)
1. Intent, Bundle, SavedStateHandle로 객체를 자주 전달할 때 매번 JSON 직렬화/역직렬화 코드를 직접 작성하는 방식은 번거롭다.
2. 객체 전달을 단순화하려면 `putAsJson`, `getAsObject` 같은 확장 함수를 만들어 공통 처리하는 것이 유용하다.
3. 이 확장 함수 방식은 Intent, Bundle, SavedStateHandle 모두에 동일한 패턴으로 적용할 수 있다.
4. Moshi를 쓰는 경우 공통 Provider와 제네릭 확장 함수를 두면 호출부를 크게 줄일 수 있다.
5. Activity 이동도 `start` 확장 함수로 감싸면 Intent 생성 코드를 더 단순하게 만들 수 있다.

## 언급된 도구·서비스
- Intent: Activity 간 데이터 전달 예시로 사용됨
- Bundle: 객체를 JSON 문자열로 저장/복원하는 대상 중 하나로 사용됨
- SavedStateHandle: 객체를 JSON 문자열로 저장/복원하는 대상 중 하나로 사용됨
- Json: 객체 전달 시 직렬화 포맷으로 사용됨
- Moshi: 객체를 JSON으로 변환하고 다시 객체로 매핑하는 데 사용됨
- Gson: Moshi 대신 선호 시 대체 가능하다고 언급됨
- Context: `start` 확장 함수를 통해 `startActivity`를 단순화하는 대상으로 사용됨
- Fragment: `start` 확장 함수를 통해 `startActivity`를 단순화하는 대상으로 사용됨
- Activity: `start` 확장 함수를 통해 `startActivity`를 단순화하는 대상으로 사용됨

## 복제 가능한 전술 (≤3)
1. JSON 직렬화/역직렬화 로직을 Intent·Bundle·SavedStateHandle 공용 제네릭 확장 함수로 통합한다.
   - 구체적 스텝: Moshi 인스턴스를 제공하는 Provider를 만든다. `putAsJson`, `getAsObject` 확장 함수를 Bundle, Intent, SavedStateHandle에 각각 구현한다. 호출부에서는 `putExtra`/`getStringExtra` 대신 해당 확장 함수만 사용하도록 치환한다.
   - 예상 리소스: Android Kotlin 프로젝트, Moshi 또는 Gson, 확장 함수 작성 시간
   - 예상 효과: 객체 전달 코드를 반복 작성하지 않고 "정말 단순화하여 객체를 전달"할 수 있다.
2. Activity 시작 패턴을 `start` 확장 함수로 감싸 호출부를 축약한다.
   - 구체적 스텝: Context, Activity, Fragment 각각에 `startActivity(Intent(...).apply(configIntent))` 형태의 제네릭 `start` 확장 함수를 추가한다. 새 화면 이동 시 `Intent(...)` 생성 대신 `start<대상Activity> { ... }` 형태로 통일한다.
   - 예상 리소스: Android Kotlin 프로젝트, Activity/Fragment 확장 함수 작성 시간
   - 예상 효과: Context, Fragment, Activity만 가지고 있다면 손쉽게 startActivity를 할 수 있다.
3. 기존 AS-IS와 TO-BE 예시를 팀 공용 코드 스타일로 문서화해 객체 전달 방식을 표준화한다.
   - 구체적 스텝: 기존의 직접 JSON 변환 예시와 확장 함수 기반 예시를 나란히 정리한다. 팀 코드리뷰 기준에서 객체 전달 시 확장 함수 사용을 기본 규칙으로 정한다. Gson 선호 팀이면 Provider 구현만 교체한다.
   - 예상 리소스: 사내 문서 또는 README, 직렬화 라이브러리 하나
   - 예상 효과: "복잡한건 심플하게, 최대한 단순하게" 사용하는 방향으로 호출부 복잡도를 낮출 수 있다.

## 원문 요약 (≤5문장)
글은 Android에서 Intent, Bundle, SavedStateHandle로 객체를 전달할 때 매번 JSON 변환 코드를 반복 작성하는 불편함을 지적한다. 작성자는 Moshi 기반의 공통 확장 함수 `putAsJson`, `getAsObject`를 만들어 객체 전달 코드를 단순화했다. 이 방식은 Bundle, Intent, SavedStateHandle 모두에 같은 패턴으로 적용된다. 추가로 Context, Activity, Fragment용 `start` 확장 함수를 제공해 Activity 이동 코드도 더 짧게 만들었다. 전체 메시지는 반복적이고 복잡한 전달 코드를 유틸 함수로 감싸 심플하게 쓰자는 것이다.

## 본문 포인트별 발췌
> Intent, Bundle, SavedStateHandle로 다양한 데이터를 넘겨받는데 특히 객체를 넘겨받을 경우가 많다.
> 여러가지 방법이 있겠지만 주로 사용하는 방법은 객체를 Json으로 변환한 뒤 다시 받는 쪽에서는 Json을 객체에 매핑시켜 데이터를 가져오는 구조로 주로 사용을 했다.
> 우선 **SIMPLE IS BEST를 사랑하는 나는 단순히 데이터를 넘기는데 너무 복잡하다고 생각이 든다**.
> Intent, Bundle, SavedStateHandle 모두 커버가능하며 정말 단순화하여 객체를 전달 할 수 있다.
> 복잡한건 심플하게, 최대한 단순하게 사용해보자.
