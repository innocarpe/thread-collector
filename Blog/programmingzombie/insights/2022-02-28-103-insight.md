## 핵심 주장 (≤5)
1. iTerm2에서 zsh 사용 후 이미지나 기호가 물음표로 보일 때는 Powerline 폰트를 설치하면 해결할 수 있다.
2. 폰트 설치 후 iTerm2의 프로필 설정에서 설치한 폰트로 직접 변경해야 정상 출력된다.
3. 작성자는 맥 환경에서 `git clone`으로 Powerline 폰트를 설치했다.
4. 작성자는 iTerm2 폰트로 `Meslo LG L for Powerline`을 선택해 문제를 해결했다.
5. Oh My Zsh 설정은 별도 블로그 글을 참고해 진행했다.

## 언급된 도구·서비스
- iTerm2: zsh 사용 환경에서 폰트를 변경하는 터미널 앱으로 언급됨
- zsh: iTerm2에서 사용 중인 셸 환경으로 언급됨
- Powerline fonts: 물음표로 보이는 문자 문제를 해결하기 위해 설치하는 폰트 모음
- `apt-get`: 리눅스 환경에서 `fonts-powerline` 설치에 사용
- `dnf`: 페도라 환경에서 `powerline-fonts` 설치에 사용
- `git clone`: 기타 환경 및 맥 환경에서 Powerline 폰트 저장소를 내려받는 데 사용
- Oh My Zsh: 작성자가 별도 참고 글을 통해 설정한 zsh 구성 도구

## 복제 가능한 전술 (≤3)
1. Powerline 폰트를 설치하고 iTerm2 폰트를 교체해 물음표 표시 문제를 해결한다.
   - 구체적 스텝: GitHub의 `powerline/fonts` 저장소로 이동한다 -> 사용 중인 OS에 맞는 방식으로 폰트를 설치한다 -> iTerm2에서 `Settings > Profiles > Text > Font`로 들어간다 -> 설치한 Powerline 계열 폰트로 변경한다 -> 터미널 출력이 정상 표시되는지 확인한다
   - 예상 리소스: 터미널, iTerm2, Powerline 폰트 저장소, 폰트 설치 시간 약간
   - 예상 효과: 원문처럼 물음표로 뜨던 내용이 정상적으로 출력된다
2. 맥 환경에서는 저장소를 직접 클론해 Powerline 폰트를 설치한다.
   - 구체적 스텝: `git clone https://github.com/powerline/fonts.git --depth=1`로 저장소를 받는다 -> `cd fonts` 후 `./install.sh`를 실행한다 -> 설치 후 iTerm2 폰트를 설치한 항목으로 바꾼다
   - 예상 리소스: Git, 터미널, 폰트 설치 스크립트
   - 예상 효과: 작성자와 동일한 방식으로 맥 환경에서 필요한 폰트를 적용할 수 있다
3. 폰트 적용 후 Powerline 호환 폰트를 명시적으로 선택한다.
   - 구체적 스텝: iTerm2 폰트 목록에서 설치된 Powerline 폰트를 찾는다 -> 원문 사례처럼 `Meslo LG L for Powerline` 같은 항목을 선택한다 -> zsh 프롬프트와 특수문자 표시 상태를 확인한다
   - 예상 리소스: iTerm2 폰트 설정 화면, 설치된 Powerline 폰트
   - 예상 효과: 폰트 미설정으로 발생하던 문자 깨짐을 줄이고 zsh 테마 요소를 정상 표시할 수 있다

## 원문 요약 (≤5문장)
이 글은 iTerm2에서 zsh 사용 후 이미지나 기호가 물음표로 나오는 문제를 Powerline 폰트 설치로 해결하는 방법을 설명한다. 리눅스, 페도라, 기타 환경별 설치 명령을 제시하고, 작성자는 맥에서 `git clone` 방식으로 설치했다고 밝힌다. 설치 후에는 iTerm2 프로필의 텍스트 설정에서 설치한 폰트로 바꿔야 한다. 작성자는 `Meslo LG L for Powerline`을 선택했고, 그 결과 물음표로 보이던 내용이 정상 출력되었다고 설명한다. Oh My Zsh 설정은 별도 블로그 글을 참고했다고 덧붙인다.
