## 핵심 주장 (≤5)
1. ADB로 상대방 디바이스에 붙어 APK의 버전명을 확인해야 할 상황이 있을 수 있다.
2. `adb shell dumpsys package my.package | grep versionName` 명령어로 버전 정보를 간단히 얻을 수 있다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- ADB: 상대방 디바이스에 붙어서 APK 버전명을 확인할 때 사용하는 도구
- `dumpsys package`: 지정한 패키지의 정보를 조회하는 adb shell 하위 명령
- `grep`: `versionName` 문자열만 추려 버전명을 확인하는 데 사용

## 언급된 다른 creator·앱
- `my.package`: 버전 정보를 조회할 대상 패키지 예시

## 복제 가능한 전술 (≤3)
1. ADB로 연결된 안드로이드 디바이스에서 특정 앱의 `versionName`만 빠르게 추출한다.
   - 구체적 스텝: 디바이스를 ADB로 연결한다 -> 확인할 앱의 패키지명을 확인한다 -> 터미널에서 `adb shell dumpsys package my.package | grep versionName` 형식으로 실행한다 -> 출력된 `versionName` 값을 기록한다
   - 예상 리소스: ADB가 설치된 터미널 환경, 연결된 안드로이드 디바이스, 패키지명
   - 예상 효과: 원문처럼 APK의 버전명을 간단히 확인할 수 있다

## 원문 요약 (≤5문장)
글은 안드로이드 디바이스에 ADB로 접속해 APK 버전명을 확인하는 방법을 다룬다.
작성자는 실제로 상대방 디바이스에 붙어서 버전명을 알아봐야 하는 상황이 있었다고 설명한다.
해결책으로 `adb shell dumpsys package my.package | grep versionName` 명령어 하나를 제시한다.
핵심은 패키지 정보 전체에서 `versionName` 항목만 추려 빠르게 확인하는 것이다.

## 본문 포인트별 발췌
> ADB로 상대방 디바이스에 붙어서 버전명을 알아봐야 하는 상황이 있었다.
> 아래의 명령어로 간단히 정보를 얻을 수 있다.
> adb shell dumpsys package my.package | grep versionName
