## 핵심 주장 (≤5)
1. Spring과 DB를 연동하기 위해 MySQL을 선택했다.
2. MySQL Workbench 대신 HeidiSQL을 사용한 이유는 회사 컴퓨터에 이미 설치되어 있었고, 사용 편의성 측면에서 선호 의견이 많았기 때문이다.
3. 윈도우에 바로 설치하는 대신 가상머신의 우분투 14.04에 MySQL을 설치하고 외부 IP 접속이 가능하도록 설정해 HeidiSQL로 접속했다.
4. MySQL 외부 접속을 허용하려면 사용자 권한 부여와 `my.cnf`의 `bind-address` 수정이 필요하다.
5. MySQL에서 데이터베이스와 테이블 생성은 일반 SQL 문으로 수행할 수 있다.

## 공개된 숫자·지표
- 우분투 버전: 14.04 (기간: 작성 시점, 출처: "가상머신에서 우분투 14.04를 설치하고 MySQL를 설치하여 외부 IP연결 가능하도록 설정하고,")
- MySQL 설치 패키지 버전: 5.5 (기간: 작성 시점, 출처: "#apt-get install mysql-server-5.5", "#apt-get install mysql-client-5.5")
- MySQL 접속 포트: 3306 (기간: 작성 시점, 출처: "#mysql -h 우분투ip -P 3306 -u root -p", "-A INPUT -p tcp -m tcp --dport 3306 -j ACCEPT")
- 예시 테이블 기본 증가값: 1 (기간: 예시 스키마 작성 시점, 출처: ") ENGINE=InnoDB AUTO_INCREMENT=1 DEFAULT CHARSET=utf8;")

## 언급된 도구·서비스
- Spring: DB 연동을 시도하는 애플리케이션 프레임워크로 언급됨
- MySQL: 사용할 DB로 선택됨
- MySQL Workbench: 검토했지만 사용하지 않은 관리 도구로 언급됨
- HeidiSQL: 외부 연결에 사용한 DB 관리 도구
- 가상머신: 우분투 환경을 설치하는 실행 환경으로 사용됨
- 우분투 14.04: MySQL을 설치한 운영체제 환경
- CentOS: `my.cnf` 위치와 방화벽 설정 예시를 설명할 때 함께 언급됨
- `vi`: `my.cnf`를 수정하는 편집기로 사용됨
- `iptables`: CentOS에서 3306 포트를 열기 위한 방화벽 설정 도구로 언급됨

## 복제 가능한 전술 (≤3)
1. 우분투에 MySQL을 설치하고 원격 접속 가능한 기본 환경을 만든다.
   - 구체적 스텝: `apt-cache search mysql-server`로 패키지를 찾고, `apt-get install mysql-server-5.5`로 서버를 설치한 뒤 `mysql -u root -p`로 접속해 설치 상태를 확인한다.
   - 예상 리소스: 우분투 14.04 환경, MySQL 5.5 패키지, 터미널
   - 예상 효과: Spring 등 애플리케이션과 연동할 MySQL 서버 환경을 마련할 수 있다.
2. 외부 IP 접속을 허용하도록 MySQL 네트워크 설정과 권한을 함께 수정한다.
   - 구체적 스텝: `mysql` DB에 접속해 `GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY '패스워드 입력';`와 `flush privileges;`를 실행하고, `/etc/mysql/my.cnf`에서 `bind-address = 127.0.0.1`를 주석 처리한 뒤 `sudo /etc/init.d/mysql restart`로 재시작한다.
   - 예상 리소스: MySQL root 계정, `my.cnf` 편집 권한, 서버 재시작 권한
   - 예상 효과: localhost 제한을 해제하고 원격 클라이언트에서 MySQL 접속을 시도할 수 있다.
3. 원격 DB 관리 도구에서 바로 사용할 스키마를 생성한다.
   - 구체적 스텝: `show databases;`로 기존 DB를 확인하고 `create database 데이터베이스명;`으로 DB를 만든 뒤 `use 데이터베이스명;` 후 `create table` 문으로 필요한 테이블을 생성한다. 글의 예시처럼 TODO 테이블 구조를 참고해 기본 키와 문자셋을 지정할 수 있다.
   - 예상 리소스: MySQL 콘솔 또는 HeidiSQL, SQL 실행 권한
   - 예상 효과: HeidiSQL 같은 도구에서 바로 조회·수정 가능한 초기 데이터베이스 구조를 준비할 수 있다.

## 원문 요약 (≤5문장)
글쓴이는 Spring과 데이터베이스를 연동하기 위해 MySQL을 사용했다. 관리 도구는 MySQL Workbench 대신 회사 PC에 이미 설치되어 있고 사용 편의성 평가가 좋았던 HeidiSQL을 선택했다. 실행 환경은 윈도우가 아니라 가상머신의 우분투 14.04였고, 외부 IP에서 접속 가능하도록 MySQL을 설정했다. 이를 위해 사용자 권한 부여, `my.cnf`의 `bind-address` 주석 처리, MySQL 재시작 절차를 정리했다. 마지막으로 데이터베이스와 테이블 생성 예시, 그리고 참고한 외부 자료 링크를 남겼다.

## 본문 포인트별 발췌
> Spring과 DB연동을 해보기위해 DB는 Mysql을 사용하게 되었다.
> MySQL에서 제공하는 workbench를 사용할까 했지만, 이미 회사 컴퓨터에 HeidiSQL 툴이 설치되어있었기 때문에 그리고,
> 가상머신에서 우분투 14.04를 설치하고 MySQL를 설치하여 외부 IP연결 가능하도록 설정하고,
> bind-address = 127.0.0.1 이 부분 주석
> mysql 에서는 sql문 사용이 가능하다
