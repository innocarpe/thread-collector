## 핵심 주장 (≤5)
1. 스플래시 화면을 `theme`로 보여주는 구성에서 특정 기종은 앱 실행과 동시에 종료될 수 있다.
2. 에러 원인은 `splash.xml`의 `<item><bitmap></bitmap></item>` 구조에서 `src`를 유효하게 읽지 못하는 데 있다.
3. `layer-list` 안의 이미지를 `<bitmap>` 대신 `android:drawable`가 달린 `<item>`으로 바꾸면 문제를 해결할 수 있다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- Android Manifest: `SplashActivity`에 `@style/SplashTheme`를 적용하는 스플래시 진입점 설정에 사용됨
- `styles.xml`: `android:windowBackground`에 `@drawable/splash`를 지정해 스플래시 배경을 연결하는 데 사용됨
- `splash.xml`: `layer-list` 기반 스플래시 드로어블을 정의하는 데 사용됨
- `@mipmap/ic_launcher`: 스플래시 중앙 이미지 리소스로 사용됨

## 언급된 다른 creator·앱
- 넥서스 5: 특정 기종에서 앱이 실행과 동시에 죽는 사례로 언급됨

## 복제 가능한 전술 (≤3)
1. 스플래시용 `layer-list`에서 `<bitmap android:src=...>` 구조를 제거하고 `<item android:drawable=...>`로 치환한다.
   - 구체적 스텝: `styles.xml`에서 `android:windowBackground`가 가리키는 스플래시 드로어블 파일을 찾고, 기존 `<item><bitmap android:gravity="center" android:src="@mipmap/ic_launcher"/></item>` 구문을 `<item android:gravity="center" android:drawable="@mipmap/ic_launcher"/>`로 교체한 뒤 여러 기종에서 앱 실행을 확인한다.
   - 예상 리소스: Android XML 편집기, 테스트 기기 또는 에뮬레이터
   - 예상 효과: 특정 기종에서 발생하던 스플래시 진입 직후 크래시를 방지하고 모든 기종에서 정상 동작을 기대할 수 있다.

## 원문 요약 (≤5문장)
작성자는 안드로이드 앱의 스플래시 화면을 `theme` 기반으로 구성했다가 일부 기종에서 앱이 실행 직후 종료되는 문제를 겪었다. 에러 로그에는 `Binary XML file line #6: <bitmap> requires a valid 'src' attribute`가 나타났다. 원인을 `splash.xml`의 `<item><bitmap></bitmap></item>` 구조로 판단했다. 이후 `layer-list` 내부에서 `<bitmap>`을 제거하고 `<item android:drawable="...">` 형태로 수정한 뒤 모든 기종에서 정상 동작을 확인했다.

## 본문 포인트별 발췌
> 앱을 만들고 스플래시 화면을 단순 theme로 보여주는데
> 아래와 같은 에러가 발생되었다.
> org.xmlpull.v1.XmlPullParserException: Binary XML file line #6: <bitmap> requires a valid 'src' attribute
> 대부분은 잘 잘동하지만 특정 기종에선(넥서스 5, etc.. ) 아예 앱 실행과 동시에 죽어버리는 불상사가 발생했다.
> 아래와 같이 수정하고 난 후에는 모두 잘 동작함을 확인하였다.
