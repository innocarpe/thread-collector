## 핵심 주장 (≤5)
1. 디렉토리내 일정기간이 지난 파일을 삭제할 수 있다.
2. 파일의 마지막 수정시간과 현재 시간의 차이를 계산하면 삭제 대상을 판별할 수 있다.
3. 예제는 마지막 수정시간이 현재로부터 3일이 지난 파일을 삭제하는 방식이다.

## 공개된 숫자·지표
- 삭제 기준: 3일 (기간: 파일 보관 기준, 출처: "아래는 파일의 마지막 수정시간이 현재로부터 3일이 지난 파일을 삭제하는 예제이다.")
- 일 단위 계산값: 24*60*60*1000 (기간: 1일, 출처: "// 일 단위")

## 언급된 도구·서비스
- Java `Calendar`: 현재 시간과 파일 수정시간을 밀리초로 계산하는 데 사용됨
- Java `Date`: 파일의 마지막 수정시간을 `Date` 객체로 변환하는 데 사용됨
- Java `File`: 디렉토리 경로 지정, 파일 목록 조회, 파일 삭제에 사용됨

## 언급된 다른 creator·앱

## 복제 가능한 전술 (≤3)
1. 파일 수정일 기준으로 오래된 파일을 정리하는 Java 스크립트를 만든다.
   - 구체적 스텝: `File`로 대상 디렉토리를 지정하고 `listFiles()`로 파일 목록을 가져온다. 각 파일의 `lastModified()` 값을 `Date`와 `Calendar`로 변환해 현재 시간과의 차이를 밀리초 및 일 단위로 계산한다. 계산된 일수가 3일을 초과하고 파일이 존재하면 `delete()`를 호출하고 삭제 파일명을 출력한다.
   - 예상 리소스: Java 실행 환경, 정리 대상 디렉토리 경로, `Calendar`/`Date`/`File` 클래스
   - 예상 효과: 마지막 수정시간이 현재로부터 3일이 지난 파일을 삭제할 수 있다.

## 원문 요약 (≤5문장)
이 글은 Java로 디렉토리 내에서 일정 기간이 지난 파일을 삭제하는 예제를 소개한다. 현재 시간과 파일의 마지막 수정시간 차이를 밀리초로 계산한 뒤 일 단위로 환산해 경과 일수를 구한다. 예제의 삭제 기준은 3일 초과다. 구현에는 `Calendar`, `Date`, `File`이 사용된다.
