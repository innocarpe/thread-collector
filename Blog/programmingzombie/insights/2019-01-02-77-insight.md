## 핵심 주장 (≤5)
1. Android에서는 `Intent(Intent.ACTION_VIEW, Uri.parse(url))`와 `context.startActivity(intent)` 두 줄로 외부 앱이나 URL 대상 화면을 실행할 수 있다.
2. 인스타그램, 유튜브, 플레이스토어 이동은 각 서비스가 처리하는 URL scheme 또는 custom scheme 값을 넣어 구현할 수 있다.
3. URL scheme은 시스템이 전달받아 실행 가능한 앱을 찾고, 해당 앱이 URL을 해석해 특정 기능을 수행하는 방식으로 동작한다.
4. 웹이나 앱에서 URL scheme으로 외부 앱을 실행할 때는 대상 앱이 설치되지 않은 경우를 대비한 예외 처리가 필요하다.
5. 앱에 `intent-filter`로 custom uri scheme을 등록하면 메인 액티비티가 아니라 특정 액티비티를 직접 실행시킬 수 있다.

## 언급된 도구·서비스

- 플레이스토어: `market://details?id=$appPackageName` 또는 웹 URL로 앱 리뷰/앱 상세 페이지를 여는 용도
- Instagram: 특정 사용자 프로필로 이동시키는 대상 서비스로 사용됨
- YouTube: 특정 영상 또는 페이지를 앱으로 실행시키는 대상 서비스로 사용됨
- 웹 브라우저: `http://` 스킴을 처리할 수 있는 앱 예시로 언급됨

## 복제 가능한 전술 (≤3)
1. 외부 서비스 앱 실행에 앱 우선, 웹 폴백 구조를 적용한다.
   - 구체적 스텝: `ACTION_VIEW` 인텐트를 만든다 → 1차 URL에 서비스별 scheme 또는 앱 전용 URL을 넣는다 → 2차 URL에 웹 fallback 주소를 넣는다 → `startActivity`를 `try-catch(ActivityNotFoundException)`로 감싸 설치 여부에 따라 1차 또는 2차 인텐트를 실행한다
   - 예상 리소스: Android 앱 코드 수정, Intent/Uri API, 외부 서비스별 URL scheme 확인 작업
   - 예상 효과: 인스타그램, 유튜브, 플레이스토어 같은 외부 목적지를 간단한 공통 함수로 실행하고, 앱 미설치 시에도 웹으로 전환할 수 있다
2. 리뷰 유도나 사용 설명 진입점을 서비스별 런처 유틸로 묶는다.
   - 구체적 스텝: 앱 패키지명 기반 플레이스토어 URL 생성 함수를 만든다 → 유튜브 ID, 인스타그램 username을 받는 함수로 분리한다 → 내부적으로 공통 `launchApp(context, firstUrl, secondUrl)`를 호출하게 구성한다
   - 예상 리소스: Android 유틸 클래스 1개, 서비스별 식별자 문자열
   - 예상 효과: 리뷰 요청, 사용 설명 영상 연결, SNS 유입을 반복 코드 없이 일관되게 처리할 수 있다
3. 앱 내부 특정 화면을 외부에서 직접 열 수 있도록 custom uri scheme을 등록한다.
   - 구체적 스텝: 대상 액티비티에 `intent-filter`를 추가한다 → `VIEW`, `DEFAULT`, `BROWSABLE` 카테고리와 `scheme`, `host`를 설정한다 → 다른 앱에서 `Intent(Intent.ACTION_VIEW, Uri.parse("scheme://host"))` 형태로 호출한다
   - 예상 리소스: AndroidManifest 설정, 호출용 인텐트 코드, 테스트용 앱 2개
   - 예상 효과: 메인 액티비티를 거치지 않고 지정한 액티비티를 직접 실행시킬 수 있다

## 원문 요약 (≤5문장)
이 글은 Android Intent와 URL scheme을 이용해 인스타그램, 유튜브, 플레이스토어 같은 외부 앱을 여는 방법을 설명한다. 핵심 구현은 `ACTION_VIEW` 인텐트에 적절한 URL을 넣어 실행하는 것이다. 작성자는 앱이 설치되지 않았을 때를 대비해 앱 URL과 웹 URL을 함께 두고 `ActivityNotFoundException`으로 폴백 처리하는 구조를 제안한다. 또한 `intent-filter`에 custom uri scheme을 등록하면 다른 앱에서 특정 액티비티를 직접 실행할 수 있음을 예제로 보여준다. 전체적으로 외부 서비스 연결과 앱 간 딥링크 처리의 기초를 실용 코드 중심으로 정리한 글이다.

## 본문 포인트별 발췌
> 핵심은 이 두줄로 가능합니다. url에다가 알맞은 값만 넣어주면 끝인데요!
> 브라우저 리스트가 나온다는 것은, http://는 URL로 시작되는 경우를 감지해서 앱을 실행하려고 하는 것을 알 수 있습니다.
> "http://", "ftp://", "market://"과 같은 문자열을 url scheme이라 부릅니다.
> ※ 웹 페이지에서 url scheme을 이용해 앱을 실행하는 경우, 앱이 미설치된 경우의 예외 처리를 구현해야 합니다.
> 따라서 위의 값으로 Intent를 실행시키면 해당 앱이 실행되는 것을 알 수 있습니다.
