# programmingzombie 반복 패턴 분석

## 반복 사용 전술
- 에러-원인-수정 순서의 문제 해결 기록: 50개 중 최소 18개 글이 특정 오류나 막힘을 바로 해결하는 메모 형식이다. `Fragment BackStack`, SQLite 9시간 오차, `split()` 특수문자, RecyclerView 체크 상태, 네트워크 상태 체크처럼 “문제 상황을 짚고, 원인을 좁히고, 바로 적용 가능한 코드 한 줄이나 설정 한 군데를 제시하는” 방식이 반복된다.
- 외부 레퍼런스 큐레이션 후 실습: 최소 10개 글이 직접 긴 설명을 쓰기보다 좋은 링크, 예제 소스, 공식 문서를 먼저 모아두는 패턴을 보인다. MDN, Google Developers, GitHub 예제, 블로그 튜토리얼, socket.io 샘플을 저장해두고 나중에 따라 치거나 재활용하는 방식이다.
- 작은 재사용 단위로 구조화: 안드로이드와 자바 글들에서 싱글턴 SQLite 헬퍼, 백그라운드 스레드 분리, `ScheduledExecutorService`, `Callable`, 공통 권한 목록처럼 “한 번 정리해두면 다음 글에서도 다시 쓰는 단위”를 만든다. 즉흥 해결보다 재사용 가능한 패턴으로 굳히려는 성향이 강하다.

## 일관되게 사용하는 도구
- Android/안드로이드: 제목 기준 25편에서 중심 무대다. UI, 권한, SQLite, WebView, 네트워크, Fragment, RecyclerView까지 앱 개발의 거의 전 영역을 안드로이드에서 실험한다.
- Java: 최소 6편 이상에서 스케줄링, 스레드, 파일 삭제, 날짜 비교, 문자열 처리 같은 백엔드·기초 런타임 문제를 다룬다. 안드로이드 앱의 기반 언어이자 문제 해결 도구로 꾸준히 쓰인다.
- Stack Overflow: 7편에서 직접 출처로 등장한다. 막히면 검색으로 해답을 찾되, 그대로 복붙하는 데서 멈추지 않고 다시 블로그 메모로 재정리하는 중간 저장소 역할을 한다.
- SQLite: 최소 4편에서 저장, 조회, 시간대 문제, 데이터 확인법까지 반복 등장한다. 개인 앱에 데이터를 남기기 위한 가장 현실적인 저장소로 일관되게 채택된다.
- Trello: 1편뿐이지만 운영 철학 측면에서 중요하다. 문제-리서치-개발-테스트-완료의 5단계 흐름은 이후 글들의 사고방식과도 맞닿아 있다.

## 반복되는 수익화 패턴
- 직접 수익화보다 역량 축적 우선: 이 50개 글에서는 광고, 결제, 매출, 유료화 같은 직접 수익화 패턴이 사실상 0회다. 대신 앱을 더 안정적으로 만들고, 개발 속도를 높이고, 시행착오를 줄이는 축적형 투자가 반복된다.
- 커리어를 간접 수익화 수단으로 봄: 2017년 글 1편에서만 채용·브랜드 관점이 분명하게 나온다. 강점을 설명할 수 있는 개발자가 장기적으로 더 큰 보상을 얻는다는 시각이며, 이는 제품 매출보다 자기 경쟁력의 복리 성장에 가깝다.

## 핵심 믿음·철학
- 기술 스택 자체보다 문제 해결을 우선한다.
> 소프트웨어 엔지니어는 주어진 문제를 소프트웨어적으로 해결하는 사람.
> 소프트웨어의 방점이 있는 것이 아니라 사실은 문제라는 것에 방점이 있다.
- 해결책을 찾으면 끝내지 않고, 왜 먹히는지까지 파고들려 한다.
> 좋은 아이디어를 원하는 것이 아니라,  될 때까지 해결하는 것이 중요하다.
- 운영 방식도 문제 해결 흐름으로 본다.
> 리스트를 왼쪽부터 순서대로 문제, 리서치, 개발, 테스트, 완료로 배치하였습니다.

## 성장 궤적
2015년 말에는 안드로이드 통신, 스케줄링, 기본 Java 개념처럼 손에 잡히는 구현 문제를 푸는 데 집중한다. 2016년으로 가면 SQLite, Spring, MySQL, 리눅스 서버, 트랜잭션, 테스트 환경처럼 앱 바깥의 저장소·서버·개발환경까지 시야가 넓어진다. 동시에 참고 링크를 모으는 아카이브형 글과, 바로 막힌 문제를 해결하는 트러블슈팅형 글이 병행되며 학습 시스템이 생긴다. 2017년에는 다시 안드로이드 실무 이슈를 다루면서도, 마지막에는 “무엇을 잘하는 사람인가”와 “문제를 어떻게 푸는가”라는 커리어 프레임으로 추상화 수준이 올라간다. 즉, 기술 팁 수집가에서 문제 해결 중심의 개발자로 자기 정체성을 정리해가는 흐름이 보인다.
