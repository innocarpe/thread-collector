## 핵심 주장 (≤5)
1. 안드로이드에서 SQLite DB 정보는 DB 파일을 기기에서 내보낸 뒤 별도 도구로 열어보면 쉽게 확인할 수 있다.
2. DB 내보내기는 Eclipse의 DDMS 또는 Droid Explorer로 수행할 수 있다.
3. 추출한 `data.sqlite` 파일은 Firefox 확장 도구인 SQLite Manager로 열어 볼 수 있다.
4. 전체 과정의 핵심은 "db export"와 "view db" 두 단계로 나눠 정리할 수 있다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- Eclipse: 안드로이드 에뮬레이터를 실행하고 DDMS의 File Explorer에서 DB 파일을 추출하는 도구
- DDMS: `data/data/{package name}/databases` 경로에서 `data.sqlite`를 찾아 pull 하는 절차에 사용
- Droid Explorer: Eclipse에서 File Explorer 내용이 보이지 않을 때 DB export 대안으로 사용
- Firefox: SQLite Manager 확장 도구를 설치하는 브라우저
- SQLite Manager: 추출한 `data.sqlite` 파일을 연결해 DB 내용을 조회하는 도구
- Android Emulator: 실행 중인 앱의 데이터 디렉터리에 접근하기 위해 사용

## 언급된 다른 creator·앱
- onjo.tistory.com: 블로그 출처 링크로 언급됨
- mysqlyog: SQLite Manager의 개념을 설명하기 위해 MySQL용 도구와 비교 사례로 언급됨

## 복제 가능한 전술 (≤3)
1. 에뮬레이터 기반으로 앱의 SQLite 파일을 추출해 외부 툴로 검토하는 2단계 점검 루틴을 만든다.
   - 구체적 스텝: Eclipse에서 프로젝트를 선택하고 Android Emulator를 실행한 뒤 DDMS perspective의 File Explorer에서 `data/data/{package name}/databases`로 이동해 `data.sqlite`를 pull한다. 이후 Firefox의 SQLite Manager에서 `Connect Database`로 해당 파일을 연다.
   - 예상 리소스: Eclipse, Android Emulator, Firefox, SQLite Manager 확장, 필요 시 Droid Explorer
   - 예상 효과: 안드로이드 앱의 SQLite 데이터를 "쉽게" 확인하고 DB 내용을 눈으로 검토할 수 있다.
2. Eclipse의 File Explorer가 동작하지 않을 때를 대비해 DB 추출 대체 경로를 준비한다.
   - 구체적 스텝: DDMS의 File Explorer에서 데이터가 보이지 않으면 Droid Explorer를 사용해 동일한 DB 파일을 export한 뒤, 이후 절차는 동일하게 SQLite Manager로 이어간다.
   - 예상 리소스: Droid Explorer, Firefox, SQLite Manager
   - 예상 효과: Eclipse 환경 문제와 무관하게 DB export 작업을 지속할 수 있다.

## 원문 요약 (≤5문장)
글은 안드로이드 앱의 SQLite 데이터를 쉽게 확인하는 방법을 초보자용 팁으로 정리한다. 핵심 절차는 DB 파일을 기기에서 추출하는 단계와 추출한 파일을 열어보는 단계로 나뉜다. 추출은 Eclipse의 DDMS 또는 Droid Explorer로 수행하고, 조회는 Firefox의 SQLite Manager를 사용한다. 작성자는 이 흐름을 MySQL에서 mysqlyog를 사용하는 것과 비슷한 개념으로 설명한다.
