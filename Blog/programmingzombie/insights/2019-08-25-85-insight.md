## 핵심 주장 (≤5)
1. 특정 앱이 설치되어 있으면 해당 앱을 실행하고, 없으면 플레이스토어 설치 화면으로 보내는 흐름으로 구현할 수 있다.
2. 이 기능은 먼저 설치 여부를 판단한 뒤, 설치 시 앱 실행, 미설치 시 스토어 이동의 3단계 로직으로 처리하면 된다.
3. 코틀린에서는 `Context` 확장 함수를 쓰면 관련 동작을 `Context` 범위에서 직접 호출할 수 있다.

## 공개된 숫자·지표
- 없음

## 언급된 도구·서비스
- 안드로이드 앱개발: 특정 앱 설치 여부를 판단하고 실행 또는 스토어 이동 로직을 구현하는 맥락
- intent: 설치된 앱을 여는 방식으로 사용
- 구글 플레이스토어: 앱이 없을 때 해당 앱의 설치화면으로 보내는 용도
- 코틀린: `Context` 확장 함수를 사용하는 언어/구현 맥락

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. 앱 딥링크 대신 "설치 여부 확인 후 실행/스토어 분기" 흐름을 기본 패턴으로 구현한다.
   - 구체적 스텝: 실행하려는 대상 앱의 설치 여부를 먼저 확인한다 -> 설치되어 있으면 `intent`로 앱을 연다 -> 설치되어 있지 않으면 구글 플레이스토어의 해당 앱 설치 화면으로 이동시킨다
   - 예상 리소스: 안드로이드 개발 환경, 코틀린 또는 안드로이드 코드 작성 시간
   - 예상 효과: 사용자가 대상 앱을 이미 설치한 경우 즉시 실행시키고, 없으면 바로 설치 유도로 전환할 수 있다
2. 코틀린 `Context` 확장 함수로 실행/스토어 이동 로직을 재사용 가능하게 묶는다.
   - 구체적 스텝: `fun Context.market(..)` 형태로 스토어 이동 함수를 만든다 -> `fun Context.openApp(..)` 형태로 앱 실행 함수를 만든다 -> `Context` 범위 안에서 필요한 곳마다 직접 호출한다
   - 예상 리소스: 코틀린, `Context` 기반 확장 함수 작성
   - 예상 효과: `Context` 영역 아래에서 함수를 바로 사용할 수 있어 호출부를 단순하게 유지할 수 있다

## 원문 요약 (≤5문장)
글은 안드로이드에서 특정 앱이 설치되어 있으면 실행하고, 없으면 플레이스토어 설치 화면으로 보내는 기본 로직을 설명한다. 작성자는 이를 설치 여부 확인, 설치 시 앱 실행, 미설치 시 스토어 이동의 3단계로 정리한다. 구현 자체는 단순하다고 강조한다. 추가 팁으로 코틀린의 `Context` 확장 함수를 사용하면 이런 동작을 `Context` 범위 안에서 바로 호출할 수 있다고 설명한다.

## 본문 포인트별 발췌
> - 특정 앱이 있으면 그 앱을 실행킨다.
> - 만약 없다면 플레이스토어에서 해당 앱을 설치하도록 유도한다.
> 1. 우선 사용자의 폰에 내가 실행시키고자 하는 앱이 설치 되어있는지 판단해야된다.
> 2. 앱이 설치가 되어있다면 그 앱을 intent를 활용하여 연다.
> 3. 설치가 되어있지 않다면 구글 플레이스토어에 그 앱의 설치화면으로 보낸다.
