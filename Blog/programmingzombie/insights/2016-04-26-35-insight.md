## 핵심 주장 (≤5)
1. 안드로이드 디바이스의 파일은 `adb pull` 명령어로 컴퓨터로 옮길 수 있다.
2. 디바이스에서 꺼낸 데이터베이스 파일을 PC로 가져오면 별도 툴로 테이블 수정과 내용 확인이 가능하다.

## 공개된 숫자·지표
- 전송된 파일 크기: 25 bytes (기간: 예시 실행 시점, 출처: `0 KB/s (25 bytes in 0.226s)`)
- 전송 소요 시간: 0.226s (기간: 예시 실행 시점, 출처: `0 KB/s (25 bytes in 0.226s)`)

## 언급된 도구·서비스
- SQLite: 관련 개발 중 작성한 데이터베이스 파일을 다루는 맥락에서 언급됨
- adb: 디바이스 파일을 컴퓨터로 가져오는 명령어로 사용됨
- 특정 툴: 가져온 데이터베이스의 테이블을 수정하고 내용을 보는 용도로 언급됨

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. 안드로이드 기기에서 필요한 파일을 `adb pull`로 PC로 추출해 로컬 도구에서 점검한다.
   - 구체적 스텝: Android SDK의 `platform-tools` 경로에서 터미널을 연다 -> `adb pull 기기파일 PC위치` 형식으로 명령어를 입력한다 -> 복사된 파일을 PC의 DB/파일 확인 툴로 열어 테이블이나 내용을 점검한다
   - 예상 리소스: `adb`가 포함된 Android SDK `platform-tools`, 파일 확인용 로컬 툴, 터미널 작업 시간
   - 예상 효과: 디바이스 내부 파일을 컴퓨터로 복사해 테이블 수정이나 내용 확인 작업을 할 수 있다

## 원문 요약 (≤5문장)
작성자는 SQLite 관련 개발 중 자신이 만든 데이터베이스 파일을 컴퓨터로 가져와 확인하고 싶었다. 이를 위해 안드로이드 디바이스의 파일을 PC로 복사하는 방법을 찾았고, `adb pull 기기파일 PC위치` 명령어를 제시했다. 예시로 `/etc/hosts` 파일을 `d:\`로 복사하는 명령과 실행 결과를 보여준다. 또한 명령 실행 후 지정한 드라이브에 파일이 복사된 것을 확인할 수 있다고 설명한다.

## 본문 포인트별 발췌
> SQLite 관련 개발 중 내가 작성한 데이터베이스 파일을 컴퓨터로 가져와서
> 특정 툴을 이용하여 테이블을 수정하고 내용을 보고싶었다.
> 찾아보던 도중 아래와 같은 방법으로 디바이스의 파일을 컴퓨터로 옮길 수 있었다.
> adb pull 기기파일 PC위치
> 지정한 F 드라이버에 해당 파일이 복사된것을 확인 할 수 있다.
