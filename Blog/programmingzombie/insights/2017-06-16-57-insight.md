## 핵심 주장 (≤5)
1. 지뢰찾기 숫자는 해당 칸을 기준으로 자신을 제외한 주변 8칸에 있는 지뢰 수를 의미한다.
2. 지뢰찾기 판정 로직은 이중 배열로 표현할 수 있으며, 특정 좌표의 숫자는 인접 배열 주소들에 있는 지뢰 개수로 계산할 수 있다.
3. 지뢰를 찾는 경우의 수는 자기 자신을 제외한 총 8가지 방향 검사로 정리할 수 있다.
4. 배열 경계를 벗어나는 좌표는 조건문으로 제외해야 `ArrayIndexOutOfBoundsException`을 예방할 수 있다.
5. 구현의 핵심은 인접 8칸 탐색 함수와 배열 범위 예외 처리 함수 두 가지다.

## 공개된 숫자·지표
- 인접 탐색 경우의 수: 8가지 (기간: 구현 시점, 출처: "지뢰를 찾을 수 있는 경우의 수는 자기 자신의 배열 주소값을 제외한 총 8가지 방법이 있다.")
- 배열 크기: 10 x 10 (기간: 예시 구현 시점, 출처: "private static final int ROW        = 10;", "private static final int COL        = 10;")
- 지뢰 개수: 10개 (기간: 예시 구현 시점, 출처: "private static final int MINE_CNT   = 10;")

## 언급된 도구·서비스
- Java: 지뢰찾기 로직 예제 코드를 구현한 언어
- Random: 랜덤한 배열 주소에 지뢰를 배치할 때 사용한 클래스
- GitHub: 코드 작성자 정보에 함께 적은 저장소 서비스

## 복제 가능한 전술 (≤3)
1. 인접 8방향을 고정 규칙으로 분해해 숫자 계산 함수를 만든다.
   - 구체적 스텝: 현재 좌표를 기준으로 `row-1/row/row+1`과 `col-1/col/col+1` 조합 중 자기 자신을 제외한 8칸을 나열하고, 각 칸에 지뢰가 있으면 카운트를 증가시킨다.
   - 예상 리소스: Java 같은 배열 기반 언어, 인접 좌표 검사 함수 1개
   - 예상 효과: 각 칸 주변 지뢰 수를 일관된 규칙으로 계산할 수 있다.
2. 경계 검사를 별도 함수로 분리해 배열 예외를 막는다.
   - 구체적 스텝: 좌표가 0 미만이거나 `ROW`, `COL` 이상이면 즉시 `false`를 반환하고, 범위 안에 있을 때만 실제 지뢰 여부를 확인한다.
   - 예상 리소스: 조건문이 들어간 헬퍼 함수 1개
   - 예상 효과: 가장자리 칸을 검사할 때도 배열 주소 범위를 벗어나지 않는다.
3. 랜덤 배치 시 중복 좌표가 나오면 같은 개수만큼 다시 시도한다.
   - 구체적 스텝: 난수로 `row`, `col`을 뽑아 이미 지뢰가 있으면 카운트를 되돌리고 루프를 한 번 더 돌린다. 비어 있으면 해당 칸을 지뢰로 설정한다.
   - 예상 리소스: `Random`, 이중 배열, 반복문
   - 예상 효과: 중복 없이 지정한 개수만큼 지뢰를 배열에 배치할 수 있다.

## 원문 요약 (≤5문장)
글쓴이는 사전과제로 지뢰찾기 구현을 받으면서 게임 규칙부터 다시 정리했다.
핵심 규칙은 한 칸의 숫자가 자기 자신을 제외한 주변 8칸의 지뢰 수를 뜻한다는 점이다.
이를 위해 이중 배열을 사용하고, 특정 좌표 주변 8방향을 검사하는 함수를 설계했다.
또한 가장자리 칸에서 배열 범위를 벗어나는 예외를 조건문으로 처리해야 한다고 설명했다.
마지막으로 10x10 배열에 지뢰 10개를 랜덤 배치하고 숫자를 출력하는 전체 Java 예제 코드를 제시했다.

## 본문 포인트별 발췌
> 한창 취업을 위해 이력서를 제출하고 있는데, 사전과제로 지뢰찾기를 구현하라는 과제를 받게되어
> 지뢰찾기에 대해서 고민해보는 시간을 가지게 되었다.
> 그렇다 이는 이중 배열을 써야하고 만약 [2,2]의 위치에 숫자 3이 있다고 가정한다면
> 지뢰를 찾을 수 있는 경우의 수는 자기 자신의 배열 주소값을 제외한 총 8가지 방법이 있다.
> 위와 같이 지뢰의 범위를 벗어나는 경우는 조건문을 걸어 제외 시켜주었다.
