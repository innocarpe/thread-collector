## 핵심 주장 (≤5)
1. 안드로이드 Instrumentation 테스트는 장치나 시뮬레이터에 직접 테스트 코드를 올려 실행하므로 비교적 느리다.
2. Android Studio에서는 데스크탑 환경에서 실행하는 Junit 기반 로컬 단위 테스트 환경을 구축할 수 있다.
3. Junit 테스트를 위해서는 `src/test/java` 폴더 구조를 사용하고 Gradle 의존성에 Junit을 추가해야 한다.
4. `Build Variant`의 `Test Artifact`를 `Unit Tests`로 설정하면 Android Studio가 해당 java 폴더를 테스트 폴더로 인식한다.
5. 테스트 폴더에서 `Run -> All Tests`로 전체 테스트를 실행할 수 있다.

## 공개된 숫자·지표
- 테스트 환경 OS 버전: Window 10 (기간: 작성 시점, 출처: "* Window 10  /  * Android Studio 2.1.1 으로 아래의 내용을 테스트 해본 결과 동일하게 잘 동작함을 확인하였습니다.")
- 테스트 환경 IDE 버전: Android Studio 2.1.1 (기간: 작성 시점, 출처: "* Window 10  /  * Android Studio 2.1.1 으로 아래의 내용을 테스트 해본 결과 동일하게 잘 동작함을 확인하였습니다.")
- Junit 의존성 버전: 4.12 (기간: 설정 시점, 출처: "testCompile 'junit:junit:4.12'")

## 언급된 도구·서비스
- Android Studio: Junit 기반 단위 테스트 환경을 구축하고 `Build Variant`, Gradle Sync, 테스트 실행에 사용됨
- Junit: 로컬 단위 테스트의 기본 프레임워크로 `testCompile` 의존성에 추가됨
- Mocking 라이브러리: Junit 기반 테스트에서 함께 사용할 수 있는 라이브러리로 언급됨
- Robolectric: 데스크탑 환경 테스트에 사용할 수 있는 프레임워크 예시로 언급됨
- Gradle: `build.gradle`에 테스트 의존성을 추가하고 Sync 하는 데 사용됨
- 안드로이드 장치 또는 시뮬레이터: Instrumentation 테스트를 올려 실행하는 환경으로 언급됨

## 복제 가능한 전술 (≤3)
1. Android Studio 프로젝트에서 로컬 Junit 테스트 폴더를 표준 구조로 전환한다.
   - 구체적 스텝: Blank Activity 프로젝트를 생성한 뒤 프로젝트 뷰를 `Project`로 바꾸고 `androidTest` 폴더에서 `Refactor -> Rename`으로 이름을 `test`로 변경해 `src/test/java` 구조를 맞춘다.
   - 예상 리소스: Android Studio, 기존 프로젝트 1개, 수분 내 설정 작업
   - 예상 효과: Android Studio가 Junit 테스트용 java 폴더를 기본 테스트 폴더로 인식하는 기반을 만든다.
2. Gradle에 Junit 의존성을 추가하고 프로젝트를 동기화한다.
   - 구체적 스텝: app 모듈의 `build.gradle` `dependencies`에 `testCompile 'junit:junit:4.12'`를 추가한 뒤 `Tools -> Android -> Sync Project with Gradle Files`를 실행한다.
   - 예상 리소스: `build.gradle` 수정 권한, Gradle Sync 1회
   - 예상 효과: 가장 기본이 되는 Junit 테스트 환경이 구성되어 로컬 단위 테스트 코드를 실행할 수 있다.
3. `Test Artifact`를 로컬 테스트로 바꾸고 간단한 계산 테스트로 환경을 검증한다.
   - 구체적 스텝: `Build Variant` 탭에서 `Test Artifact`를 `Unit Tests`로 설정하고, `Calculator`와 `CalculatorTest`를 만든 다음 테스트 java 폴더에서 `우클릭 -> Run -> All Tests`를 실행한다.
   - 예상 리소스: 테스트 클래스 2개, Junit assertion 사용
   - 예상 효과: 안드로이드 장치나 시뮬레이터 없이 데스크탑 환경에서 테스트가 정상 동작하는지 빠르게 확인할 수 있다.

## 원문 요약 (≤5문장)
글은 Android Studio에서 Junit 기반 로컬 단위 테스트 환경을 만드는 기본 절차를 단계별로 설명한다. 작성자는 Instrumentation 테스트가 장치나 시뮬레이터를 사용해 상대적으로 느리다고 짚고, 데스크탑 환경에서 실행하는 방식의 장점을 소개한다. 핵심 설정은 `src/test/java` 폴더 구조, `build.gradle`의 Junit 의존성 추가, `Test Artifact`를 `Unit Tests`로 맞추는 것이다. 이후 `Calculator` 예제로 간단한 덧셈 테스트를 작성하고 `Run -> All Tests`로 실행하는 흐름을 보여준다.

## 본문 포인트별 발췌
> 안드로이드 Instrumentation 테스트의 경우에는 안드로이드 장치 또는 시뮬레이터에 직접 테스트 코드를 올려서 테스트가 진행됩니다.
> 이러한 과정 때문에 비교적 느립니다.
> 이 방식에서는 보통 Junit을 기반으로 다양한 Mocking 라이브러리 또는 Robolectric과 같은 프레임워크를 사용하여 테스트가 진행됩니다.
> app의 build.gradle 파일을 열어서 dependency에 testCompile 구문을 추가합니다.
> 테스트 java 폴더에서 우클릭 -> Run -> All Tests를 통해 테스트를 실행할 수 있습니다.
