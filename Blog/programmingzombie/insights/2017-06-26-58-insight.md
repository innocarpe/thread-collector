## 핵심 주장 (≤5)
1. USB 연결선이나 충전 단자 문제를 피하기 위해 물리적 연결 없이 APK를 설치·실행하는 방법이 필요하다.
2. ADB를 사용하면 물리적 연결 없이도 원하는 안드로이드 단말에 APK를 설치하고 실행해볼 수 있다.
3. 무선 설치를 시작하기 전에 단말의 USB debugging을 활성화해야 한다.
4. Android Studio 터미널에서 adb를 쓰려면 SDK 내부 adb 실행파일의 경로를 전역에서 사용할 수 있게 지정해야 한다.
5. 초기 USB 연결 후 `adb tcpip 5555`, `adb connect device_ip_address`, `adb devices` 순으로 진행하면 Wi-Fi로 단말 연결 여부를 확인할 수 있다.

## 공개된 숫자·지표
- TCP/IP 대기 포트: 5555 (기간: 설정 시점, 출처: "$ adb tcpip 5555")
- 개발자 옵션 기본 숨김 기준 버전: Android 4.2 이상 (기간: 해당 OS 버전 기준, 출처: "Android 4.2 이상 버전에서는 개발자 옵션 화면이 기본적으로 숨겨져 있다.")
- 빌드번호 탭 횟수: 일곱 번 (기간: 개발자 옵션 활성화 시점, 출처: "이 옵션을 표시하려면 각 안드로이드폰 단말의 설정 > 디바이스정보으로 이동하여 빌드번호를 일곱 번 누른다.")

## 언급된 도구·서비스
- ADB: USB 없이 APK를 단말에 설치하고 실행하기 위한 연결 도구
- Android Studio: Terminal에서 adb 명령을 실행하는 개발 환경
- Terminal: Android Studio 기준으로 adb 명령을 입력하는 위치
- USB debugging: adb 사용을 위해 단말에서 먼저 활성화해야 하는 개발자 옵션
- Wi-Fi: 물리적 선연결 없이 단말과 연결하는 통신 방식

## 언급된 다른 creator·앱
- 없음

## 복제 가능한 전술 (≤3)
1. 개발용 안드로이드 단말을 Wi-Fi ADB 대상으로 전환해 무선 설치 환경을 만든다.
   - 구체적 스텝: 단말에서 USB debugging을 활성화한다. Android Studio 터미널에서 adb 실행 경로가 잡혀 있는지 확인한다. USB로 단말을 연결한 뒤 `adb tcpip 5555`를 실행한다. USB 케이블을 분리하고 단말의 현재 Wi-Fi IP를 확인한다. `adb connect device_ip_address`로 연결한 뒤 `adb devices`로 `device_ip_address:5555 device`가 표시되는지 확인한다.
   - 예상 리소스: 안드로이드 단말, Android Studio, adb, USB 케이블(초기 1회 연결용), Wi-Fi 네트워크
   - 예상 효과: 물리적인 선연결 없이도 대상 단말에 연결해 빌드 및 실행 시 해당 단말을 사용할 수 있다.

## 원문 요약 (≤5문장)
글은 안드로이드 개발 중 USB 연결선 문제나 충전 단자 손상 경험 때문에 물리적 연결 없이 APK를 설치·실행하는 방법을 찾은 과정을 설명한다. 작성자는 ADB를 이용하면 USB 없이도 원하는 단말에 APK를 설치하고 실행할 수 있다고 말한다. 이를 위해 먼저 단말에서 USB debugging을 켜고, Android Studio 터미널에서 adb 실행 경로를 사용할 수 있게 설정해야 한다. 이후 USB로 한 번 연결한 상태에서 `adb tcpip 5555`를 실행하고, 케이블을 분리한 뒤 단말의 Wi-Fi IP로 `adb connect`를 수행한다. 마지막으로 `adb devices`로 연결을 확인하면 Android Studio에서 해당 단말로 빌드 및 실행할 수 있다.

## 본문 포인트별 발췌
> 하지만 이게 오래 하다보면 연결선에 문제가 생기는 경우도 있었고, 폰에 충전 단자가 손상되어 폰 충전이 되지 않는 현상을 겪은 적이 있었다.
> ADB를 알게 되었고, 굳이 물리적 연결을 하지 않더라도 APK파일을 내가 원하는 단말에 설치하고 실행해볼 수 있었다.
> USB를 통해 연결된 기기에서 adb를 사용하려면, 기기 시스템 설정의 개발자 옵션 아래에서 USB debugging을 활성화해야 한다.
> ADB 명령어를 Android Studio에서 바로 사용한다고 먹히질 않는다. 이유는 sdk내부에 존재하는 adb 실행파일의 경로를 지정해주지 않았기 때문이다.
> Android studio를 빌드 및 실행 해보면 해당 단말이 연결 되어 있는 것을 확인 할 수 있다.
