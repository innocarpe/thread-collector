# adam-wathan 반복 패턴 분석

## 반복 사용 전술
- 작업 공개로 수요를 검증: 최소 4번 등장. 이메일 샘플 챕터, 트위터 진행 공유, 라이브스트리밍, 무료 스크린캐스트를 통해 출시 전 관심을 먼저 만든다. 책 런치에서는 샘플 챕터와 업데이트 메일로 리스트를 1,500명 수준까지 키웠고, Tailwind는 라이브스트림 반응이 오픈소스 공개의 직접 계기가 됐다.
- 작은 단위로 시작해 반복 후 추상화: 최소 5번 등장. 제품은 작은 tripwire나 얼리 버전으로, 기술 설계는 유틸리티나 단순 패턴으로 먼저 만들고 반복이 확인되면 컴포넌트·프레임워크·코스로 승격한다. CSS 글, Vue 글, Tailwind 탄생 서사 전반에 같은 사고가 반복된다.
- 실제 사용 사례로 검증: 최소 4번 등장. 추상 이론보다 직접 만든 프로젝트, 브라우저 재현 테스트, 서로 다른 제품 적용 사례를 통해 주장한다. Safari 확대 테스트, Next.js 레이아웃 패턴 비교, Tailwind를 여러 프로젝트에 이식한 사례가 대표적이다.

## 일관되게 사용하는 도구
- 이메일 리스트/ConvertKit: 출시 준비와 판매 전환용. 책 런치에서는 샘플 배포, 미구매자 후속 메일, 마감 공지까지 이메일이 핵심 채널이었다.
- Twitter: 아이디어 검증, 진행 공유, 배포 증폭용. 책 출시, Tailwind 초기 공개, 독자 피드백 수집에 반복 사용한다.
- 라이브스트리밍·스크린캐스트·YouTube: 교육과 마케팅을 동시에 수행하는 도구. 개념이 어려운 주제는 무료 영상으로 진입장벽을 낮추고, 심화 코스나 제품으로 연결한다.
- Tailwind CSS, Vue.js, Next.js, PostCSS: 단순 구현 도구가 아니라 자신의 문제 해결 철학을 실험하는 매체로 쓴다. 특히 PostCSS 전환은 유지보수 자신감과 확장성을 키운 전환점이었다.

## 반복되는 수익화 패턴
- 무료 콘텐츠에서 유료 심화 상품으로 전환: 책 샘플, 블로그 글, 무료 스크린캐스트로 관심을 만들고 책·코스·UI 킷으로 연결한다.
- 티어드 패키징과 업셀: 2017년 책 런치에서 $29, $59, $135 패키지로 3일간 $61,392를 만들었고, 평균 판매가는 $67.84였다.
- 오픈소스에서 상업 제품으로 확장: Tailwind CSS 커뮤니티를 먼저 키운 뒤 Tailwind UI를 출시했고, 출시 약 5개월 만에 매출 200만 달러 돌파 직전, Tailwind Labs는 2년 미만에 400만 달러 이상 매출을 만들었다.
- 기존 교육 비즈니스를 현금흐름 기반으로 유지하며 신사업에 재투자: Test-Driven Laravel, Advanced Vue Component Design, Refactoring UI가 Tailwind 전업 전환의 안전판 역할을 했다.

## 핵심 믿음·철학
- 반복되지 않은 추상화는 경계하고, 현실에서 검증된 패턴만 승격한다.
- 교육도 제품도 “먼저 보여주고, 반응을 보고, 그다음 확장”이 기본 프레임이다.
- 복잡성은 새 기능보다 새 규칙과 새 추상화에서 온다고 본다.
- 원문 인용: "Instead, **think about _dependency direction._**"
- 원문 인용: "it's better to err on the side of \"too many slot props\" than too few"
- 원문 인용: "The utilities were the only things that were truly “portable”, while the component styles were always too opinionated to reuse on another design."
- 원문 인용: "This is the benefit of working in public"

## 성장 궤적
2017년의 Adam Wathan은 소규모 이메일 리스트와 트위터 팔로워를 기반으로 디지털 책을 런치해 독립 기반을 만들었다. 이 시기 핵심 역량은 런치 설계, 패키징, 직접 판매였다. 2017~2018년에는 CSS와 Vue 글에서 자신의 기술 철학을 정교화했다. 추상적 모범 사례를 반박하고, 유틸리티·렌더리스·scoped slots 같은 조합 가능한 설계를 밀어붙이며 “복잡성을 줄이는 시스템 사고”를 다듬었다. 2018년에는 코스, 팟캐스트, 라이브스트림, 발표를 병행하면서 교육 크리에이터이자 도구 제작자로 확장했다. 2019년에는 Tailwind CSS를 가장 큰 임팩트 프로젝트로 규정하고 전업 전환을 선언하며, 개인 상품 비즈니스에서 생태계 운영자로 피벗했다. 2020년에는 Tailwind를 실패한 사이드 프로젝트의 부산물에서 멀티밀리언 달러 사업으로 키운 과정을 정리하며, 자신의 경쟁력이 단순 제작이 아니라 반복 가능한 문제를 공개적으로 제품화하는 능력에 있음을 증명했다.
