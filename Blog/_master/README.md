# Blog/_master

7명 creator의 개별 replication-playbook을 종합해 1인 앱 포트폴리오 사업자를 위한 크로스 크리에이터 마스터 플레이북을 자동 생성한 폴더다. 개별 creator 분석을 읽기 전에 이 폴더를 먼저 참조하면 전략 방향과 실행 순서를 한 번에 잡을 수 있다.

## 산출물

| 파일 | 설명 |
|------|------|
| [master-playbook.md](./master-playbook.md) | 7명 종합 마스터 플레이북 — 수렴 패턴, 전략 비교, 30/60/90 실행 플랜, 도구 스택, 함정, 수치 베이스라인, 의사결정 트리 |
| [quick-reference.md](./quick-reference.md) | 1페이지 치트시트 — 핵심 원칙 5가지, Day 0 체크리스트, 판단 기준 숫자표, 핵심 파일 링크 |

## 재생성 방법

이 폴더의 산출물을 새로 생성하거나 업데이트하려면:

1. 7개 replication-playbook.md를 모두 읽는다.
   - `Blog/programmingzombie/insights/replication-playbook.md`
   - `Blog/sebastian-rohl/insights/replication-playbook.md`
   - `Blog/yuma-ueno/insights/replication-playbook.md`
   - `Blog/adam-wathan/insights/replication-playbook.md`
   - `Blog/roman-koch/insights/replication-playbook.md`
   - `Blog/weyoume/insights/replication-playbook.md`
   - `Blog/hussein-el-feky/insights/replication-playbook.md`
2. 각 creator의 `overview.md`, `patterns.md`를 보조 참고로 읽는다.
3. `master-playbook.md` 구조(TL;DR, 7명 매핑, Part 1~7)에 따라 새로 작성한다.
4. `quick-reference.md`를 master-playbook.md 내용에서 추출해 업데이트한다.
5. 원문에 없는 수치 창작 금지. 추측이면 "추정"이라고 명시.
