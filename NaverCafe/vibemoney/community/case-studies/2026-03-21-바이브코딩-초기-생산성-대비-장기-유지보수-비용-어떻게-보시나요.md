# 바이브코딩, 초기 생산성 대비 장기 유지보수 비용 어떻게 보시나요?

요즘 바이브코딩 관련해서 찾아보는 중인데
확실히 빠르게 결과 뽑는 데는 효율적인 방식 같더라고요.
​
근데 한편으로는
설계 과정을 최소화하는 방식이라
프로젝트 규모 커질수록 유지보수 비용이 더 커지는 구조 아닌가 싶기도 합니다.
​
특히 팀 단위 협업에서는
코드 일관성이나 컨텍스트 공유 측면에서 한계가 있을 것 같은데
​
실제 경험 있으신 분들은
이 부분 어떻게 보시는지 궁금합니다.
