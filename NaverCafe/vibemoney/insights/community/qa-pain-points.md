## 2. 초보자 단계별 막힘 포인트

**진입 경로**: 스레드(Threads.net)에서 만능시바님 게시글 → 카페 가입 → "어디서 시작해야 하나" 질문

### 단계 1: 개념 이해 (가장 많이 막힘)
- "바이브코딩이 뭔가요?" 수준부터 시작하는 경우 다수
- "커서, 클로드, 안티그래비티, GPT 다 다른 건가요?" — 개념 구분 자체가 안 됨
- "AI 구독이 왜 여러 개 필요해요?" — 모델 vs IDE vs 플랫폼 구분 안 됨
- **진단**: 진입 장벽이 지식보다 용어. 최소한의 용어 사전이 없으면 진도 못 나감.

### 단계 2: 환경 설정 / 툴 선택
- 안티그래비티 설치 → MCP 연결 시 채팅 중단 오류 (article_id 126)
- 오픈클로 권한 설정 이해 못함 (article_id 4009)
- Windows vs Mac 차이 질문 반복 (article_id 2356)
- 맥미니 구매 여부 고민 — 36건 이상 언급. "몰트봇 때문에 맥미니 열풍" (article_id 2220)
- **진단**: 환경 설정 단계에서 이미 큰 비용 의사결정(맥미니 구매)을 해야 한다는 인식 → 진입 심리적 장벽 높아짐.

### 단계 3: 첫 앱/서비스 만들기
- "뭘 만들지 모르겠어요" — 아이디어 부재가 첫 실행 장벽 (article_id 142, 2326, 6576)
- "만들다 보면 산으로 가요" — AI가 기억을 잃거나 의도와 다른 기능 개발 (article_id 227)
- GitHub 사용법 모름 → 롤백 불가 → "처음부터 다시" 반복 (article_id 227 팁 글에서 언급)
- **진단**: "첫 프로젝트 완성"까지 기술 막힘보다 AI 컨텍스트 관리 실패가 더 빈번한 좌절 원인.

### 단계 4: 배포
- "배포에서 막혀있어요" (article_id 385) — 비교적 구체적이지만 짧은 언급
- "실제 운영으로 돌아가는 서비스로 업그레이드하고 싶어요" (article_id 4168)
- VPS 서버 선택 (article_id 6964)
- **진단**: Vercel 무료 배포까지는 어찌어찌 하지만 그 이후(커스텀 도메인, 서버, DB 연결)에서 막힘.

### 단계 5: 수익화
- 결제 시스템 연동: PG사 계약 복잡성 (article_id 6698) — "토스페이, 카카오페이 같은 걸 바로 붙일 수 있나요?" 수준
- 첫 고객 어디서 찾냐: 지인 소개 vs SNS vs 크몽/외주 플랫폼 (article_id 4013)
- "마케팅이 없으면 수익화가 안 된다"는 인식은 있는데 방법 모름 (article_id 6567)
- **진단**: 기술 완성 후 비기술 영역(마케팅, 결제, 세금/계약)에서 막힘. 이 구간에서 포기율 높을 것.

## 3. 반복적으로 나오는 불안/고민

### "나도 될까?" 류 — 가능성 검증 욕구

가장 압도적인 심리 패턴. 전체 QnA의 30~40% 추정.

- "코딩 아예 모르는 문과인데 가능한가요?" (article_id 216)
- "비개발자는 어디까지 가능한가요?" (article_id 220)
- "일자무식도 가이드 보고 개발이 가능한가요?" (article_id 328)
- "저 진짜 아무것도 모르거든요!!" (article_id 2749)
- "완전 생짜 초보입니다. 무엇부터 시작해야 할까요?" (article_id 4354)
- "쌩초보는 무엇부터 시작해야 합니까" (article_id 6970, 2026-04-11 — 최근까지 지속)

**패턴 분석**: 이 질문들은 진짜 정보를 구하는 게 아님. 커뮤니티에서 "맞아 나도 됐어"라는 사회적 허락을 받고 싶은 것. 답변이 있어도 같은 질문이 수백 번 반복되는 이유.

### "이게 정말 돈이 되나?" 류 — 수익화 현실 불신

- "바이브코딩으로 실제로 돈 버는 분들 계신가요?" (article_id 4013)
- "직장인분들 혹시 바이브코딩으로 수익화 성공하신분들 있나요?" (article_id 515)
- "실제로 수익화 성공하신분 계신가요?" (article_id 4112)
- "바이브 코딩이 수익이 되나요?" (article_id 4078)
- "부업을 본업으로 바꾸기까지는 얼마나 걸릴까요?" (article_id 588)

**패턴 분석**: 스레드/SNS에서 성공담은 많이 봤지만 "진짜인지 모르겠다"는 의심과 검증 욕구. 구체적인 수익 구조와 기간이 궁금한 것. 추상적인 "가능합니다"로는 해소 안 됨.

### "지금 해도 늦지 않았나?" 류 — 진입 타이밍 불안

- "미래가 어떻게 될까요. 5년 안에 AI로 일자리 많이 없어지겠죠?" (article_id 258)
- "진작 할걸 싶기도 하지만 지금이라도 여기 온 거 다행이라 생각합니다" (article_id 313)
- "시류에 편승하고자 배우려고 하는데..." (article_id 587)

**패턴 분석**: FOMO(놓칠 것 같은 두려움)와 자책이 혼재. 커뮤니티 활동이 곧 불안 해소 행동.

### "올인을 해야 하나?" 류 — 집중 vs 분산 딜레마

- "육아하며 돈 벌고 싶어서 이것저것 하고 있어요. 근데 진짜 잡다하게 하니 뭐 하나 제대로 하는 게 없네요" (article_id 565)
- "개발하는 사람인데 수익화 영역은 개발 능력보단 영업 + SNS라 생각해서 배우러 왔는데 바이브코딩만으로 충분한가요?" (article_id 523)

## 4. 해결 안 된 채 방치되는 질문들

파일 크기 500 bytes 미만 + 이후 반복 질문이 없는 패턴 기준.

### 답변이 거의 없는 구조적 질문들

| 질문 | 왜 방치되는가 |
|------|-------------|
| "마케팅 말고 바이브코딩 수익화 — 어떻게 가능한 건가요?" (article_id 4254) | 정답이 없음. 커뮤니티가 "만드는 법"에 집중되어 있고 "파는 법"에는 약함 |
| "자동화 수익화 방향 문의 — 실제 수익화 아이템과 워크플로우가 궁금합니다" (article_id 3928) | 성공한 사람이 구체적 내용 공개 꺼림. 커뮤니티에 성공 사례가 추상적으로만 공유됨 |
| "결제 시스템 어떻게 붙이나요?" (article_id 6698) | PG사 계약 복잡성 + 개인사업자 등록 등 비기술 영역. 바이브코딩 커뮤니티의 전문성 범위 밖 |
| "비용효율적이면서 성능 좋은 코딩툴" (article_id 4118) | 계속 새 툴이 나와서 답이 빠르게 낡음. 커뮤니티 내 권위자 답변 부재 |
| "서비스 비용은 어느 정도 생각하면 될까요?" (article_id 4028) | 서버비, API비, 구독비 통합 정리 자료 없음 |
| "해외를 대상으로도 자동화 가능할까요?" (article_id 6756) | 해외 SNS 정책, 법적 이슈 등 — 커뮤니티 경험 범위 밖 |
| "웹 제작 시 계약서 내용" (article_id 6850) | 법적/계약 영역. 커뮤니티가 답 못 해줌 |
| "클로드 코드 토큰 제한 어떻게 해요?" (article_id 4014) | Pro → Max 업그레이드 외 대안이 없음. 실용적 해법 부재 |

### 시간이 지나도 반복되는 패턴 (해결됐다면 사라졌을 것)
- "뭐부터 시작해야 하나요?"가 2025년 12월~2026년 4월까지 동일하게 반복 → 입문 가이드가 있어도 새 입문자가 계속 유입되거나, 기존 가이드가 검색이 안 됨
- "비개발자도 가능한가요?" — 수백 건의 긍정 답변에도 불구하고 계속 나옴 → 질문 의도가 정보 수집이 아닌 확신 받기

## 5. Temperstone이 해결할 수 있는 문제들

### 5-1. "내 수준/목적에 맞는 도구 조합 추천" 인터랙티브 가이드
**수요 근거**: "뭐 써야 해요?" 류 질문 40건+. 커뮤니티 답변은 매번 같은 답이지만 개인화가 없음.
**형태**: 간단한 설문(코딩 경험 / 목표 / 예산) → 맞춤 스택 추천 + 이유.
**Temperstone 연결**: 도구 추천 결과에 Temperstone 서비스/콘텐츠 자연스럽게 연결.

### 5-2. "첫 서비스 완성까지 체크리스트" 콘텐츠/툴
**수요 근거**: 아이디어 → 개발 → 배포 → 결제 연동 → 첫 고객까지의 전체 로드맵이 없음.
**특히 공백인 구간**: GitHub 사용법 + 배포 + PG사 결제 연동. 이 세 가지가 연속으로 막히는 구간.
**형태**: 단계별 체크리스트 + 막히는 포인트별 해결책 링크. 커뮤니티 글보다 정제된 형태.

### 5-3. "AI 도구 비용 최적화 가이드"
**수요 근거**: API vs 유료 계정 혼란, 가족 공유 꿀팁 수요, 오픈클로 비용 최적화 요청.
**형태**: "월 예산 N만원으로 쓸 수 있는 최적 조합" 포맷. 주기적 업데이트 필요.
**Temperstone 연결**: 뉴스레터 구독 유도 포인트로 활용 가능.

### 5-4. "수익화 첫 단계 가이드 — 기술 말고 비기술 영역"
**수요 근거**: "마케팅은 어떻게 하나요?", "첫 고객은 어디서 나왔나요?", "PG사 계약은 어떻게 하나요?" — 이 세 가지가 커뮤니티 내 완전 공백.
**형태**: "만들고 나서 할 일" 시리즈 콘텐츠. (결제 연동 → 첫 고객 찾기 → SNS 마케팅 기초 → 세금/사업자 기초)
**차별점**: 커뮤니티가 "만드는 법"에만 집중되어 있어 "파는 법"은 블루오션.

### 5-5. "나도 됐어" — 실제 수익화 사례 아카이브
**수요 근거**: "실제로 돈 버는 분 계신가요?" 반복 질문. 성공담에 대한 구체적 검증 욕구.
**형태**: 익명 또는 닉네임 기반의 구체적 수익화 케이스 아카이브. 금액/기간/방법 포함.
**주의점**: 커뮤니티에서 성공한 사람들이 구체적 수치를 공개하길 꺼리는 이유 파악 필요. 인터뷰 또는 설문 형태가 현실적.

### 5-6. 오픈클로 보안/설정 가이드
**수요 근거**: 오픈클로 관련 보안 불안 3건+, 권한 설정 혼란 1건. 2026년 2월 이후 언급 급증.
**형태**: "오픈클로 안전하게 쓰는 법" 1페이지 가이드. 보안 이슈 팩트체크 + 설정 방법.
**기회**: 커뮤니티 내에서 아직 명확한 가이드가 없음. 선점 가능.
