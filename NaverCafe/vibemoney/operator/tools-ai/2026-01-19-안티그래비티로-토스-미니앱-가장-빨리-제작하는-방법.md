# 안티그래비티로 토스 미니앱, 가장 빨리 제작하는 방법

거의 근 2주간 SNS나 단톡 등 활동이 거의 없었는데요,
그동안 SaaS 제작 + 토스 미니앱 제작 등 이것저것 바빴습니다.
당분간도 이것 때문에 무척 바쁠 예정이네요.
​
오늘은, 제가 삽질한 내용을 바탕으로
안티그래비티로 토스 미니앱,
일단 만들고 빨리 테스트 돌리는 법 정리했습니다.
​
AI 딸깍으로 될 거라고 생각하시면 직접 해보세요.
아마 백이면 백.. 시간 갈게 되실 겁니다..
​
암튼, 시간 아껴드리려고 쓰는 글입니다.
시작합니다.
                            
                        
                    
                    
                                
                    
                        
                            
                                
                            
                        
                    
                                
                    
                        
                            
                                    
​
Toss Mini App 제작하기 (React + TypeScript)
Toss Mini App 개발 환경 구축부터 첫 테스트 화면 출력까지의 과정을 정리한 가이드입니다.
​
참고로 토스 미니앱은 Webview 방식, 네이티브 방식 2개 있습니다.
네이티브 생소하시면 바로 웹뷰로 하시면 됩니다.
복잡한 앱 자체가 없는 생태계라 웹뷰로 해도 크게 문제 없습니다.
​
참고로 진행 과정 이해 안 되시면
복사해서 프롬프트에 붙여넣으세요.
​
1. 프로젝트 초기화
1.1 Vite 프로젝트 생성
Vite를 사용하여 React + TypeScript 기반의 프로젝트를 생성합니다.
​
React-TS 템플릿 사용
npm create vite@latest . -- --template react-ts 
​
1.2 핵심 패키지 설치
Toss Mini App 개발에 필수적인 프레임워크와 디자인 시스템(TDS)을 설치합니다.
​
Bash
# Toss 웹 프레임워크 설치 npm install @apps-in-toss/web-framework
# Toss 디자인 시스템(TDS) 모바일 패키지 설치
# (의존성 충돌 발생 시 --legacy-peer-deps 옵션을 사용하세요)
​
npm install @toss/tds-mobile --legacy-peer-deps 
                            
                        
                    
                    
                                
                    
                        
                            
                                
                            
                        
                    
                                
                    
                        
                            
                                    
2. 환경 구성 및 설정
2.1 ait init 실행
기본 환경 구성을 위해 ait init 명령어를 실행합니다.
​
# 앱 이름을 설정하여 초기화 진행
npx ait init --template framework --app-name 1min-vible 
​
2.2 granite.config.ts 설정
프로젝트 루트에 있는 granite.config.ts 파일을 수정합니다.
실기기 테스트를 위해 본인의 로컬 IP 주소를 정확히 입력해야 합니다.
이건 AI 딸깍 아니고, ipconfig 해서 알아낸 다음 IP 주소 정확히 넣으세요.
​
TypeScript
                            
                        
                    
                    
                                
                    
                        
                            
                                
                                    import { defineConfig } from '@apps-in-toss/web-framework/config';

export default defineConfig({
  appName: '앱 네임 적으세요(콘솔 참고)',
  brand: {
    displayName: '보이는 앱 이름(콘솔 참고)',
    primaryColor: '#3182F6',
    icon: '',
  },
  web: {
    host: '아이피 적으세요',
    port: 5173,
    commands: {
      dev: 'vite --host',
      build: 'vite build',
    },
  },
  permissions: [],
});

                                
                            
                        
                    
                    
                                
                    
                        
                            
                                    
2.3 package.json 스크립트 수정
Vite와 Metro 서버를 동시에 관리할 수 있도록 scripts 부분을 수정합니다.
JSON
"scripts": { "dev": "granite dev", "build": "granite build", ... } 
​
이 부분 안 하시면
테스트 시 '메트로 어쩌고' 에러 발생합니다.
                            
                        
                    
                    
                                
                    
                        
                            
                                
                            
                        
                    
                                
                    
                        
                            
                                    
3. 서비스 구현
AI를 통해 서비스 구현하되, 기존 프레임웍 등
주어진 환경에서 벗어나면 바로 에러 뱉어냅니다.
​
SKills.md나 Rules 잘 세팅해서 하세요.
                            
                        
                    
                    
                                
                    
                        
                            
                                
                            
                        
                    
                                
                    
                        
                            
                                    
4. 서버 실행 및 점검
4.1 포트 충돌 해결 (필요 시)
이미 사용 중인 포트가 있다면 프로세스를 종료해야 합니다. (Windows 기준)
PowerShell
# 포트 사용 중인 PID 확인 netstat -ano | findstr :8081 netstat -ano | findstr :5173 # 프로세스 종료 taskkill /F /PID <PID번호> 
​
4.2 ADB 서버 실행합니다.
CMD에서 2개 명령어 입력합니다.
                            
                        
                    
                    
                                
                    
                        
                            
                                
                                    adb reverse tcp:8081 tcp:8081
adb reverse tcp:5173 tcp:5173
                                
                            
                        
                    
                    
                                
                    
                        
                            
                                    
​
4.3 개발 서버 시작
아래 명령어를 실행하면 Vite(웹) 서버와 Metro(네이티브) 서버가 동시에 가동됩니다.
​
npm run dev 
                            
                        
                    
                    
                                
                    
                        
                            
                                
                            
                        
                    
                                
                    
                        
                            
                                    
5. 요약 및 주의사항
IP 주소 확인: 실제 기기 테스트 시 granite.config.ts의 host IP가 서버 PC의 실제 로컬 IP와 일치해야 합니다.
의존성 관리: 설치 중 에러가 발생하면 --legacy-peer-deps 옵션을 적극 활용하세요.
빌드: 최종 배포용 .ait 파일은 npm run build를 통해 생성할 수 있습니다.
